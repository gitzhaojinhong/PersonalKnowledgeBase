
## 1.1 优化思路

兑换码不是简单的随机字符串，它背后有严格的设计约束。业务需求可归纳为六点：

- **可读性好**：长度不超过10个字符；字符集限定为24个大写字母+8个数字（排除易混淆的0、1、I、O）
- **数据量大**：优惠活动频繁，需要支持10亿以上的兑换码量
- **唯一性**：10亿兑换码必须全局唯一，不能重复
- **不可重兑**：必须高效校验兑换状态，避免重复兑换
- **防止爆刷**：兑换码规律性不能太强，不能轻易被人猜测或伪造
- **高效**：生成、校验的算法必须保证效率，避免对数据库造成较大压力

**常见方案的排除：**

| 方案 | 问题 |
|------|------|
| UUID | 128位，远超50位限制，转码后长度很长 |
| Snowflake | 64位，超出50位限制 |
| 纯随机字符串 | 无法保证唯一性，且难以校验是否被篡改 |

**最终选择：自增id + Base32转码 + 按位加权签名 + BitMap验重。**

自增id仅需32位即可支持42亿数据量，完全满足10亿以上需求；Base32将二进制数映射为要求的字符集；签名防止爆刷和伪造；BitMap实现O(1)重兑校验，全程不查数据库。

## 1.2 方案设计

### 1.2.1 兑换码结构设计

兑换码分为**明文**（二进制）和**密文**（字符串）。

**明文结构（共50位）：**

```
| 校验码(14位) | 新鲜值(4位) | 序列号(32位) |
     ↑             ↑            ↑
   防篡改      区分优惠券     自增唯一标识
```

- **序列号**：单调递增数字，由Redis自增生成，保证全局唯一
- **新鲜值**：取优惠券id的低4位，同一张券的兑换码都有相同标记
- **载荷**：新鲜值(4位) + 序列号(32位) = 36位
- **校验码**：对载荷按位加权求和后对2^14取余，防止伪造和篡改

**密文结构：**

50位二进制通过Base32转码，每5位对应一个字符，恰好得到**10位字符串**。

### 1.2.2 生成流程

```
优惠券id → 取低4位得 fresh
               ↓
Redis自增 → 得 serialNum（32位）
               ↓
        拼接 payload = fresh(4位) + serialNum(32位)
               ↓
        按位加权计算 → 校验码 checkCode（14位）
               ↓
        用checkCode后4位选异或密钥 key
               ↓
        payload ^= key 得混淆后的 payload2
               ↓
        拼接明文 = checkCode(14位) + payload2(36位)
               ↓
        Base32转码 → 10位兑换码字符串
```

### 1.2.3 解析与验签流程

```
兑换码字符串 → Base32解码 → 50位数字
               ↓
        高14位取 c1（原校验码）
        低36位取 payload（混淆后的载荷）
               ↓
        用c1后4位选异或密钥 key
               ↓
        payload ^= key 还原原始载荷
               ↓
        高4位得 fresh，整体重新加权计算 → c2
               ↓
        c1 == c2 ? 有效兑换码 : 被篡改/无效
               ↓
        低32位得 serialNum（自增序列号）
```

### 1.2.4 重兑校验方案

兑换状态只有两种：已兑换 / 未兑换，天然适合用**BitMap**存储。

- 以兑换码的**自增序列号**作为偏移量（bit位索引）
- Redis中BitMap最大支持2^32个bit位，而序列号刚好是32位，完美匹配
- 校验时只需执行 `GETBIT` 命令，时间复杂度O(1)，完全不用访问数据库

### 1.2.5 异步生成方案

发放优惠券时，如果领取方式是**兑换码方式**且优惠券状态为**待发放**，则需要生成兑换码。

**为何选择异步生成：**

- 兑换码数量等于发放数量（可能上千），同步生成接口耗时长
- 发放优惠券是管理端低频操作，对实时性要求不高
- 异步生成可大幅降低接口响应时间，提升管理员体验

**线程池配置：**

| 参数 | 值 | 说明 |
|------|-----|------|
| 核心线程数 | 2 | 低频业务，无需太多常驻线程 |
| 最大线程数 | 5 | 上限控制在CPU核数附近 |
| 队列容量 | 200 | 缓冲突发任务 |
| 线程名前缀 | exchange-code-handler- | 便于日志排查 |
| 拒绝策略 | CallerRunsPolicy | 极端情况下由调用线程执行 |

**异步生成流程：**

```
发放优惠券接口
    ↓
判断：领取方式 == 兑换码 && 原状态 == 待发放
    ↓
提交异步任务到线程池
    ↓
异步任务内部：
    1. Redis自增获取一批序列号（增量 = 发放数量）
    2. 遍历序列号，逐个调用 CodeUtil.generateCode(serialNum, couponId)
    3. 批量组装 ExchangeCode 对象
    4. saveBatch 批量写入数据库
    5. Redis ZSet 记录该优惠券的最大序列号（member=couponId, score=maxSerialNum）
```

> 注意：只有状态从**待发放**变为发放时才生成兑换码。如果优惠券被暂停后再次发放，不会重复生成。

## 1.3 代码实现

### 1.3.1 Base32工具类

Base32将二进制数每5位分为一组，每组映射为一个字符。字符表已排除易混淆字符。

```java
package com.tianji.promotion.utils;

import cn.hutool.core.text.StrBuilder;

public class Base32 {
    private final static String baseChars = "6CSB7H8DAKXZF3N95RTMVUQG2YE4JWPL";

    public static String encode(long raw) {
        StrBuilder sb = new StrBuilder();
        while (raw != 0) {
            int i = (int) (raw & 0b11111);
            sb.append(baseChars.charAt(i));
            raw = raw >>> 5;
        }
        return sb.toString();
    }

    public static long decode(String code) {
        long r = 0;
        char[] chars = code.toCharArray();
        for (int i = chars.length - 1; i >= 0; i--) {
            long n = baseChars.indexOf(chars[i]);
            r = r | (n << (5 * i));
        }
        return r;
    }
}
```

### 1.3.2 兑换码生成与解析工具

```java
package com.tianji.promotion.utils;

import com.tianji.common.constants.RegexConstants;
import com.tianji.common.exceptions.BadRequestException;

public class CodeUtil {
    /**
     * 异或密钥表，用于最后的数据混淆
     */
    private final static long[] XOR_TABLE = {
            61261925471L, 61261925523L, 58169127203L, 64169927267L,
            64169927199L, 61261925629L, 58169127227L, 64169927363L,
            59169127063L, 64169927359L, 58169127291L, 61261925739L,
            59169127133L, 55139281911L, 56169127077L, 59169127167L
    };
    private final static int FRESH_BIT_OFFSET = 32;
    private final static int CHECK_CODE_BIT_OFFSET = 36;
    private final static int FRESH_MASK = 0xF;
    private final static int CHECK_CODE_MASK = 0b11111111111111;
    private final static long PAYLOAD_MASK = 0xFFFFFFFFFL;
    private final static long SERIAL_NUM_MASK = 0xFFFFFFFFL;
    /**
     * 序列号加权运算的秘钥表（16组）
     */
    private final static int[][] PRIME_TABLE = {
            {23, 59, 241, 61, 607, 67, 977, 1217, 1289, 1601},
            {79, 83, 107, 439, 313, 619, 911, 1049, 1237},
            {173, 211, 499, 673, 823, 941, 1039, 1213, 1429, 1259},
            {31, 293, 311, 349, 431, 577, 757, 883, 1009, 1657},
            {353, 23, 367, 499, 599, 661, 719, 929, 1301, 1511},
            {103, 179, 353, 467, 577, 691, 811, 947, 1153, 1453},
            {213, 439, 257, 313, 571, 619, 743, 829, 983, 1103},
            {31, 151, 241, 349, 607, 677, 769, 823, 967, 1049},
            {61, 83, 109, 137, 151, 521, 701, 827, 1123},
            {23, 61, 199, 223, 479, 647, 739, 811, 947, 1019},
            {31, 109, 311, 467, 613, 743, 821, 881, 1031, 1171},
            {41, 173, 367, 401, 569, 683, 761, 883, 1009, 1181},
            {127, 283, 467, 577, 661, 773, 881, 967, 1097, 1289},
            {59, 137, 257, 347, 439, 547, 641, 839, 977, 1009},
            {61, 199, 313, 421, 613, 739, 827, 941, 1087, 1307},
            {19, 127, 241, 353, 499, 607, 811, 919, 1031, 1301}
    };

    /**
     * 生成兑换码
     * @param serialNum 递增序列号（32位）
     * @param fresh 新鲜值（建议取优惠券id低4位）
     * @return 兑换码（10位字符串）
     */
    public static String generateCode(long serialNum, long fresh) {
        // 1. 计算新鲜值，只保留低4位
        fresh = fresh & FRESH_MASK;
        // 2. 拼接payload：fresh(4位) + serialNum(32位)
        long payload = fresh << FRESH_BIT_OFFSET | serialNum;
        // 3. 计算校验码
        long checkCode = calcCheckCode(payload, (int) fresh);
        // 4. payload做异或混淆
        payload ^= XOR_TABLE[(int) (checkCode & FRESH_MASK)];
        // 5. 拼接明文：校验码(14位) + payload(36位)
        long code = checkCode << CHECK_CODE_BIT_OFFSET | payload;
        // 6. Base32转码
        return Base32.encode(code);
    }

    private static long calcCheckCode(long payload, int fresh) {
        // 1. 根据fresh选一组加权码表
        int[] table = PRIME_TABLE[fresh];
        // 2. payload每4位乘加权数，求和，取最后14位
        long sum = 0;
        int index = 0;
        while (payload > 0) {
            sum += (payload & 0xf) * table[index++];
            payload >>>= 4;
        }
        return sum & CHECK_CODE_MASK;
    }

    /**
     * 解析并验证兑换码
     * @param code 兑换码字符串
     * @return 序列号（自增id）
     */
    public static long parseCode(String code) {
        if (code == null || !code.matches(RegexConstants.COUPON_CODE_PATTERN)) {
            throw new BadRequestException("无效兑换码");
        }
        // 1. Base32解码
        long num = Base32.decode(code);
        // 2. 低36位是payload
        long payload = num & PAYLOAD_MASK;
        // 3. 高14位是校验码
        int checkCode = (int) (num >>> CHECK_CODE_BIT_OFFSET);
        // 4. 异或还原
        payload ^= XOR_TABLE[(checkCode & FRESH_MASK)];
        // 5. 取高4位得fresh
        int fresh = (int) (payload >>> FRESH_BIT_OFFSET & FRESH_MASK);
        // 6. 重新计算校验码并比对
        if (calcCheckCode(payload, fresh) != checkCode) {
            throw new BadRequestException("无效兑换码");
        }
        return payload & SERIAL_NUM_MASK;
    }
}
```

package com.tianji.promotion.config;

import lombok.extern.slf4j.Slf4j;

import java.util.concurrent.Executor;
import java.util.concurrent.ThreadPoolExecutor;

@Slf4j
@Configuration
@EnableAsync
public class PromotionConfig {

    @Bean
    public Executor generateExchangeCodeExecutor() {
        ThreadPoolTaskExecutor executor = new ThreadPoolTaskExecutor();
        executor.setCorePoolSize(2);
        executor.setMaxPoolSize(5);
        executor.setQueueCapacity(200);
        executor.setThreadNamePrefix("exchange-code-handler-");
        executor.setRejectedExecutionHandler(new ThreadPoolExecutor.CallerRunsPolicy());
        executor.initialize();
        return executor;
    }
}

@Transactional
@Override
public void beginIssue(CouponIssueFormDTO dto) {
    // 1. 查询优惠券
    Coupon coupon = getById(dto.getId());
    if (coupon == null) {
        throw new BadRequestException("优惠券不存在！");
    }
    // 2. 状态校验：必须是待发放或暂停
    if (coupon.getStatus() != CouponStatus.DRAFT && coupon.getStatus() != PAUSE) {
        throw new BizIllegalException("优惠券状态错误！");
    }
    // 3. 判断是否是立刻发放
    LocalDateTime issueBeginTime = dto.getIssueBeginTime();
    LocalDateTime now = LocalDateTime.now();
    boolean isBegin = issueBeginTime == null || !issueBeginTime.isAfter(now);
    // 4. 更新优惠券状态
    Coupon c = BeanUtils.copyBean(dto, Coupon.class);
    if (isBegin) {
        c.setStatus(ISSUING);
        c.setIssueBeginTime(now);
    } else {
        c.setStatus(UN_ISSUE);
    }
    updateById(c);

    // 5. 判断是否需要生成兑换码
    //    领取方式必须是兑换码，且之前的状态必须是待发放（避免暂停后重复发放导致重复生成）
    if (coupon.getObtainWay() == ObtainType.ISSUE && coupon.getStatus() == CouponStatus.DRAFT) {
        coupon.setIssueEndTime(c.getIssueEndTime());
        codeService.asyncGenerateCode(coupon);
    }
}

**异步生成兑换码实现：**

```java
package com.tianji.promotion.service.impl;

import com.baomidou.mybatisplus.extension.service.impl.ServiceImpl;
import com.tianji.promotion.domain.po.Coupon;
import com.tianji.promotion.domain.po.ExchangeCode;
import com.tianji.promotion.mapper.ExchangeCodeMapper;
import com.tianji.promotion.service.IExchangeCodeService;
import com.tianji.promotion.utils.CodeUtil;
import org.springframework.data.redis.core.BoundValueOperations;
import org.springframework.data.redis.core.StringRedisTemplate;
import org.springframework.scheduling.annotation.Async;
import org.springframework.stereotype.Service;

import java.util.ArrayList;
import java.util.List;

import static com.tianji.promotion.constants.PromotionConstants.*;

@Service
public class ExchangeCodeServiceImpl extends ServiceImpl<ExchangeCodeMapper, ExchangeCode>
        implements IExchangeCodeService {

    private final StringRedisTemplate redisTemplate;
    private final BoundValueOperations<String, String> serialOps;

    public ExchangeCodeServiceImpl(StringRedisTemplate redisTemplate) {
        this.redisTemplate = redisTemplate;
        this.serialOps = redisTemplate.boundValueOps(COUPON_CODE_SERIAL_KEY);
    }

    @Override
    @Async("generateExchangeCodeExecutor")
    public void asyncGenerateCode(Coupon coupon) {
        Integer totalNum = coupon.getTotalNum();
        // 1. Redis自增获取一批序列号
        Long result = serialOps.increment(totalNum);
        if (result == null) {
            return;
        }
        int maxSerialNum = result.intValue();
        List<ExchangeCode> list = new ArrayList<>(totalNum);
        for (int serialNum = maxSerialNum - totalNum + 1; serialNum <= maxSerialNum; serialNum++) {
            // 2. 生成兑换码
            String code = CodeUtil.generateCode(serialNum, coupon.getId());
            ExchangeCode e = new ExchangeCode();
            e.setCode(code);
            e.setId(serialNum);
            e.setExchangeTargetId(coupon.getId());
            e.setExpiredTime(coupon.getIssueEndTime());
            list.add(e);
        }
        // 3. 批量保存到数据库
        saveBatch(list);
        // 4. 写入Redis ZSet，记录该优惠券的最大序列号
        redisTemplate.opsForZSet().add(COUPON_RANGE_KEY, coupon.getId().toString(), maxSerialNum);
    }
}
```

## 1.4 面试要点

---

**面试官：你们优惠券兑换码是如何生成的？请设计一个支持20亿以上唯一兑换码的方案，长度不超过10，只包含字母数字，且保证生成和校验高效。**

首先要考虑校验的高效性，最佳方案肯定是**自增序列号**。因为自增序列号可以借助**BitMap**验证兑换状态，完全不用查询数据库，效率极高。

要满足20亿的兑换码需求，只需要31个bit位就够了，也就是在Integer取值范围内。按32位来算，可支持42亿数据规模。

但仅用自增序列还不够，规律太明显，容易被人爆刷。所以还需要设计**加密验签算法**。具体思路如下：

- 32位自增序列每4位分为一组，共8组，转为10进制
- 提前准备16组加权数组作为密钥
- 生成时随机一个4位新鲜值（0~15），取对应组的加权数组
- 对8个数字按位加权求和，结果作为签名

最后把签名后14位、新鲜值4位、自增序列32位拼接，得到50位二进制数。再与一个大质数做异或运算混淆，最后基于Base32转码，恰好得到10位字符串，符合要求。

密钥包括加权数组和异或密钥表，不能泄露，必要时可定期更换。

验签时：Base32解码 → 异或还原 → 重新计算签名 → 比对一致性 → 取低32位得序列号 → BitMap查重。全程不访问数据库。

---

**面试官：你在项目中哪些地方用到过线程池？**

我在实现优惠券兑换码生成时使用了线程池。

发放优惠券时，如果发现领取方式是兑换码方式，需要生成兑换码。但兑换码数量可能很多，同步生成会导致接口耗时过长。因此采用线程池异步生成。

线程池参数配置：核心线程2，最大线程5，队列200，拒绝策略CallerRunsPolicy。因为发放优惠券是低频管理端操作，线程数无需太高。队列用于缓冲突发任务，极端情况下由调用线程处理，让管理员稍微等待即可。
