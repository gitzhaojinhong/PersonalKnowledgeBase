## 常见错误与注意事项

1. **忘记设置name属性**：表单控件没有name属性时，数据不会随表单提交，这是最常见的表单问题之一

2. **button标签忘记设置type**：button标签的type默认值是submit，如果不写type="button"，点击按钮会意外触发表单提交

3. **img标签忘记设置alt**：不仅影响SEO和无障碍访问，图片加载失败时用户也无法知道图片内容

4. **中文编码乱码**：确保文件保存编码和meta标签声明的编码一致（统一使用UTF-8）

5. **h1使用过多**：每个页面应只有一个h1，搜索引擎会根据h1判断页面主题

6. **标题层级跳跃**：不要从h1直接跳到h3，应保持层级递进（h1 > h2 > h3）

7. **p标签中嵌套块级元素**：p标签中不能放div、h1~h6、ul等块级元素

8. **a标签嵌套a标签**：a标签不能嵌套a标签，这是HTML规范明确禁止的

9. **重复id**：id在页面中必须唯一，重复id会导致CSS选择器和JavaScript操作行为不可预期

10. **使用br制造间距**：br是语义上的换行，不应作为增加间距的工具，间距应通过CSS控制

11. **表格用于布局**：虽然表格可以用于页面布局（早期Web的常见做法），但这是不语义化的做法，应使用CSS布局（Flexbox、Grid）

12. **HTML属性不写引号**：虽然省略引号在某些情况下可以工作，但当属性值包含空格时会出错。始终使用双引号是最安全的做法

13. **不闭合的标签**：虽然部分标签在HTML5中可以省略闭合标签（如li、p），但显式闭合所有标签是更好的编码习惯

14. **自动播放被阻止**：现代浏览器要求视频/音频自动播放时必须静音（muted），否则autoplay不会生效

15. **target="_blank"的安全风险**：使用target="_blank"打开新页面时，新页面可以通过window.opener访问原页面的location对象（可能被恶意利用）。建议添加rel="noopener noreferrer"：

```html
<a href="https://example.com" target="_blank" rel="noopener noreferrer">安全链接</a>
```
