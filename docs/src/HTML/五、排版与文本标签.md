## 5.1 语义化标签

### 5.1.1 什么是语义化

语义化是指使用具有明确含义的标签来表达内容的作用和结构，而不是只关注标签的视觉效果。

例如：
- h1标签的效果是文字变大变粗（视觉效果）
- h1标签的语义是"页面的一级标题"（语义含义）

**语义化原则**：标签的默认视觉效果不重要（后期可以通过CSS随意修改），**语义才是最重要的**。选择标签时应优先考虑"这段内容是什么"，而不是"这段内容看起来怎样"。

### 5.1.2 语义化的优势

1. **代码可读性好**：开发团队协作时，语义化的代码更容易理解和维护
2. **有利于SEO**：搜索引擎通过标签语义理解页面内容结构，合理使用标题标签能显著提升搜索排名
3. **无障碍访问**：屏幕阅读器等辅助设备依赖标签语义来为视障用户提供正确的朗读和导航
4. **便于维护**：结构清晰的代码在后期修改和扩展时效率更高

## 5.2 块级元素与行内元素

### 5.2.1 元素分类

HTML元素按显示方式分为两大类：

**块级元素**：独占一行（即使内容没有填满整行，后续内容也会从下一行开始）

常见块级元素：h1~h6、p、div、ul、ol、li、table、tr、td、th、form、fieldset、header、footer、nav、section、article、aside、blockquote、address、pre、hr

**行内元素**：不独占一行，与其他行内元素在同一行中从左到右排列，只有当一行排满时才会换行

常见行内元素：span、a、em、strong、img、input、label、code、sub、sup、br、cite、dfn、del、ins、q、kbd、var、b、i、u、small、mark、abbr

**行内块元素**（介于两者之间）：不独占一行，但可以设置宽度和高度

常见行内块元素：img、input（表单控件本质上都是行内块元素）

> 注意：块级和行内的区分是HTML层面的概念，实际显示效果可以通过CSS的display属性修改。但在选择标签时，仍应遵循HTML语义，而不是为了布局需要随意选择标签。

### 5.2.2 嵌套规则

1. 块级元素中可以包含行内元素和块级元素（块级元素几乎什么都能装）
2. 行内元素中可以包含行内元素，但一般不能包含块级元素
3. 特殊规则：
   - h1~h6不能互相嵌套（标题里不要再放标题）
   - p标签中不能包含h1~h6、p、div等块级元素

## 5.3 排版标签

排版标签用于组织页面的大段内容结构，是页面的"骨架"。

### 5.3.1 标题标签（h1~h6）

h1~h6共六级标题，h1最重要，h6最次要。标题标签是块级元素。

```html
<h1>一级标题 — 每个页面最好只有一个</h1>
<h2>二级标题</h2>
<h3>三级标题</h3>
<h4>四级标题</h4>
<h5>五级标题</h5>
<h6>六级标题</h6>
```

使用规范：
- 每个页面有且仅有一个h1（页面的主标题）
- 标题层级不要跳跃（h1直接到h3），应保持逻辑递进
- h1~h6不能互相嵌套

### 5.3.2 段落标签（p）

p标签用于定义一个段落，是块级元素。浏览器会在段落前后自动添加间距。

```html
<p>这是第一个段落。</p>
<p>这是第二个段落。</p>
```

> 注意：p标签中不能包含块级元素（h1~h6、p、div、ul等），否则浏览器的行为可能不符合预期。

### 5.3.3 通用容器（div）

div标签没有任何语义，是块级元素，用于将内容分组、整体布局。

```html
<div class="header">
    <h1>网站名称</h1>
    <p>副标题</p>
</div>
<div class="content">
    <p>主要内容区域</p>
</div>
```

类比：div就像一个大包装袋，可以装各种内容。当没有更合适的语义标签时，用div作为通用容器。

## 5.4 常用文本标签

文本标签用于包裹词汇、短语等细粒度内容，通常写在排版标签内部。文本标签绝大多数是行内元素。

### 5.4.1 em标签

语义：要着重阅读的内容。默认以斜体显示。

```html
<p>学习HTML是前端开发的 <em>第一步</em>，也是最基础的一步。</p>
```

### 5.4.2 strong标签

语义：十分重要的内容，语气比em更强。默认以加粗显示。

```html
<p><strong>警告：</strong>请勿在正式环境中使用测试账号。</p>
```

em和strong的区别：em表示"强调"（语气上的重读），strong表示"重要"（内容本身的重要性）。

### 5.4.3 span标签

没有语义的行内容器，用于包裹短语或文字片段，便于通过CSS设置样式或通过JavaScript操作。

```html
<p>价格：<span class="price">¥99.00</span></p>
```

类比：div是大包装袋（块级），span是小包装袋（行内）。

## 5.5 不常用文本标签

以下标签在语义化要求高的场景中酌情使用，日常编码不必过于纠结。

**cite** — 作品标题（书籍、歌曲、电影等）

```html
<p>我最喜欢的电影是 <cite>肖申克的救赎</cite>。</p>
```

**dfn** — 特殊术语或专属名词（首次出现时使用）

```html
<p><dfn>HTML</dfn> 即超文本标记语言。</p>
```

**del与ins** — 删除的文本与插入的文本，常用于展示修改记录

```html
<p>会议时间：<del>周一</del> <ins>周三</ins></p>
```

**sub与sup** — 下标与上标，用于数学公式、化学式、注释等

```html
<p>水的化学式是 H<sub>2</sub>O</p>
<p>E = mc<sup>2</sup></p>
```

**code** — 标记一段行内代码

```html
<p>使用 <code>console.log()</code> 输出调试信息。</p>
```

**samp** — 从正常上下文中提取的设备输出内容

```html
<p>系统返回：<samp>Error: 404 Not Found</samp></p>
```

**kbd** — 键盘输入文本，表示文本是用户通过键盘输入的

```html
<p>按 <kbd>Ctrl</kbd> + <kbd>S</kbd> 保存文件。</p>
```

**abbr** — 缩写，配合title属性说明全称

```html
<p><abbr title="HyperText Markup Language">HTML</abbr> 是网页的基础。</p>
```

**bdo** — 更改文本方向，配合dir属性使用（ltr从左到右，rtl从右到左）

```html
<p><bdo dir="rtl">这段文字会从右到左显示</bdo></p>
```

**var** — 标记变量名，常与code标签配合使用

```html
<p>使用 <code><var>count</var> = 0</code> 初始化计数器。</p>
```

**small** — 附属细则（版权声明、法律免责条款等）

```html
<p><small>版权所有 © 2024</small></p>
```

**b** — 摘要中的关键字、评论中的产品名称（仅表示视觉加粗，无强调语义）

```html
<p><b>iPhone 15</b> 于2023年9月发布。</p>
```

**i** — 人物的思想活动或所说的话；现在多用于呈现字体图标（如FontAwesome）

```html
<p>他心想：<i>这下完了。</i></p>
<!-- 图标使用 -->
<i class="fa fa-user"></i>
```

**u** — 与正常内容有反差的文本（如标注不合适的描述、拼写错误）

```html
<p>这个词的写法是 <u>错误的</u>。</p>
```

**q与blockquote** — 短引用与长引用。q是行内元素，blockquote是块级元素

```html
<p>鲁迅说过：<q>世上本没有路，走的人多了便有了路。</q></p>

<blockquote>
    <p>这是一个段落级别的长引用，会以缩进形式显示。</p>
</blockquote>
```

**address** — 地址信息，是块级元素

```html
<address>
    联系邮箱：example@mail.com<br>
    地址：北京市海淀区
</address>
```

> 备注：blockquote和address是块级元素，其余文本标签都是行内元素。有些语义感较弱的标签（如small、b、u）在HTML5中虽然保留了语义，但实际开发中使用频率较低。

## 5.6 辅助排版标签

**br** — 换行（单标签）

```html
<p>第一行内容<br>第二行内容</p>
```

> 注意：不要用多个br来增加段落间距，应使用p标签或CSS的margin属性控制间距。

**hr** — 主题分隔（单标签），在语义上表示段落之间的主题转换

```html
<h2>第一章</h2>
<p>内容...</p>
<hr>
<h2>第二章</h2>
<p>内容...</p>
```

> 注意：hr的语义是"分隔"，如果只是想画一条装饰性的水平线，应使用CSS的border实现。

**pre** — 预格式化文本，保留文本中的空格、换行和缩进，通常用于展示代码块

```html
<pre>
function hello() {
    console.log("Hello World");
}
</pre>
```

> 注意：pre标签中如果需要显示HTML标签本身（如代码示例），需要配合HTML实体使用（如用 `&lt;` 代替 `<`）。
