## 7.1 列表

### 7.1.1 有序列表（ol + li）

用于有顺序或侧重顺序的内容，浏览器会自动为每项添加序号。

```html
<h2>安装步骤</h2>
<ol>
    <li>下载安装包</li>
    <li>运行安装程序</li>
    <li>完成配置</li>
</ol>
```

ol标签的常用属性：
- type — 序号类型：1（数字，默认）、A（大写字母）、a（小写字母）、I（大写罗马数字）、i（小写罗马数字）
- start — 起始序号（HTML5中此属性已不推荐使用，建议通过CSS控制）
- reversed — 序号倒序（布尔属性）

### 7.1.2 无序列表（ul + li）

用于无顺序或不侧重顺序的内容，浏览器会在每项前显示一个项目符号。

```html
<h2>我喜欢的城市</h2>
<ul>
    <li>北京</li>
    <li>上海</li>
    <li>成都</li>
</ul>
```

### 7.1.3 列表嵌套

列表中的某一项可以再包含一个子列表，用于表达多层级的内容结构。

```html
<h2>前端技术栈</h2>
<ul>
    <li>HTML
        <ul>
            <li>语义化标签</li>
            <li>表单控件</li>
        </ul>
    </li>
    <li>CSS
        <ul>
            <li>布局（Flexbox、Grid）</li>
            <li>动画</li>
        </ul>
    </li>
    <li>JavaScript
        <ul>
            <li>DOM操作</li>
            <li>异步编程</li>
        </ul>
    </li>
</ul>
```

> 注意：嵌套列表时，子列表（ul/ol）必须写在父级的li标签内部，结构要完整。li标签不建议单独使用（不在ul或ol中的li没有语义）。

### 7.1.4 自定义列表（dl + dt + dd）

自定义列表用于表示术语名称及其描述的对应关系。一个dt可以有多个dd。

```html
<h2>学习方法</h2>
<dl>
    <dt>做好笔记</dt>
    <dd>笔记是复习的抓手，帮助巩固记忆</dd>
    <dd>可以是电子版或纸质版</dd>

    <dt>多加练习</dt>
    <dd>只有自己敲出来的代码才是真正掌握的</dd>

    <dt>别怕出错</dt>
    <dd>出错是学习过程中的正常现象</dd>
</dl>
```

标签说明：
- **dl**（description list）：定义一个自定义列表
- **dt**（description term）：定义术语名称
- **dd**（description details）：定义术语描述（一个dt可以有零个或多个dd）

## 7.2 表格

### 7.2.1 表格结构

一个语义完整的表格由以下部分组成：

- **caption** — 表格标题（可选）
- **thead** — 表格头部区域
- **tbody** — 表格主体区域
- **tfoot** — 表格脚注区域（可选）

每个区域由tr（table row，行）组成，行中包含th（table header，表头单元格）或td（table data，数据单元格）。

```html
<table border="1">
    <caption>学生信息表</caption>

    <thead>
        <tr>
            <th>姓名</th>
            <th>性别</th>
            <th>年龄</th>
        </tr>
    </thead>

    <tbody>
        <tr>
            <td>张三</td>
            <td>男</td>
            <td>18</td>
        </tr>
        <tr>
            <td>李四</td>
            <td>女</td>
            <td>20</td>
        </tr>
    </tbody>

    <tfoot>
        <tr>
            <td colspan="3">共计：2人</td>
        </tr>
    </tfoot>
</table>
```

> 注意：即使不写thead、tbody、tfoot，浏览器也会自动将内容包裹在tbody中。但显式写出这些标签有利于语义化和CSS控制。

### 7.2.2 表格属性

**table标签属性**：

width — 设置表格宽度（像素值或百分比）

height — 设置表格最小高度（内容超出时会自动撑高）

border — 设置表格边框宽度（仅控制最外侧边框，单元格边框由CSS控制）

cellspacing — 设置单元格之间的间距（设为0可消除间隙）

**thead/tbody/tfoot/tr标签属性**（三者属性相同）：

align — 水平对齐方式：left（左对齐，默认）、center（居中）、right（右对齐）

valign — 垂直对齐方式：top（顶部）、middle（居中，默认）、bottom（底部）

height — 设置区域高度

**th/td标签属性**：

width/height — 设置单元格宽高。设置某单元格宽度后，整列宽度随之确定；设置某单元格高度后，整行高度随之确定

align/valign — 对齐方式

rowspan — 跨行合并（垂直方向，值为合并的行数）

colspan — 跨列合并（水平方向，值为合并的列数）

### 7.2.3 跨行跨列

使用rowspan和colspan实现单元格的合并，被合并的单元格（被"吃掉"的位置）不需要写对应的td/th标签。

```html
<table border="1">
    <tr>
        <td rowspan="2">跨两行</td>
        <td>正常单元格</td>
        <td>正常单元格</td>
    </tr>
    <tr>
        <!-- 第一列被上面的rowspan吃掉了，所以这里只写2个td -->
        <td>正常单元格</td>
        <td>正常单元格</td>
    </tr>
    <tr>
        <td colspan="3">跨三列</td>
        <!-- 后面两列被colspan吃掉了 -->
    </tr>
</table>
```

> 实际开发中，表格的样式（边框、间距、背景色等）都通过CSS控制，HTML属性仅用于定义结构和基本尺寸。
