## 11.1 HTML全局属性

全局属性是适用于所有HTML元素的通用属性。

**id** — 元素的唯一标识符，页面内不能重复。用途：
- CSS选择器：`#myId { ... }`
- JavaScript操作：`document.getElementById('myId')`
- label关联：`<label for="myId">`
- URL锚点：`<a href="#myId">`

```html
<div id="header">页面头部</div>
```

**class** — 为元素指定一个或多个类名（多个类名用空格分隔），主要用于CSS选择器和JavaScript操作。

```html
<div class="container main-content">内容</div>
```

**style** — 直接设置内联CSS样式（优先级较高，但不利于维护，实际开发中尽量少用）。

```html
<p style="color: red; font-size: 16px;">红色文字</p>
```

**title** — 设置元素的提示信息，鼠标悬停时会显示。常用于超链接和图片，提供额外的说明文字。

```html
<a href="https://www.example.com" title="点击访问示例网站">示例</a>
<img src="photo.jpg" alt="风景照" title="拍摄于2024年春天">
```

**hidden** — 布尔属性，隐藏元素。与CSS的 `display: none` 类似，但hidden是HTML属性层面控制的。

```html
<div hidden>这段内容被隐藏了</div>
```

**draggable** — 设置元素是否可拖动（true/false），配合JavaScript的拖放API使用。

```html
<div draggable="true">可以拖动我</div>
```

**contenteditable** — 设置元素是否可被用户直接编辑（true/false）。

```html
<div contenteditable="true">点击此处可以直接编辑文字</div>
```

> 注意：contenteditable只改变前端显示，不会自动将修改保存到服务器。需要配合JavaScript和后端接口实现保存功能。

**spellcheck** — 设置是否对元素内容进行拼写和语法检查（true/false）。

```html
<textarea spellcheck="true"></textarea>
<input type="text" spellcheck="false">
```

**data-\*** — 自定义数据属性，用于在HTML标签上存储页面私有的自定义数据，供JavaScript读取使用。`*` 部分可以替换为任意名称（必须至少一个字符，且不包含大写字母）。

```html
<div id="user" data-id="1001" data-role="admin" data-last-login="2024-01-01">
    用户信息
</div>

<script>
    // JavaScript中通过dataset对象读取
    var user = document.getElementById('user');
    console.log(user.dataset.id);        // 输出: 1001
    console.log(user.dataset.role);      // 输出: admin
    console.log(user.dataset.lastLogin); // 输出: 2024-01-01 (自动驼峰转换)
</script>
```

命名规则：data-后面的名称会被自动转换为驼峰命名（`data-last-login` → `lastLogin`，`data-user-name` → `userName`）。

**tabindex** — 设置元素的Tab键聚焦顺序。值为数字，正数表示顺序（从小到大），0表示按DOM顺序，-1表示可通过JavaScript聚焦但不参与Tab键遍历。

```html
<input tabindex="1" placeholder="第一个聚焦">
<input tabindex="2" placeholder="第二个聚焦">
<input tabindex="3" placeholder="第三个聚焦">
```

**lang** — 指定元素内容的语言，可覆盖html标签上的lang设置。

```html
<p>这是一段中文。</p>
<p lang="en">This is English.</p>
```

**dir** — 设置元素内容的文本方向（ltr从左到右 / rtl从右到右）。

```html
<p dir="rtl">这段文字从右到左显示</p>
```

## 11.2 iframe内嵌框架

iframe标签用于在当前页面中嵌入另一个完整的HTML页面。

```html
<iframe src="./embedded.html" name="content" width="100%" height="400" frameborder="0"></iframe>
```

属性说明：

**src** — 嵌入页面的URL地址

**name** — 框架的名称，可以与超链接的target属性配合使用，在指定iframe中打开链接

**width/height** — 设置框架的宽高

**frameborder** — 是否显示边框（0不显示 / 1显示，HTML5中已不推荐使用，用CSS的border替代）

**sandbox** — 安全沙箱属性，限制iframe中的页面能力（如禁止脚本、禁止表单提交等），防止嵌入的恶意页面影响父页面

**allow** — 使用Feature Policy指定iframe允许使用的特定功能（如摄像头、麦克风、定位等）

**loading** — 懒加载属性（lazy / eager），设为lazy时iframe会在接近可视区域时才加载

### 11.2.1 iframe的实际应用场景

**场景一：与超链接配合，实现局部页面切换**

```html
<nav>
    <a href="page1.html" target="content">页面一</a>
    <a href="page2.html" target="content">页面二</a>
    <a href="page3.html" target="content">页面三</a>
</nav>
<iframe name="content" width="100%" height="500" frameborder="0"></iframe>
```

点击导航链接时，对应页面会在iframe中打开，实现无需刷新整个页面的内容切换效果。

**场景二：嵌入第三方内容**

```html
<!-- 嵌入百度地图 -->
<iframe src="https://map.baidu.com/..." width="600" height="400"></iframe>

<!-- 嵌入视频（如B站视频） -->
<iframe src="https://player.bilibili.com/..." width="800" height="600" allowfullscreen></iframe>
```

> 注意：iframe会增加页面的资源消耗和加载时间。搜索引擎对iframe内容的抓取能力有限（可能影响SEO）。如果嵌入的页面与父页面不同源（跨域），由于同源策略的限制，父子页面之间的JavaScript通信会受到限制。现代开发中iframe主要用于嵌入无法通过API集成的第三方内容。

## 11.3 HTML实体与特殊字符

### 11.3.1 为什么需要HTML实体

在HTML中，某些字符具有特殊含义（如 `<` 定义标签的开始，`>` 定义标签的结束，`&` 定义实体的开始）。如果需要在页面中显示这些字符本身，必须使用HTML实体来表示，否则浏览器会将它们当作标签或实体语法来解析。

### 11.3.2 实体的语法

HTML实体由三部分组成：`&` + 实体名称（或 `&#` + 实体编号）+ `;`

两种写法：
- 命名实体：`&nbsp;`（便于记忆）
- 数字实体：`&#160;`（兼容性更好）

```html
<!-- 显示小于号 -->
<p>5 &lt; 10</p>

<!-- 显示空格（多个普通空格会被浏览器合并为一个） -->
<p>姓&nbsp;&nbsp;&nbsp;名</p>
```

### 11.3.3 常用字符实体速查

`&nbsp;` / `&#160;` — 空格（不换行空格，浏览器不会合并）

`&lt;` / `&#60;` — 小于号（<）

`&gt;` / `&#62;` — 大于号（>）

`&amp;` / `&#38;` — 和号（&）

`&quot;` / `&#34;` — 双引号（"）

`&apos;` / `&#39;` — 单引号（'）

`&copy;` / `&#169;` — 版权符号（©）

`&reg;` / `&#174;` — 注册商标（®）

`&trade;` / `&#8482;` — 商标（™）

`&yen;` / `&#165;` — 人民币符号（¥）

`&euro;` / `&#8364;` — 欧元符号（€）

`&pound;` / `&#163;` — 英镑符号（£）

`&cent;` / `&#162;` — 分（¢）

`&times;` / `&#215;` — 乘号（×）

`&divide;` / `&#247;` — 除号（÷）

> 完整的HTML实体列表请参考：HTML Standard (whatwg.org)

## 11.4 HTML5兼容性处理

虽然现代浏览器对HTML5的支持已经很好，但在某些特殊场景（如需要兼容旧版IE、国产双核浏览器等）下，仍需要做一些兼容性处理。

### 11.4.1 IE渲染模式设置

```html
<!-- 设置IE总是使用最新的文档模式进行渲染 -->
<meta http-equiv="X-UA-Compatible" content="IE=edge">

<!-- 优先使用Webkit内核渲染（针对360浏览器等双核浏览器） -->
<meta name="renderer" content="webkit">
```

第一个meta让IE使用最高版本的渲染模式（避免进入怪异模式或IE7兼容模式）。

第二个meta让国产双核浏览器优先使用Webkit/Chromium内核而不是IE内核。这些浏览器有"极速模式"（Webkit内核）和"兼容模式"（IE内核），不设置时可能默认使用IE内核导致页面显示异常。

### 11.4.2 HTML5新标签的IE兼容

旧版IE（IE8及以下）不认识HTML5新增的语义化标签（如header、nav、article等），这些标签会被当作普通行内元素处理，无法通过CSS设置宽高等块级样式。

解决方案：使用html5shiv脚本，通过JavaScript在IE中动态创建这些新标签，使其能被正常解析。

```html
<!--[if lt IE 9]>
<script src="https://cdn.bootcdn.net/ajax/libs/html5shiv/3.7.3/html5shiv.min.js"></script>
<![endif]-->
```

原理：html5shiv在IE中执行 `document.createElement('header')` 等代码，让IE"认识"这些标签并应用默认的块级样式。

### 11.4.3 IE条件注释

条件注释是IE浏览器特有的语法，只有IE会解析其中的内容，其他浏览器会将其当作普通HTML注释忽略。用于针对不同IE版本加载不同的资源。

语法格式：`<!--[if 条件]> 内容 <![endif]-->`

条件表达式：

lt — 小于（less than）

lte — 小于等于（less than or equal）

gt — 大于（greater than）

gte — 大于等于（greater than or equal）

! — 逻辑非

示例：

```html
<!--[if IE 8]>仅IE8可见<![endif]-->
<!--[if gt IE 8]>IE8以上版本可见<![endif]-->
<!--[if lt IE 8]>IE8以下版本可见<![endif]-->
<!--[if gte IE 8]>IE8及以上可见<![endif]-->
<!--[if lte IE 8]>IE8及以下可见<![endif]-->
<!--[if !IE]>非IE浏览器可见<![endif]-->
```

> 注意：条件注释在IE10中已被移除，仅适用于IE9及以下版本。IE已于2022年6月正式退役，这些兼容方案目前主要用于维护遗留系统。
