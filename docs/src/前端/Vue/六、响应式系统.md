## 6.1 什么是响应式数据

`setup` 中的数据只要一有修改，模板中使用到数据的位置**自动更新**，这就是响应式。

在 Vue3 中实现响应式数据的语法有两个：**`ref`** 和 **`reactive`**。

## 6.2 ref — 基本类型数据的响应式

**知识点：**

1. 原始类型实现响应式的语法是：使用 `ref()` 函数包裹，例如：`let username = ref("jack")`，`username` 具有响应式。
2. `ref()` 会将原始类型转换为 `RefImpl` 对象。因此 `username` 变量是一个 `RefImpl` 对象。
3. 原始类型数据被放到了 `RefImpl` 对象的 `value` 属性上，**在脚本中需要修改时，语法是：`username.value`**
4. **在模板的插值语法中不需要 `.value`**，因为插值语法底层会自动 `.value`

```vue
<template>
    <div class="person">
        <!-- 模板中不需要 .value，底层会自动处理 -->
        <h2>姓名：{{ name }}</h2>
        <h2>年龄：{{ age }}</h2>
        <button @click="changeName">修改名字</button>
        <button @click="changeAge">修改年龄</button>
        <button @click="showTel">查看联系方式</button>
    </div>
</template>
<script lang="ts" setup>
    import { ref } from 'vue';

    let name = ref('张三');
    let age = ref(20);
    let tel = '13000000000';  // 不需要响应式，不用 ref

    function changeName() {
        // 用 ref 包裹的数据是 RefImpl 对象，操作数据时一定要加 .value
        name.value = 'zhangsan';
    }
    function changeAge() {
        age.value += 1;
    }
    function showTel() {
        alert(`手机号：${tel}`);
    }
</script>
<style scoped>
    .person {
        background-color: skyblue;
        box-shadow: 0 0 15px;
        border-radius: 15px;
        padding: 15px;
    }
    button { margin: 0 5px; }
</style>
```

> **VS Code 小技巧：** Volar 插件可以自动补全 `.value`，在 VS Code 设置 → 扩展 → Vue → 勾选自动补全选项。

> **注意：** 在 TS 中，如果要导入一个类型，需要在类型前面添加 `type` 关键字。

## 6.3 reactive — 对象类型数据的响应式

**知识点：**

1. 如果一个对象要实现响应式效果，需要使用 `reactive()` 函数包裹。
2. 底层实现原理使用了 JS 内置的 `Proxy` 实现。**`Proxy` 是原生 JS 自带的函数。**
3. **被 `reactive()` 包裹的响应式对象默认支持深层次响应式。**
4. `reactive()` **无法处理基本类型数据**的响应式，只能处理对象类型的数据。

```vue
<template>
    <div class="person">
        <h2>姓名：{{ person.name }}</h2>
        <h2>年龄：{{ person.age }}</h2>
        <button @click="changeName">修改名字</button>
        <button @click="changeAge">修改年龄</button>
    </div>
</template>
<script lang="ts" setup>
    import { reactive } from 'vue';

    // reactive 包裹对象，生成 Proxy 代理对象
    let person = reactive({ name: '张三', age: 20 });

    function changeName() {
        person.name = 'zhangsan';  // 直接修改属性，不需要 .value
    }
    function changeAge() {
        person.age += 1;
    }
</script>
<style scoped>
    .person {
        background-color: skyblue;
        box-shadow: 0 0 15px;
        border-radius: 15px;
        padding: 15px;
    }
    button { margin: 0 5px; }
</style>
```

### 6.3.1 深层次响应式测试

被 `reactive` 包裹的对象，深层次属性也是支持响应式的：

```vue
<template>
    <div>
        <div>{{ obj.a.b.c }}</div>
        <button @click="changeValue">修改</button>
    </div>
</template>
<script lang="ts" setup>
    import { reactive } from 'vue';

    let obj = reactive({
        a: {
            b: {
                c: 100
            }
        }
    });

    const changeValue = () => {
        obj.a.b.c = 200;  // 深层次属性修改，仍然触发响应式
    }
</script>
```

## 6.4 reactive 的数组响应式

```vue
<template>
    <div>
        <h2>购物清单</h2>
        <ul>
            <li v-for="(item, index) in shoppingList" :key="item.id">
                {{ index + 1 }}. {{ item.name }} - 数量: {{ item.quantity }}
                <button @click="removeItem(index)">删除</button>
            </li>
        </ul>
        <button @click="changeFirstItemQuantity">将第一个商品的数量修改为 5</button>
    </div>
</template>
<script lang="ts" setup>
    import { reactive } from 'vue'

    const shoppingList = reactive([
        { id: 1, name: '苹果', quantity: 3 },
        { id: 2, name: '牛奶', quantity: 1 },
        { id: 3, name: '面包', quantity: 2 }
    ])

    function removeItem(index: number) {
        shoppingList.splice(index, 1);
    }
    const changeFirstItemQuantity = () => {
        shoppingList[0].quantity = 5;
    }
</script>
```

Vue 能够检测到以下数组方法并触发视图更新：

- `push()`：在数组**末尾**添加一个或多个元素
- `pop()`：删除并返回数组的**最后一个**元素
- `shift()`：删除并返回数组的**第一个**元素
- `unshift()`：在数组**开头**添加一个或多个元素
- `splice()`：在指定位置**删除/替换/添加**元素（功能最强大）
- `sort()`：对数组元素进行**排序**（默认按字符串 Unicode 排序）
- `reverse()`：将数组元素**顺序反转**

这些方法都会**直接修改原数组**，在 Vue 中使用时能触发响应式更新。

## 6.5 ref — 对象类型数据的响应式

`ref` 不仅可以实现基本类型数据的响应式，还可以实现对象类型数据的响应式：

```vue
<template>
    <div>
        <h2>购物清单</h2>
        <ul>
            <li v-for="(item, index) in shoppingList" :key="item.id">
                {{ index + 1 }}. {{ item.name }} - 数量: {{ item.quantity }}
                <button @click="removeItem(index)">删除</button>
            </li>
        </ul>
        <button @click="changeFirstItemQuantity">将第一个商品的数量修改为 5</button>
    </div>
</template>
<script lang="ts" setup>
    import { ref } from 'vue'

    const shoppingList = ref([
        { id: 1, name: '苹果', quantity: 3 },
        { id: 2, name: '牛奶', quantity: 1 },
        { id: 3, name: '面包', quantity: 2 }
    ])

    function removeItem(index: number) {
        shoppingList.value.splice(index, 1);  // 注意：需要加 .value
    }
    const changeFirstItemQuantity = () => {
        shoppingList.value[0].quantity = 5;   // 注意：需要加 .value
    }
</script>
```

## 6.6 reactive 的失效情况

**核心标准：看最外层的引用指向是否发生变化，如果变化就会失去响应式，没有变化则不会失去响应式。**

`reactive` 重新分配一个新对象，会**失去响应式**。

```vue
<template>
    <div>
        <ul>
            <li>姓名：{{ person.name }}</li>
            <li>年龄：{{ person.age }}</li>
        </ul>
        <button @click="changeName">修改名字</button>
        <button @click="changeAge">修改年龄</button>
        <button @click="changePerson">修改用户信息</button>
    </div>
</template>
<script lang="ts" setup>
    import { reactive } from 'vue';

    let person = reactive({ name: 'jack', age: 20 });

    const changeName = () => { person.name = 'jackson'; }
    const changeAge = () => { person.age++; }

    const changePerson = () => {
        // ❌ 响应式失效（最外层引用指向发生变化）
        // person = { name: 'lucy', age: 18 };
        // ❌ 响应式同样失效
        // person = reactive({ name: 'lucy', age: 18 });

        // ✅ 方式一：一个属性一个属性修改
        // person.name = 'lucy';
        // person.age = 18;

        // ✅ 方式二：使用 Object.assign 合并（老对象没变，只是属性值被更新）
        Object.assign(person, { name: 'lucy', age: 18 });
    }
</script>
```

**`ref` 的情况：**

```vue
<script lang="ts" setup>
    import { ref } from 'vue';

    let person = ref({ name: 'jack', age: 20 });

    const changePerson = () => {
        // ✅ 这样可以（修改的是 .value，最外层 ref 对象本身没变）
        person.value = { name: 'lucy', age: 18 };

        // ❌ 这样会失去响应式（重新指向了一个全新的 ref 对象）
        // person = ref({ name: 'lucy', age: 18 });
    }
</script>
```

## 6.7 ref 与 reactive 的使用原则

1. 如果需要一个**基本类型**数据的响应式，**必须使用 `ref`**
2. 如果需要一个响应式对象，且**层次不深**，`ref` 和 `reactive` 都可以。
3. 如果需要一个响应式对象，且**层次较深**，建议使用 `reactive`。

## 6.8 toRefs 与 toRef

### 6.8.1 对象解构语法（背景知识）

```javascript
let user = { username: 'jack', age: 20 };

// 对象解构语法
let { username, age } = user;
// 等同于：
// let username = user.username;
// let age = user.age;

// 数组也支持解构
let [a, b, c] = [1, 2, 3];
```

### 6.8.2 toRefs

`toRefs` 函数将一个响应式对象（reactive 对象）转换为一个新对象，新对象的**每个属性都是 ref**，并且**仍然连接到原始响应式对象**。主要用途：**解构时保持响应性**。

**问题演示（解构后失去响应式）：**

```vue
<script lang="ts" setup>
    import { reactive } from 'vue';

    let person = reactive({ name: 'jack', age: 18 });

    // 对象解构后，name 和 age 不再是响应式的
    let { name, age } = person;

    const changeName = () => { name += '*'; }   // ❌ 不会更新视图
    const changeAge = () => { age++; }           // ❌ 不会更新视图
</script>
```

**使用 toRefs 保持响应性：**

```vue
<template>
    <ul>
        <li>姓名：{{ name }}</li>
        <li>年龄：{{ age }}</li>
    </ul>
    <button @click="changeName">修改名字</button>
    <button @click="changeAge">修改年龄</button>
</template>
<script lang="ts" setup>
    import { reactive, toRefs } from 'vue';

    let person = reactive({ name: 'jack', age: 18 });

    // toRefs 解构后，name 和 age 都是 ref，并连接到原始对象
    let { name, age } = toRefs(person);

    const changeName = () => { name.value += '*'; }  // ✅ 响应式生效
    const changeAge = () => { age.value++; }          // ✅ 响应式生效
</script>
```

### 6.8.3 toRef

`toRef` 函数为响应式对象（reactive 对象）的**某个特定属性**创建一个 ref 引用，这个 ref 会与原属性保持同步的响应式连接。主要用途：**将响应式对象的单个属性作为 ref 传递时保持响应性**。

```vue
<template>
    <ul>
        <li>年龄：{{ age }}</li>
    </ul>
    <button @click="changeAge">修改年龄</button>
</template>
<script lang="ts" setup>
    import { reactive, toRef } from 'vue';

    let person = reactive({ name: 'jack', age: 18 });

    // 为 reactive 响应式对象的 age 属性创建一个 ref 引用
    let age = toRef(person, 'age');

    const changeAge = () => {
        age.value++;  // 修改会同步到 person.age
    }
</script>
```

## 6.9 浅层响应式 API

### 6.9.1 shallowRef

**官方描述：** 通过 `shallowRef()` 可以绕开深度响应，浅层式 API 创建的状态只在其顶层是响应式的，对所有深层的对象不会做任何处理，避免了对每一个内部属性做响应式所带来的性能成本，这让属性的访问更快，可提升性能。

**特点：** 只跟踪**引用值**的变化，不关心值内部的属性变化。

```vue
<template>
    <div>
        <h4>计数器：{{ count }}</h4>
        <h4>姓名：{{ person.name }}</h4>
        <h4>年龄：{{ person.age }}</h4>
        <button @click="add">计数器加 1</button>
        <button @click="changeName">修改姓名</button>
        <button @click="changeAge">修改年龄</button>
        <button @click="changePerson">修改整个人</button>
    </div>
</template>
<script lang="ts" setup>
    import { shallowRef } from 'vue';

    let count = shallowRef(0);
    let person = shallowRef({ name: 'jackson', age: 20 });

    function add() { count.value++; }          // ✅ shallowRef 对基本类型有响应式
    function changeName() { person.value.name = 'lucy'; } // ❌ shallowRef 内部属性无响应式
    function changeAge() { person.value.age++; }          // ❌ shallowRef 内部属性无响应式
    function changePerson() {
        person.value = { name: 'king', age: 30 };         // ✅ shallowRef 整体替换有响应式
    }
</script>
```

### 6.9.2 shallowReactive

**特点：** 对象的**顶层属性**是响应式的，但**嵌套的对象属性**不是。

```vue
<script lang="ts" setup>
    import { shallowReactive } from 'vue';

    let person = shallowReactive({
        name: 'jackson',   // ✅ 顶层属性，有响应式
        age: 20,           // ✅ 顶层属性，有响应式
        address: {
            city: 'BeiJing'  // ❌ 嵌套属性，无响应式
        }
    });

    function changeName() { person.name = 'lucy'; }          // ✅ 响应式
    function changeAge() { person.age++; }                    // ✅ 响应式
    function changeCity() { person.address.city = 'TianJin'; } // ❌ 不触发视图更新
</script>
```

## 6.10 只读 API

创建一个响应式对象的**只读副本**，保护响应式对象不被他人修改。

- `readonly`：**深只读**（较常用）
- `shallowReadonly`：**浅只读**（对象顶层属性只读，深层次属性不是只读的）

```vue
<script lang="ts" setup>
    import { reactive, readonly, shallowReadonly } from 'vue';

    let person = reactive({
        name: 'jackson',
        age: 20,
        address: { city: 'BeiJing' }
    });

    // 创建深只读副本（参数必须是响应式对象）
    let person1 = readonly(person);
    // 创建浅只读副本
    let person2 = shallowReadonly(person);

    function changeName1() { person1.name = 'lucy'; }          // ❌ 报错：深只读
    function changeAge1() { person1.age++; }                   // ❌ 报错：深只读
    function changeCity1() { person1.address.city = 'TianJin'; } // ❌ 报错：深只读

    function changeName2() { person2.name = 'lucy'; }          // ❌ 报错：浅只读（顶层属性）
    function changeAge2() { person2.age++; }                   // ❌ 报错：浅只读（顶层属性）
    function changeCity2() { person2.address.city = 'TianJin'; } // ✅ 可以：浅只读不覆盖深层
</script>
```

## 6.11 toRaw 与 markRaw

### 6.11.1 toRaw

`toRaw` 可以获取响应式对象对应的**原始普通对象**。

**使用场景：** 在需要将响应式对象传递给非 Vue 的库或外部系统时，使用 `toRaw` 确保它们收到的是普通对象。

```vue
<script lang="ts" setup>
    import { reactive, toRaw } from 'vue';

    let person = reactive({ name: 'jackson', age: 30 });

    console.log('响应式对象：', person);     // Proxy { name: 'jackson', age: 30 }

    let rawPerson = toRaw(person);
    console.log('普通对象：', rawPerson);   // { name: 'jackson', age: 30 }
</script>
```

### 6.11.2 markRaw

被 `markRaw` 标记的对象**永远不会变成响应式对象**。

**使用场景：** 使用第三方库时，为了防止把第三方库的某个对象变为响应式对象，可以使用 `markRaw` 进行标记。

```vue
<script lang="ts" setup>
    import { markRaw, reactive } from 'vue';

    // 被 markRaw 标记的对象永远都不会变成响应式对象
    let person = markRaw({ name: 'jackson', age: 30 });
    // 即使手动使用 reactive 包裹，person 仍然是一个非响应式对象
    let person2 = reactive(person);

    function changeName() { person2.name = 'lucy'; }  // ❌ 不触发视图更新
    function changeAge() { person2.age++; }            // ❌ 不触发视图更新
</script>
```

## 6.12 customRef（自定义 ref）

`customRef` 可以完成自定义 `ref` 的效果，实现带有延迟性的响应式对象。

### 6.12.1 实现延迟响应式效果

```vue
<template>
    <div>
        <h4>消息: {{ msg }}</h4>
        <input type="text" v-model="msg">
    </div>
</template>
<script lang="ts" setup>
    import { customRef } from 'vue';

    let timer: number;
    let initValue = 'hello';

    // 自定义 ref：修改 2 秒后才触发响应式更新
    let msg = customRef((track, trigger) => {
        return {
            // 响应式对象 msg 被读取时自动调用
            get() {
                track(); // 告诉 Vue：这个值有人在盯着，变化了要通知我
                return initValue;
            },
            // 响应式对象 msg 被修改时自动调用
            set(value) {
                clearTimeout(timer);
                timer = setTimeout(() => {
                    initValue = value;
                    trigger(); // 告诉 Vue：这个值变了，快通知所有盯着它的人！
                }, 2000);
            }
        }
    });
</script>
```

### 6.12.2 封装为可复用 Hook

新建 `hooks/useMsgRef.ts`：

```typescript
import { customRef } from "vue";

export default function(initValue: string, delay: number) {
    let timer: number;
    let msg = customRef((track, trigger) => {
        return {
            get() {
                track();
                return initValue;
            },
            set(value) {
                clearTimeout(timer);
                timer = setTimeout(() => {
                    trigger();
                    initValue = value;
                }, delay);
            }
        }
    });
    return { msg };
}
```

在组件中使用：

```vue
<template>
    <div>
        <h4>立即响应式消息: {{ msg2 }}</h4>
        <input type="text" v-model="msg2">
        <hr>
        <h4>延迟性响应式消息（3 秒后更新）: {{ msg }}</h4>
        <input type="text" v-model="msg">
    </div>
</template>
<script lang="ts" setup>
    import { ref } from 'vue';
    import useMsgRef from '../hooks/useMsgRef';

    let msg2 = ref('hello');
    let { msg } = useMsgRef('hello', 3000);  // 注意：返回值要解构
</script>
```
