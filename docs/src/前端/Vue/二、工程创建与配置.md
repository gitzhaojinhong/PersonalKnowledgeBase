## 2.1 基于 vue-cli 创建（不推荐）

vue-cli 底层基于 webpack 实现，**已不推荐使用**，了解即可。

### 2.1.1 检查 Node.js 和 npm 版本

确保 Node.js 版本 ≥ 14.0.0，npm 版本 ≥ 5.6。

```bash
node -v  # 检查 Node.js 版本
npm -v   # 检查 npm 版本
```

### 2.1.2 全局安装 @vue/cli

```bash
npm install -g @vue/cli
vue --version  # 确认 @vue/cli 版本（需要 ≥ 4.5.0）
```

### 2.1.3 创建并启动项目

```bash
vue create my-vue3-project
# 选择 Manually select features → 选择 Vue 3.x
cd my-vue3-project
npm run serve   # 默认运行在 http://localhost:8080
# Ctrl+C 终止服务
```

## 2.2 基于 Vite 创建【官方推荐】

### 2.2.1 了解 Vite

- **Vite** 是新一代前端构建工具（尤雨溪编写），官网：[https://vitejs.cn](https://vitejs.cn)
- **webpack 和 vite 都是构建工具**，将开发者编写的源代码经过转换、打包、优化，生成生产环境可部署的静态文件。
- 轻量快速的热重载（**HMR**），能实现极速的服务启动。
- 对 **TypeScript、JSX、CSS** 等支持开箱即用。
- 真正的**按需编译**，不再等待整个应用编译完成。

**webpack vs vite 的核心区别：**

```plain
webpack：从入口 entry 进来，分析所有的路由，分析每个路由下的所有模块，
         然后绑定，最后服务器准备好了。较慢。
         → 像提前做好所有菜的餐厅，顾客来了直接上菜，但准备时间长

vite：   先把服务器启动起来准备好，根据用户的请求来加载对应的路由和模块，
         不访问的路由和模块可以先不加载。更快，更智能。
         → 像现点现做的火锅店，锅底（基础环境）先上，食材（模块）随要随煮
```

### 2.2.2 环境要求

确保 Node.js 版本 ≥ 16.0.0，npm 版本 ≥ 7.0.0。

```bash
node -v
npm -v
```

### 2.2.3 配置淘宝镜像（国内用户）

```bash
npm config set registry https://registry.npmmirror.com
```

### 2.2.4 创建 Vue3 项目

> **注意：`create-vite` 是 vite 提供的脚手架，它本身并不是 vite。**

```bash
npm create vite@6.5.0
# 输入项目名：hello_vue3
# 选择框架：Vue
# 选择语言：TypeScript
```

### 2.2.5 安装依赖并启动

```bash
cd hello_vue3
npm install    # 安装 package.json 中声明的所有依赖，等同于 maven 下载 jar 包
npm run dev    # 启动开发服务器，访问 http://localhost:5173
npm run build  # 构建生产版本，输出到 dist/ 目录
# Ctrl+C 终止服务器
```

> **注意：** `npm install` 之后，`node_modules` 目录中的 `vite` 才是 Vite 的真身，通过它来构建前端项目。

## 2.3 项目结构说明

```plain
my-vue-app/
├── .vscode/           # VS Code 编辑器配置（如 extensions.json 描述需要安装的插件）
├── dist/              # 生产环境构建输出目录
├── node_modules/      # 项目依赖的 npm 包
├── public/            # 静态资源（不被 Vite 构建处理，直接复制到 dist 根路径）
└── src/               # 源代码目录（前端工程师的工作结果）
    ├── App.vue        # Vue 根组件
    ├── assets/        # 图片、字体等静态资源（会被 webpack 解析打包，与 public 不同）
    ├── components/    # 可复用的 Vue 组件
    ├── main.ts        # 应用入口文件（初始化 Vue 实例）
    ├── style.css      # 全局样式文件
    └── vite-env.d.ts  # Vite 环境变量类型声明（让 ts 认识 .jpg .txt 等文件）
├── .gitignore         # 指定 Git 忽略的文件和目录
├── index.html         # 主 HTML 文件（应用入口，body 中引入了 main.ts）
├── package.json       # 项目配置和依赖管理（类似 maven 中的 pom.xml）
├── package-lock.json  # 锁定依赖版本，确保安装一致性
├── README.md          # 项目说明文档
├── tsconfig.app.json  # 应用源码的 TypeScript 编译配置（继承 tsconfig.json）
├── tsconfig.json      # 主 TypeScript 配置文件
├── tsconfig.node.json # Node.js 相关脚本（构建工具、配置文件）的 TS 编译配置
└── vite.config.ts     # Vite 构建工具配置（安装插件、配置代理等整个工程的配置文件）
```

## 2.4 VS Code 插件配置

### 2.4.1 安装 Vue.volar 插件

使用 VS Code 打开 Vue3 项目后，右下角会自动有提示，安装 `Vue.volar` 扩展。

安装 `Vue.volar` 后，获得以下所有能力：

- **语法高亮**（`.vue` 文件中的模板、脚本、样式）
- **类型检查**（TypeScript 支持）
- **智能提示**（组件 props、事件、插槽）
- **模板表达式类型推导**（如 `{{ user.name }}` 会检查 `user` 的类型）
- **快速跳转定义**（`Ctrl+Click` 跳转到组件或变量定义）

### 2.4.2 安装图标主题插件（可选）

安装 `Material Icon Theme` 插件，让工程中的目录和文件图标显示更美观。

### 2.4.3 安装 Vue.js devtools

Vue.js Devtools 是 Vue.js 官方提供的浏览器开发者工具插件，用于调试和优化 Vue.js 应用程序，可直观查看、分析和修改 Vue 组件的状态、事件、路由和性能。

**安装步骤（普通方式）：**

1. 搜索极简插件官网：[https://chrome.zzzmh.cn/](https://chrome.zzzmh.cn/)，下载 `Vue.js devtools` 压缩包。
2. 打开 Chrome → 右上角三个点 → 扩展程序 → 管理扩展程序 → 打开"开发者模式"
3. 解压下载的压缩包，将 `xxx_chrome.zzzmh.cn.crx` 文件拖入"管理扩展程序"页面，点击添加扩展程序。
4. 重新打开浏览器，F12 打开控制面板，看到 `Vue` 选项卡即表示安装成功。

**Win11 Chrome 不支持拖动安装时：**

1. 用 **7zip** 解压 `.crx` 文件（亲测只能用 7zip）。
2. 点击"加载未打包的扩展程序"，选择 7zip 解压的目录完成安装。

## 2.5 VS Code 配置 Vue3 代码模板

在 VS Code 中输入 `vue3` 回车，自动生成组件模板。

**配置步骤：**

1. `Ctrl + Shift + P`
2. 输入 `Preferences: Configure User Snippets`
3. 选择 `vue.json`
4. 添加如下配置：

```json
{
  "Vue 3 Component": {
    "prefix": "vue3",
    "body": [
      "<template>",
      "</template>",
      "",
      "<script lang='ts' setup>",
      "</script>",
      "",
      "<style scoped>",
      "</style>"
    ],
    "description": "Generate a Vue 3 single-file component with <script setup>"
  }
}
```

在任意 `.vue` 文件中输入 `vue3` 回车，即可自动生成代码片段。
