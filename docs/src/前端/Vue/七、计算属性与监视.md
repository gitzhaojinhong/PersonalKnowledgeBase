# 七、计算属性与监视

## 7.1 计算属性（computed）

### 7.1.1 为什么需要计算属性

Vue 官方建议**模板中只应该包含简单的表达式**，复杂的逻辑应该放在计算属性或方法中。模板中的复杂表达式会：

- 降低模板的可读性
- 使逻辑难以复用
- 增加维护难度

### 7.1.2 只读计算属性

```vue
<template>
    姓：<input type="text" v-model="firstName"><br>
    名：<input type="text" v-model="lastName"><br>
    全名：{{ fullName }}<br>
    <!-- 多次使用时，直接从缓存中取，不再执行计算。计算属性支持缓存。 -->
    全名：{{ fullName }}
</template>
<script lang="ts" setup>
    import { computed, ref } from 'vue';

    let firstName = ref('张');
    let lastName = ref('三');

    // 计算属性（只读）
    // 只要关联的属性值发生变化，计算属性的函数就会被调用一次
    // 如果关联的所有属性值都没有发生变化，计算属性的值会从缓存中获取（支持缓存）
    let fullName = computed(() => {
        return firstName.value.slice(0, 1).toUpperCase() + firstName.value.slice(1) + '-' + lastName.value;
    });
    // 注意：计算属性返回的也是一个 Ref 响应式对象
</script>
```

### 7.1.3 计算属性 vs 方法

方法也可以实现相同的功能，但**没有缓存**，每次使用都会重新调用：

```vue
<template>
    全名：{{ fullName() }}<br>  <!-- 每次渲染都会执行一次 fullName() -->
    全名：{{ fullName() }}<br>  <!-- 又执行一次，共两次 -->
</template>
<script lang="ts" setup>
    import { ref } from 'vue';
    let firstName = ref('张');
    let lastName = ref('三');

    function fullName() {
        console.log('fullName() 方法执行了');  // 每次使用就打印一次
        return firstName.value.slice(0, 1).toUpperCase() + firstName.value.slice(1) + '-' + lastName.value;
    }
</script>
```

**结论：** 计算属性比方法效率更高，因为支持缓存机制。

### 7.1.4 可读可写计算属性

只读的计算属性是不可修改的，如果需要可读可写的计算属性：

**语法格式：**

```vue
<script lang="ts" setup>
  import { type WritableComputedRef, computed } from 'vue';

  let computedProperty: WritableComputedRef<string> = computed({
    get() {
      // 当读取计算属性的值时，该方法会被调用
      return 值;
    },
    set(val) {
      // 当修改计算属性的值时，该方法会被调用
    }
  });
</script>
```

**完整示例：**

```vue
<template>
    姓：<input type="text" v-model="firstName"><br>
    名：<input type="text" v-model="lastName"><br>
    <!-- 这里会自动调用计算属性的 get() 方法 -->
    全名：{{ fullName }}<br>
    <button @click="changeFullName">将全名修改为 Jack-son</button>
</template>
<script lang="ts" setup>
    import { computed, ref } from 'vue';

    let firstName = ref('张');
    let lastName = ref('三');

    let fullName = computed({
        get() {
            return firstName.value.slice(0, 1).toUpperCase() + firstName.value.slice(1) + '-' + lastName.value;
        },
        set(val) {
            // 修改计算属性时，更新原始数据
            const [first, last] = val.split('-');
            firstName.value = first;
            lastName.value = last;
        }
    });

    const changeFullName = () => {
        // 这里会自动调用计算属性的 set() 方法
        fullName.value = 'jack-son';
    };
</script>
```

**总结：**

- **只读计算属性：** `computed(回调函数)`
- **可读可写计算属性：** `computed({对象中写 get 和 set 方法})`

## 7.2 watch 监视

**作用：监视数据的变化，当数据变化时执行回调函数处理业务。**

**监视 vs 计算属性：**
- 监视目的：监视某个数据的变化，当数据变化时，执行回调函数处理业务。
- 计算属性目的：最终返回一个计算之后的值，依赖缓存机制。

**Vue3 中 watch 只能监视以下四种数据：**

1. **一个函数，返回一个值（getter 函数）**
2. **一个 ref**
3. **一个响应式对象**
4. **...或是由以上类型的值组成的数组**

### 7.2.1 监视 ref 定义的基本类型数据

**默认情况下：**

1. 监视 `ref(0)` 时，监视的是 `0` 这个数字的变化，只要数字变化就能监视到。
2. 监视 `ref({name:"jack", age: 20})`，监视的是对象有没有变成新对象。如果要监视对象中的属性，需要开启深度监视。

**语法格式：**

```vue
<script lang="ts" setup>
    import { watch } from 'vue';

    // 只要被监视的对象中的数据发生变化，立即执行回调函数
    // 注意：第一个参数不能写 count.value，只能写 count（ref 对象本身）
    watch(被监视的对象, (新值, 旧值) => {
        // 监视后的逻辑
    }, { 配置项 });
</script>
```

**完整示例：**

```vue
<template>
    <div>
        <h2>count = {{ count }}</h2>
        <button @click="addOne">计数器加 1</button>
    </div>
</template>
<script lang="ts" setup>
    import { ref, watch } from 'vue';

    let count = ref(0);
    const addOne = () => { count.value++; }

    // 监视 count，只要 count 一发生变化，立即执行回调函数
    watch(count, (newVal, oldVal) => {
        console.log('计数器加 1 了', newVal, oldVal);
    });
</script>
```

**停止监视：**

```vue
<script lang="ts" setup>
    import { ref, watch } from 'vue';

    let count = ref(0);
    const addOne = () => { count.value++; }

    // watch() 函数的返回值是一个函数，调用它可以停止监视
    const stopWatch = watch(count, (newVal, oldVal) => {
        console.log('计数器加 1 了', newVal, oldVal);
        if (count.value >= 10) {
            stopWatch();  // 计数器达到 10 时停止监视
        }
    });
</script>
```

### 7.2.2 监视 ref 定义的对象类型数据

**监视 ref 定义的对象类型数据时，直接写数据名，监视的是对象的【地址值】，如果想监视对象内部的数据，需要手动开启深度监视。**

**未开启深度监视（监视地址值）：**

```vue
<script lang="ts" setup>
    import { ref, watch } from 'vue';

    let person = ref({ name: 'jack', age: 30 });

    function changeName() { person.value.name += '*'; }
    function changeAge() { person.value.age++; }
    function changePerson() {
        person.value = { name: 'lucy', age: 30 };  // 修改了地址值，会触发监视
    }

    // 未开启深度监视：只有修改整个对象（地址值变化）才会触发
    watch(person, (newVal, oldVal) => {
        console.log('person 被修改了', newVal, oldVal);
    });
</script>
```

**开启深度监视（`deep: true`）：**

```vue
<script lang="ts" setup>
    import { ref, watch } from 'vue';

    let person = ref({ name: 'jack', age: 30 });

    function changeName() { person.value.name += '*'; }
    function changeAge() { person.value.age++; }
    function changePerson() {
        person.value = { name: 'lucy', age: 30 };
    }

    // 开启深度监视：对象内部属性变化和整体替换都会触发
    watch(person, (newVal, oldVal) => {
        console.log('person 被修改了', newVal, oldVal);
    }, { deep: true });
</script>
```

**关于 newVal 和 oldVal：**

1. 实际开发中，`newVal` 用的较多，`oldVal` 几乎不用，因此回调函数一般写一个参数即可。
2. 如果是**修改整个对象**：`newVal` 和 `oldVal` 两个变量保存的对象地址不同，`newVal` 引用指向新对象，`oldVal` 引用指向旧对象。
3. 如果是**修改对象内部属性**：`newVal` 和 `oldVal` 两个引用指向同一个对象，打印结果相同，都显示更新后的效果。

**开启立即监视（`immediate: true`）：**

```vue
<script lang="ts" setup>
    import { ref, watch } from 'vue';

    let person = ref({ name: 'jack', age: 30 });

    // immediate: true — 页面加载完毕后立即执行一次
    watch(person, (newVal, oldVal) => {
        console.log('person 被修改了', newVal, oldVal);
    }, { deep: true, immediate: true });
</script>
```

### 7.2.3 监视 reactive 定义的对象数据

**重点：监视 reactive 定义的对象数据时，默认开启深度监视，并且深度监视无法关闭。**

```vue
<template>
    <h2>姓名：{{ person.name }}</h2>
    <h2>年龄：{{ person.age }}</h2>
    <button @click="changeName">修改姓名</button>
    <button @click="changeAge">修改年龄</button>
    <button @click="changePerson">修改整个人</button>
</template>
<script lang="ts" setup>
    import { reactive, watch } from 'vue';

    let person = reactive({ name: 'jack', age: 18 });

    function changeName() { person.name += '*'; }
    function changeAge() { person.age++; }
    function changePerson() {
        // reactive 对象不能整体替换，使用 Object.assign
        Object.assign(person, { name: 'lucy', age: 22 });
    }

    // reactive 默认开启深度监视，{ deep: false } 设置也无法关闭
    watch(person, (newVal, oldVal) => {
        console.log('person 被修改了', newVal, oldVal);
    }, { deep: false });  // 无效，deep 默认开启且无法关闭
</script>
```

### 7.2.4 监视响应式对象中的某个属性

**结论：**

1. 监视响应式对象中的某个属性时，如果属性是**基本类型**的，需要将该属性**包装为 getter 函数**。
2. 监视响应式对象中的某个属性时，如果属性是**对象类型**的，推荐将其**包装为 getter 函数**，并同时**开启深度监视**。

```vue
<script lang="ts" setup>
    import { reactive, watch } from 'vue';

    let person = reactive({
        name: '张三',
        address: {
            city: '北京',
            street: '朝阳街道'
        }
    });

    // 监视基本类型属性：必须提供 getter 函数
    watch(() => person.name, (newVal, oldVal) => {
        console.log('person.name 被修改了', newVal, oldVal);
    });

    // 监视对象类型属性（不推荐）：直接写对象可以监视到内部属性变化，但对象被整体替换时不会触发
    // watch(person.address, (newVal, oldVal) => { ... });

    // 监视对象类型属性（推荐）：getter 函数 + deep: true
    // 对象内部属性变化 和 对象被整体替换 都能监视到
    watch(() => person.address, (newVal, oldVal) => {
        console.log('person.address 被修改了', newVal, oldVal);
    }, { deep: true });
</script>
```

### 7.2.5 监视多个数据（数组形式）

```vue
<script lang="ts" setup>
    import { reactive, watch } from 'vue';

    let person = reactive({
        name: '张三',
        address: { city: '北京', street: '朝阳街道' }
    });

    // 同时监视 person.name 和 person.address.street
    watch([() => person.name, () => person.address.street], (newVal, oldVal) => {
        console.log('数据被修改了', newVal, oldVal);
        // newVal 是一个数组，对应被监视的各项新值
    });
</script>
```

## 7.3 watchEffect 模糊监视

`watch` 可以看做**精确监视**，`watchEffect` 可以做到**模糊监视**。

**`watch` 和 `watchEffect` 对比：**

1. 都能监听响应式数据的变化，但监听方式不同。
2. `watch`：要**明确指出**监视的数据。
3. `watchEffect`：**不需要明确指出**监视的数据（函数中用到哪些属性，它就会**自动监视**这些属性），且默认立即执行一次。

**场景：监视学生语文和数学成绩，根据成绩输出评级。**

使用 `watch` 实现：

```vue
<script lang="ts" setup>
    import { ref, watch } from 'vue';

    let chineseScores = ref(0);
    let mathScores = ref(0);

    function changeChineseScores() { if (chineseScores.value < 100) chineseScores.value += 10; }
    function changeMathScores() { if (mathScores.value < 100) mathScores.value += 10; }

    // watch：需要明确列出监视的数据
    watch([chineseScores, mathScores], (val) => {
        let [chScore, maScore] = val;
        if (chScore >= 90 && maScore >= 90) console.log('成绩：优秀');
        else if (chScore >= 80 && maScore >= 80) console.log('成绩：良好');
        else if (chScore >= 60 && maScore >= 60) console.log('成绩：及格');
        else console.log('成绩：不及格');
    }, { immediate: true });
</script>
```

使用 `watchEffect` 实现（更简洁）：

```vue
<script lang="ts" setup>
    import { ref, watchEffect } from 'vue';

    let chineseScores = ref(0);
    let mathScores = ref(0);

    function changeChineseScores() { if (chineseScores.value < 100) chineseScores.value += 10; }
    function changeMathScores() { if (mathScores.value < 100) mathScores.value += 10; }

    // watchEffect：自动追踪函数内用到的所有响应式数据，无需明确指定
    // 且默认立即执行一次
    watchEffect(() => {
        if (chineseScores.value >= 90 && mathScores.value >= 90) console.log('成绩：优秀');
        else if (chineseScores.value >= 80 && mathScores.value >= 80) console.log('成绩：良好');
        else if (chineseScores.value >= 60 && mathScores.value >= 60) console.log('成绩：及格');
        else console.log('成绩：不及格');
    });
</script>
```
