## 6.1 图片标签（img）

img是单标签，用于在页面中展示图片。

### 6.1.1 基本属性

```html
<img src="./images/logo.png" alt="网站Logo" width="200" height="100">
```

各属性说明：

**src**（必填）— 图片的路径，可以是相对路径或绝对路径。图片无法加载时整个img区域会显示为空白或替代图标。

**alt** — 图片的替代文字描述，图片无法显示时会出现在图片位置。作用：
1. 搜索引擎通过alt属性理解图片内容（最主要的SEO作用）
2. 图片加载失败时，浏览器会显示alt文字
3. 屏幕阅读器会朗读alt内容，帮助视障用户理解

> 开发建议：每张img都应设置alt属性，使用简洁准确的自然语言描述图片内容。

**width/height** — 设置图片显示的宽度和高度（单位为像素）。只设置其中一个时，另一个会按比例自动计算。实际开发中推荐通过CSS控制尺寸，而不是使用HTML属性。

### 6.1.2 图片路径

**相对路径** — 以当前文件所在位置为参考点

./ — 当前目录（可省略）

```html
<img src="./photo.jpg">
<img src="photo.jpg">
```

下一级目录/ — 向下一级进入子文件夹

```html
<img src="./images/photo.jpg">
```

../ — 上一级目录

```html
<img src="../images/photo.jpg">
```

> 注意：相对路径依赖于当前文件的位置，如果文件被移动，路径需要同步修改。

**绝对路径** — 从根位置开始的完整路径

本地绝对路径：`E:/a/b/photo.jpg`（很少使用，更换设备后路径失效）

网络绝对路径：`http://www.example.com/images/photo.png`

> 注意：使用网络图片时，如果对方服务器开启了防盗链，图片可能无法正常显示。

### 6.1.3 常见图片格式

**jpg/jpeg** — 有损压缩格式
- 优点：支持色彩丰富，压缩率高，文件体积较小
- 缺点：不支持透明背景，不支持动画
- 场景：产品宣传图、照片展示等色彩丰富的场景

**png** — 无损压缩格式
- 优点：支持色彩丰富，支持透明背景，画质更高
- 缺点：文件体积比jpg大，不支持动画
- 场景：需要透明背景的图标、logo、高质量图片

**gif** — 仅支持256种颜色
- 优点：支持简单透明背景，支持动画
- 缺点：色彩有限，不适合照片
- 场景：简单的动态图标、表情包

**bmp** — 无压缩格式
- 优点：完全保留图片细节
- 缺点：文件体积极大，不支持透明，不支持动画
- 场景：对画质要求极高的特殊场景

**webp** — 谷歌推出的新一代格式
- 优点：体积小、质量高、支持透明和动画，综合性能优于jpg和png
- 缺点：部分老旧浏览器不支持
- 场景：现代Web项目中的通用图片格式

**base64** — 将图片编码为文本字符串
- 原理：将图片的二进制数据通过Base64算法编码成一串文本
- 优点：不受文件路径限制，减少HTTP请求数
- 缺点：文本体积比原始图片大约33%，不适合大图
- 场景：小图标、内嵌图片、邮件中的图片

```html
<!-- base64图片直接作为src的值 -->
<img src="data:image/png;base64,iVBORw0KGgoAAAANSUhEUg..." alt="内嵌图标">
```

> 开发建议：实际项目中优先使用webp格式，配合jpg/png作为降级方案。图片资源建议通过构建工具进行压缩优化。

## 6.2 超链接（a标签）

### 6.2.1 基本概念

a标签（anchor，锚）用于创建超链接，实现页面间的跳转或页面内的定位。

超链接可以实现：
1. 跳转到其他页面
2. 跳转到指定文件（或触发下载）
3. 跳转到页面内的锚点位置
4. 唤起设备上的应用（拨号、邮件、短信等）

### 6.2.2 标签属性

```html
<a href="https://www.example.com" target="_blank" title="点击查看">链接文字</a>
```

各属性说明：

**href**（核心属性）— 指定跳转目标地址，可以是URL、文件路径、锚点ID或特殊协议。

**target** — 控制链接在何处打开：
- `_self`：在当前窗口打开（默认值）
- `_blank`：在新窗口/新标签页打开

**title** — 鼠标悬停时显示的提示文字，可用于补充说明链接目标。

**download** — 指定下载文件的名称（配合href使用），强制触发浏览器下载而不是打开文件。

### 6.2.3 页面跳转

```html
<!-- 跳转到外部网站 -->
<a href="https://www.baidu.com" target="_blank">去百度</a>

<!-- 跳转到本地页面 -->
<a href="./about.html">关于我们</a>

<!-- 跳转到上级目录的页面 -->
<a href="../index.html">返回首页</a>
```

> 注意：a标签是行内元素，但它可以包裹除自身以外的几乎所有元素（包括块级元素），这在语义和结构上是合法的。

### 6.2.4 文件跳转与下载

浏览器会根据文件类型决定行为：能直接打开的文件（如jpg、mp4、pdf）会在线展示，无法打开的文件（如zip、exe）会触发下载。

```html
<!-- 浏览器能直接打开的文件 -->
<a href="./photo.jpg">查看照片</a>
<a href="./document.pdf">查看PDF</a>

<!-- 浏览器无法打开的文件，自动触发下载 -->
<a href="./data.zip">下载资源</a>

<!-- 使用download属性强制触发下载，并指定下载文件名 -->
<a href="./movie.mp4" download="宣传视频.mp4">下载视频</a>
```

download属性的值是下载后保存的文件名，如果不指定则使用原文件名。

### 6.2.5 锚点跳转

锚点是网页中的一个标记点，点击链接后页面会自动平滑滚动到该位置。

**第一步：设置锚点**

```html
<!-- 方式一：使用a标签的name属性 -->
<a name="section1"></a>

<!-- 方式二：使用任意标签的id属性（推荐） -->
<h2 id="section1">章节一</h2>
<p>段落内容...</p>
```

推荐使用id属性方式，因为id可以设置在任何元素上，语义更清晰。

**第二步：链接到锚点**

```html
<!-- 跳转到当前页面的锚点 -->
<a href="#section1">跳转到章节一</a>

<!-- 跳转到页面顶部 -->
<a href="#">回到顶部</a>

<!-- 跳转到其他页面的锚点 -->
<a href="about.html#section1">去关于页面的章节一</a>

<!-- 刷新当前页面（href为空字符串） -->
<a href="">刷新页面</a>

<!-- 执行JavaScript代码 -->
<a href="javascript:alert('Hello');">点我弹窗</a>
```

> 注意：name和id属性区分大小写，且id值不建议以数字开头。`href="#"` 会跳转到页面顶部并会在URL中添加一个 `#`，如果不希望改变URL，可以使用JavaScript的 `scrollTo()` 方法。

### 6.2.6 唤起应用

通过特定的URI协议，可以唤起设备上的原生应用：

```html
<!-- 唤起拨号 -->
<a href="tel:10010">拨打电话</a>

<!-- 唤起邮件客户端 -->
<a href="mailto:example@mail.com">发送邮件</a>

<!-- 唤起短信 -->
<a href="sms:10086">发送短信</a>
```

实际开发中，这类链接主要用在移动端页面。桌面浏览器中tel和sms协议可能无法生效，mailto协议则取决于用户是否配置了邮件客户端。
