## 1.1 计算机基础

计算机是用于高速计算的电子设备，具备数值计算、逻辑计算和存储记忆功能。

计算机由**硬件 + 软件**组成：
- **硬件**：看得见摸得着的物理部件（CPU、内存、硬盘、显示器等）
- **软件**：指挥硬件工作的指令集合

软件分类：
- **系统软件**：Windows、Linux、macOS、Android、HarmonyOS 等，负责管理硬件资源和提供基础运行环境
- **应用软件**：微信、QQ、Chrome、PhotoShop 等，运行在系统软件之上，完成特定用户任务

## 1.2 软件架构：C/S与B/S

软件按架构分为两大类：

**C/S架构（Client/Server，客户端/服务器）**
- 需要安装专用的客户端程序
- 功能更新需要重新安装或更新客户端
- 通常不跨平台（Windows客户端无法直接在macOS运行）
- 开发更具针对性，能充分利用本地硬件性能

**B/S架构（Browser/Server，浏览器/服务器）**
- 无需安装客户端，通过浏览器直接访问
- 服务端更新后，用户刷新即可使用最新版本
- 天然跨平台，只要有浏览器就能访问
- 开发更具通用性，但受浏览器能力限制

名词解释：
- **C** => Client（客户端）
- **B** => Browser（浏览器）
- **S** => Server（服务器）——为软件提供数据和计算服务的设备

前端工程师主要负责编写 B/S 架构中的网页（呈现界面、实现交互）。

> 备注：大前端时代，可以用前端技术栈（如 Electron、React Native）做出 C/S 架构的应用，甚至用 Node.js 搭建服务器。

## 1.3 浏览器

浏览器是网页运行的平台，是用户访问互联网资源的主要工具。

### 1.3.1 常见浏览器

主流浏览器：Chrome、Safari、Edge、Firefox、Opera 等。

### 1.3.2 浏览器内核

浏览器内核（渲染引擎）负责将HTML、CSS、JavaScript解析并渲染为用户可见的页面。不同内核对网页标准的支持程度不同，开发时需要关注兼容性。

Trident — IE浏览器（已停止维护，现代开发无需兼容）

Gecko — Firefox浏览器

Webkit — Safari、Chrome（早期版本）

Blink — Chrome（2013年起）、Opera、Edge（2020年起）——目前市场占有率最高的内核

## 1.4 网页基本概念

- **网址（URL）**：浏览器地址栏中输入的地址，用于定位互联网资源
- **网页**：浏览器呈现的每一个页面，本质是一个HTML文件
- **网站**：由多个相互关联的网页组成的集合，通常共享同一域名
- **网页标准**：网页制作需遵循统一的技术规范，确保在不同浏览器中表现一致

网页标准的构成（三大核心技术）：
- **HTML**：定义网页的结构和内容（"骨架"）
- **CSS**：控制网页的样式和布局（"皮囊"）
- **JavaScript**：实现网页的交互和动态效果（"肌肉"）
