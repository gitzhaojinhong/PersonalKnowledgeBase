## 8.1 表单基本概念

表单（form）是网页中用于收集用户输入数据并将其提交到服务器的交互区域。登录注册、搜索、留言、问卷调查等场景都离不开表单。

## 8.2 表单基本结构

```html
<form action="/submit" method="post" target="_blank">
    <input type="text" name="username">
    <input type="password" name="password">
    <button type="submit">登录</button>
</form>
```

form标签的属性：

**action** — 表单提交的目标地址（URL），数据会发送到这个地址。如果不设置，默认提交到当前页面。

**method** — 表单提交方式：
- get：将数据拼接到URL后面（`?key=value&key2=value2`），数据量有限（约2KB），地址栏可见，适用于搜索等无敏感数据的场景
- post：将数据放在请求体中发送，数据量无限制，地址栏不可见，适用于登录、注册等涉及敏感数据的场景

**target** — 提交后页面的打开方式（`_self` 当前窗口，`_blank` 新窗口）。

**novalidate** — 禁用HTML5内置的表单验证，适用于使用自定义JavaScript验证的场景。

```html
<form action="/submit" method="post" novalidate>
    <!-- 表单内的required、pattern等验证属性将不生效 -->
</form>
```

## 8.3 文本输入框

```html
<input type="text" name="username" value="默认值" maxlength="20" placeholder="请输入用户名">
```

属性说明：

name — 数据的名称，提交表单时作为键名出现在请求参数中（如 `?username=张三`）。name是表单数据提交的核心标识，没有name的输入框数据不会被提交。

value — 输入框的默认值。用户未输入时，提交的就是这个默认值。对于文本框来说，value只是初始文字，用户可以修改。

maxlength — 限制输入的最大字符数，超出部分无法输入。

placeholder（HTML5新增）— 占位提示文字，显示在输入框中，用户开始输入后自动消失。placeholder不是默认值，不会被提交。

## 8.4 密码输入框

```html
<input type="password" name="pwd" maxlength="20" placeholder="请输入密码">
```

密码框的属性与文本框基本一致，区别在于输入的内容会以圆点或星号形式显示，防止旁人看到。

> 注意：密码框的value属性虽然可以设置默认值，但在实际开发中不建议使用（密码不应硬编码在HTML中）。

## 8.5 单选框

```html
<input type="radio" name="gender" value="male" id="male">
<label for="male">男</label>
<input type="radio" name="gender" value="female" id="female">
<label for="female">女</label>
```

属性说明：

name — 数据名称。**多个单选框的name值必须相同**才能实现"同组互斥"的单选效果（同一时间只能选中一个）。

value — 选中后提交的数据值。单选框必须设置value（没有value时提交的是"on"）。

checked — 布尔属性，设置该单选框默认选中。

## 8.6 复选框

```html
<input type="checkbox" name="hobby" value="reading" id="reading">
<label for="reading">阅读</label>
<input type="checkbox" name="hobby" value="music" id="music">
<label for="music">音乐</label>
<input type="checkbox" name="hobby" value="sports" id="sports" checked>
<label for="sports">运动</label>
```

属性说明：

name — 数据名称。同一组复选框的name值通常相同（提交时多个值以同名参数的形式发送）。

value — 选中后提交的数据值。复选框必须设置value。

checked — 布尔属性，设置该复选框默认选中。多个复选框可以同时被选中。

## 8.7 隐藏域

```html
<input type="hidden" name="userId" value="10086">
```

隐藏域在页面上不可见，但提交表单时会将数据发送到服务器。常用于携带用户不需要看到也不需要修改的数据，如用户ID、会话token、表单标识等。

## 8.8 文件选择框

```html
<input type="file" name="avatar" accept="image/*">
```

属性说明：

name — 数据名称

accept — 限制可选择的文件类型：
- `accept="image/*"` — 所有图片类型
- `accept=".jpg,.png"` — 指定扩展名
- `accept="video/mp4,video/avi"` — 指定MIME类型

multiple — 布尔属性，允许选择多个文件

> 注意：使用文件上传时，form标签的method必须为post，且必须设置 `enctype="multipart/form-data"`，否则文件无法正确上传。

```html
<form action="/upload" method="post" enctype="multipart/form-data">
    <input type="file" name="avatar">
    <button type="submit">上传</button>
</form>
```

enctype属性的三种值：
- application/x-www-form-urlencoded（默认）— 普通文本数据
- multipart/form-data — 包含文件上传时必须使用
- text/plain — 纯文本（很少使用）

## 8.9 按钮

### 8.9.1 提交按钮

点击后触发表单提交。

```html
<!-- 使用input标签，value指定按钮文字 -->
<input type="submit" value="提交">

<!-- 使用button标签，标签内容即为按钮文字 -->
<button type="submit">提交</button>
```

### 8.9.2 重置按钮

点击后将表单内所有控件恢复为初始值（注意：恢复的是HTML中定义的默认值，不是清空）。

```html
<input type="reset" value="重置">
<button type="reset">重置</button>
```

### 8.9.3 普通按钮

不会触发表单提交，通常配合JavaScript使用。

```html
<input type="button" value="点击">
<button type="button">点击</button>
```

> 注意：button标签的type属性默认值是submit，如果不指定type="button"，点击button会导致表单提交，这是一个常见的错误来源。

### 8.9.4 按钮使用注意

1. 不要给button标签设置name属性，否则提交表单时会把按钮也当作数据提交
2. input标签创建的按钮通过value属性指定按钮文字
3. button标签创建的按钮通过标签内容指定文字，且可以包含HTML内容（如图标）

## 8.10 文本域

```html
<textarea name="message" rows="5" cols="30" placeholder="请输入留言">默认内容</textarea>
```

属性说明：

rows — 默认显示的行数，影响文本域的高度

cols — 默认显示的列数（按字符宽度计算），影响文本域的宽度

name — 数据名称

disabled — 布尔属性，禁用文本域

placeholder — 占位提示文字（HTML5新增）

> 注意：textarea是双标签，默认值写在标签之间，而不是value属性。textarea不支持type属性。实际开发中文本域的尺寸通常通过CSS的width和height控制，rows和cols仅作为初始尺寸。

## 8.11 下拉框

```html
<select name="province">
    <option value="">请选择省份</option>
    <option value="bj">北京</option>
    <option value="sh">上海</option>
    <option value="gd" selected>广东</option>
    <option value="sc" disabled>四川（暂不可选）</option>
</select>
```

select标签属性：

name — 数据名称

disabled — 禁用整个下拉框

multiple — 布尔属性，允许选择多个选项（按住Ctrl/Cmd多选）

size — 下拉框可见的选项数量（默认为1，即单行下拉；大于1时显示为列表框）

option标签属性：

value — 该选项提交到服务器的实际数据。如果不设置value，提交的是标签之间的文字内容。建议始终设置value。

selected — 布尔属性，设置该选项默认选中

disabled — 布尔属性，禁用该选项（灰色显示，用户无法选择）

```html
<!-- 多选下拉框 -->
<select name="cities" multiple size="4">
    <option value="bj">北京</option>
    <option value="sh">上海</option>
    <option value="gz">广州</option>
    <option value="sz">深圳</option>
</select>
```

> 注意：多选下拉框的name通常加 `[]` 后缀（如 `name="cities[]"`），以便服务端识别为数组。

## 8.12 禁用表单控件

给表单控件设置disabled属性即可禁用。input、textarea、button、select、option都支持disabled。

禁用后的效果：
- 控件显示为灰色
- 用户无法交互（无法输入、选择、点击）
- 该控件的数据不会随表单提交

```html
<input type="text" name="username" disabled>
<select disabled>
    <option>不可用选项</option>
</select>
```

> 实际开发中，disabled常用于条件禁用（如未勾选协议前提交按钮不可用），通常通过JavaScript动态控制。

## 8.13 label标签

label标签用于关联表单控件，扩展控件的点击区域。关联后，点击label的文字等同于点击对应的表单控件（如点击"用户名"文字会让输入框获取焦点），提升用户体验和表单的易用性。

**方式一：label的for属性值等于控件的id值（推荐）**

```html
<label for="username">用户名：</label>
<input id="username" type="text" name="username">
```

**方式二：将表单控件包裹在label标签内部**

```html
<label>
    密码：
    <input type="password" name="pwd">
</label>
```

> 开发建议：推荐使用方式一（for+id），因为它将label和控件解耦，HTML结构更灵活，CSS样式也更容易控制。

## 8.14 fieldset与legend标签

fieldset用于将表单控件分组，legend是分组的标题。使用后浏览器会在控件组周围显示一个边框。

```html
<form>
    <fieldset>
        <legend>基本信息</legend>
        <label for="account">账户：</label>
        <input id="account" type="text" name="account" maxlength="20">
        <br><br>
        <label for="pwd">密码：</label>
        <input id="pwd" type="password" name="pwd" maxlength="20">
    </fieldset>

    <fieldset>
        <legend>个人信息</legend>
        性别：
        <input type="radio" name="gender" value="male" id="male">
        <label for="male">男</label>
        <input type="radio" name="gender" value="female" id="female">
        <label for="female">女</label>
    </fieldset>
</form>
```

fieldset标签的disabled属性可以一次性禁用组内所有控件：

```html
<fieldset disabled>
    <legend>不可编辑区域</legend>
    <input type="text" name="field1">
    <input type="text" name="field2">
</fieldset>
```

## 8.15 HTML5新增表单属性

HTML5为表单控件新增了一系列实用属性，简化了开发中常见的功能实现。

**placeholder** — 输入提示文字（非默认值）

```html
<input type="text" name="phone" placeholder="请输入手机号">
```

**required** — 必填验证，未填写时表单无法提交，浏览器会弹出提示

```html
<input type="text" name="email" required>
```

**autofocus** — 页面加载后自动获取焦点，一个页面中建议只设置一个

```html
<input type="text" name="search" autofocus>
```

**autocomplete** — 控制浏览器自动补全功能（on开启/off关闭）

```html
<input type="text" name="newPassword" autocomplete="off">
```

适用范围：文字输入类控件。密码框和多行输入框默认不支持自动补全。

**pattern** — 通过正则表达式验证输入格式，提交时进行校验

```html
<input type="text" name="phone" pattern="^1[3-9]\d{9}$" placeholder="请输入手机号">
<input type="text" name="idcard" pattern="^\d{17}[\dXx]$" placeholder="请输入身份证号">
```

适用范围：文本输入类控件（多行输入不支持）。空输入框不触发验证，常与required配合使用。

**multiple** — 允许输入多个值

```html
<input type="file" name="files" multiple>
<input type="email" name="emails" multiple>
```

**form** — 将控件关联到指定的form（即使控件不在该form内部）

```html
<form id="myForm" action="/submit">
    <button type="submit">提交</button>
</form>
<input type="text" name="username" form="myForm">
```

**formaction** — 覆盖form的action属性，指定单独的提交地址

```html
<button type="submit" formaction="/submit2">提交到备用地址</button>
```

**formmethod** — 覆盖form的method属性

```html
<button type="submit" formmethod="get">以GET方式提交</button>
```

## 8.16 HTML5新增input类型

HTML5扩展了input标签的type属性，提供了更多语义化的输入控件，同时部分类型自带格式验证功能。

| type值 | 功能 | 提交时验证 | 移动端特性 | 附加属性 |
|--------|------|-----------|-----------|----------|
| email | 邮箱输入框 | 是（空值不验证） | 弹出邮箱键盘（含@） | multiple支持多个邮箱 |
| url | URL输入框 | 是（空值不验证） | 弹出URL键盘（含.com等） | — |
| tel | 电话输入框 | 否 | 弹出数字拨号键盘 | pattern配合正则验证格式 |
| number | 数字输入框 | 是（空值不验证） | 弹出数字键盘 | min、max、step |
| search | 搜索输入框 | 否 | 同普通键盘 | 右侧带清除按钮 |
| range | 范围滑动条 | 否 | — | min、max、step、value |
| color | 颜色选择器 | 否 | — | value为十六进制颜色值 |
| date | 日期选择器 | 否 | 弹出日期滚轮 | min、max限制范围 |
| month | 月份选择器 | 否 | 弹出月份滚轮 | — |
| week | 周选择器 | 否 | 弹出周滚轮 | — |
| time | 时间选择器 | 否 | 弹出时间滚轮 | min、max、step |
| datetime-local | 日期时间选择器 | 否 | 弹出日期时间滚轮 | min、max、step |

使用示例：

```html
<!-- 邮箱 -->
<input type="email" name="email" placeholder="请输入邮箱" required>

<!-- 数字，配合min/max/step限制范围和步长 -->
<input type="number" name="age" min="0" max="150" step="1">

<!-- 范围滑动条 -->
<input type="range" name="volume" min="0" max="100" value="50">

<!-- 颜色选择器 -->
<input type="color" name="theme" value="#ff0000">

<!-- 日期选择器 -->
<input type="date" name="birthday" min="1900-01-01" max="2024-12-31">

<!-- 时间选择器，step为30分钟 -->
<input type="time" name="meeting" step="1800">

<!-- 多个邮箱 -->
<input type="email" name="cc" multiple placeholder="多个邮箱用逗号分隔">
```

number类型的min、max、step属性：
- min — 允许的最小值
- max — 允许的最大值
- step — 步长（每次增减的值），如 `step="0.01"` 允许输入小数

> 注意：HTML5新增类型的显示效果因浏览器而异。旧版浏览器不认识这些type值时，会自动降级为text类型，不影响基本功能。

## 8.17 datalist标签

datalist为输入框提供预定义的建议选项列表，用户可以从下拉建议中选择，也可以自行输入。

```html
<input type="text" name="city" list="cityList" placeholder="请输入或选择城市">
<datalist id="cityList">
    <option value="北京">
    <option value="上海">
    <option value="广州">
    <option value="深圳">
</datalist>
```

工作原理：input通过list属性关联到datalist（通过id），datalist中的option即为建议选项。用户输入时浏览器会自动过滤匹配项。

> 注意：datalist只是"建议"，用户完全可以输入不在列表中的值。如果需要严格限制为列表中的值，应使用select下拉框。

## 8.18 output标签

output标签用于展示计算结果或操作输出，是语义化的展示元素。

```html
<form oninput="result.value = Number(a.value) + Number(b.value)">
    <input type="number" name="a" value="0"> +
    <input type="number" name="b" value="0"> =
    <output name="result" for="a b">0</output>
</form>
```

属性说明：
- for — 关联的控件ID列表（用空格分隔），表示结果来源于哪些控件
- name — 数据名称
