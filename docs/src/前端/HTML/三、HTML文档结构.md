## 3.1 文档声明（DOCTYPE）

文档声明告诉浏览器当前网页使用的HTML版本，浏览器据此决定采用哪种渲染模式来解析页面。如果省略文档声明，浏览器会进入"怪异模式"，可能导致页面样式异常。

**HTML5的写法**（推荐）：

```html
<!DOCTYPE html>
```

大小写均可（`<!DOCTYPE HTML>`、`<!doctype html>`），效果相同。

> 注意：文档声明必须在网页的第一行，且位于html标签的外侧，它本身不是HTML标签。

## 3.2 HTML根标签与语言设置

html标签是整个网页的根元素，所有其他标签都写在它内部。

```html
<html lang="zh-CN">
```

lang属性用于指定网页的主要语言，作用：
1. 让浏览器在需要时提供翻译提示
2. 帮助搜索引擎理解页面语言，优化搜索结果的准确性
3. 辅助屏幕阅读器使用正确的语音朗读

**lang属性编写规则**：

推荐写法（语言代码-国家/地区代码）：

zh-CN — 中文-中国大陆（简体中文）

zh-TW — 中文-中国台湾（繁体中文）

zh — 中文（不指定地区）

en-US — 英语-美国

en-GB — 英语-英国

ja-JP — 日语-日本

ko-KR — 韩语-韩国

已不推荐的旧写法：zh-Hans（简体）、zh-Hant（繁体）

参考资料：W3School《语言代码参考手册》、《国家/地区代码参考手册》；W3C《Language tags in HTML》。

## 3.3 head标签

head标签包含网页的元数据配置，这些内容不会直接呈现在页面上，但对浏览器的行为、搜索引擎的理解、社交媒体的展示等有重要影响。

```html
<head>
    <!-- 字符编码声明 -->
    <meta charset="UTF-8">
    <!-- 网页标题 -->
    <title>网页标题</title>
</head>
```

title标签指定浏览器标签页上显示的标题，每个页面应有且仅有一个title。title的内容对SEO影响很大，建议简洁明了地描述页面主题。

## 3.4 字符编码

### 3.4.1 什么是字符编码

计算机只能识别二进制数据，字符编码就是将字符（文字、符号）与二进制数字之间建立对应关系的规则。不同的编码规则决定了能表示哪些字符。

### 3.4.2 常见字符集

**ASCII**：最早的标准编码，收录128个字符（英文大小写字母、数字、常用符号），占用1个字节。无法表示中文。

**ISO 8859-1**：在ASCII基础上扩充了西欧语言字符，共256个字符，占用1个字节。仍无法表示中文。

**GB2312**：中国国家标准，收录6763个常用汉字和682个符号，占用2个字节。覆盖了绝大多数日常用字。

**GBK**：GB2312的扩展版本，支持的汉字和符号达到20000+，同时支持繁体中文，向下兼容GB2312。

**UTF-8**：Unicode的可变长度编码实现，包含世界上所有语言的所有文字与符号。使用1~4个字节表示一个字符（英文1字节、中文3字节）。是目前最通用的编码方式。

### 3.4.3 使用原则

- 编码和解码必须使用同一种字符集，否则会出现乱码。例如用GBK编码保存的文件，用UTF-8打开就会乱码
- 开发中统一使用UTF-8编码是最稳妥的选择

### 3.4.4 在HTML中声明编码

通过meta标签的charset属性指定文档编码，建议写在head标签的第一行，让浏览器尽早知道编码方式：

```html
<meta charset="UTF-8">
```

## 3.5 body标签

body标签中的内容会呈现在网页页面上，是用户可见的部分。页面的所有结构、内容、交互元素都写在body内部。

```html
<body>
    <!-- 用户可见的内容写在body中 -->
</body>
```

## 3.6 HTML标准结构

将以上各部分组合起来，就是一份标准的HTML文档：

```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
    <meta charset="UTF-8">
    <title>网页标题</title>
</head>
<body>

</body>
</html>
```

**快捷生成**：在VSCode中输入 `!`，随后回车即可快速生成HTML5标准结构。

补充说明：
- Emmet生成的结构中会包含viewport等额外meta标签，暂时用不到可以按需保留或删除
- 可以通过配置VSCode的Emmet插件来定制生成结构的属性和内容
- 在项目根目录放置 `favicon.ico` 图标文件，浏览器会自动将其识别为网站图标
- 也可以通过link标签手动指定图标路径：`<link rel="icon" href="./favicon.ico">`

**【检查】和【查看网页源代码】的区别**：
- 【查看网页源代码】（Ctrl+U）：看到的是程序员编写的原始HTML代码
- 【检查】（F12或右键=>检查）：看到的是浏览器渲染后的DOM结构，JavaScript动态生成的内容也能看到

## 3.7 meta标签详解

meta标签用于配置网页的元信息，位于head标签内，是单标签。它是HTML中配置项最多的标签之一，对SEO、浏览器行为、移动端适配等都有直接影响。

### 3.7.1 字符编码

```html
<meta charset="UTF-8">
```

指定文档字符编码为UTF-8，确保中文等非英文字符正常显示。每个HTML页面都应包含此配置，且建议写在head标签的第一行。

### 3.7.2 IE兼容性配置

```html
<meta http-equiv="X-UA-Compatible" content="IE=edge">
```

设置IE浏览器总是使用最新的文档模式进行渲染。IE浏览器有多个渲染模式（IE5怪异模式、IE7标准模式、IE8标准模式等），不设置此项时IE可能使用旧版渲染引擎，导致CSS3和HTML5特性无法正常工作。现代开发中IE已退役，此配置主要用于兼容遗留系统。

### 3.7.3 移动端视口配置

```html
<meta name="viewport" content="width=device-width, initial-scale=1.0">
```

设置视口宽度等于设备屏幕宽度，初始缩放比例为1.0。这是移动端适配的基础配置，缺少此配置时，移动端浏览器会将页面缩小显示（通常默认980px宽度），导致页面文字和布局偏小。

各配置项含义：
- width=device-width：视口宽度等于设备宽度
- initial-scale=1.0：初始缩放比例为1.0（不缩放）
- maximum-scale=1.0：允许用户最大放大到1倍（限制用户缩放）
- user-scalable=no：禁止用户缩放（不推荐，影响无障碍访问）

### 3.7.4 网页关键词（SEO）

```html
<meta name="keywords" content="HTML, CSS, JavaScript, 前端开发, Web教程">
```

告诉搜索引擎该网页的关键字，辅助搜索排名。关键词数量建议控制在8-12个，用英文逗号分隔，内容应与页面主题高度相关。

> 注意：现代搜索引擎对keywords的权重已大幅降低，Google已公开表示不再使用此标签作为排名因素，但百度等国内搜索引擎仍会参考。

### 3.7.5 网页描述信息（SEO）

```html
<meta name="description" content="一份全面的HTML速查笔记，涵盖文档结构、标签用法、表单、多媒体等核心知识点。">
```

搜索引擎结果页面中展示的摘要文字，通常显示80字以内。描述的内容直接影响用户是否点击搜索结果，应简洁准确地概括页面内容。

### 3.7.6 搜索引擎爬虫控制

```html
<meta name="robots" content="noindex, nofollow">
```

控制搜索引擎爬虫对该页面的行为，适用于测试页面、私密页面等不需要被搜索引擎收录的场景。

常用值：

index — 允许搜索爬虫索引此页面（默认行为）

noindex — 要求搜索爬虫不索引此页面

follow — 允许搜索爬虫跟随页面上的链接继续抓取（默认行为）

nofollow — 要求搜索爬虫不跟随页面上的链接

all — 等价于 index, follow

none — 等价于 noindex, nofollow

noarchive — 要求搜索引擎不缓存页面快照

nocache — noarchive的替代名称

可以组合使用，如 `content="noindex, follow"` 表示不索引当前页但继续跟踪页内链接。

### 3.7.7 网页作者

```html
<meta name="author" content="作者名">
```

标注网页的作者信息，不会显示在页面上，可通过查看源代码看到。

### 3.7.8 网页生成工具

```html
<meta name="generator" content="Visual Studio Code">
```

标注用于生成该网页的工具或软件名称。

### 3.7.9 网页版权信息

```html
<meta name="copyright" content="2023-2027 版权所有">
```

标注网页的版权声明信息。

### 3.7.10 页面自动刷新与跳转

```html
<!-- 5秒后自动刷新当前页面 -->
<meta http-equiv="refresh" content="5">

<!-- 10秒后自动跳转到百度 -->
<meta http-equiv="refresh" content="10;url=http://www.baidu.com">
```

content属性的数字表示秒数，url指定跳转目标地址。省略url则仅刷新当前页面。

> 注意：此方式会使用户失去对页面的控制，且对SEO不友好。现代开发中推荐使用JavaScript的 `location.href` 实现跳转。
