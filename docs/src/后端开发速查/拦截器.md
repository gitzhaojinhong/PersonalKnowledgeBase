# 一、SpringBoot中使用拦截器

---

## 1.1 概述

拦截器（Interceptor）用于在请求到达 Controller 之前或之后进行拦截，可对请求和响应进行特定处理。

常见应用场景：
- 登录验证：判断用户是否已登录，未登录则跳转登录页
- 权限校验：根据用户权限控制访问，拒绝未授权请求
- 请求日志：记录请求地址、参数、时间等，用于排查问题和性能分析
- 统一响应处理：添加响应头信息、调整响应内容格式等

---

## 1.2 拦截器与过滤器的区别

| 对比维度 | 过滤器（Filter） | 拦截器（Interceptor） |
|:---|:---|:---|
| 作用层面 | Servlet 层面，作用于请求进入容器之后 | Controller 层面，作用于请求到达控制器前后 |
| 配置方式 | FilterRegistrationBean 或 @WebFilter | 实现 HandlerInterceptor + WebMvcConfigurer |
| 能否操作请求/响应对象 | 能 | 能 |
| 能否获取 Controller 方法信息 | 不能 | 能（通过 Object handler 参数，可强转为 HandlerMethod） |
| 是否基于 Spring | 否，基于 Servlet 规范 | 是，基于 SpringMVC |
| 执行时机 | 早于拦截器 | 晚于过滤器 |
| 对不同 DispatcherType 的拦截 | 默认只拦截 REQUEST 类型（ERROR/FORWARD/INCLUDE/ASYNC 需显式配置） | 所有请求类型均拦截（除非通过 excludePathPatterns 排除） |

---

## 1.3 执行流程

单个请求的完整执行链路：

```
请求
  ↓
Filter（Servlet 层面，可多个）
  ↓
Interceptor#preHandle（顺序执行，全部返回 true 才继续）
  ↓
Controller（处理器方法）
  ↓
Interceptor#postHandle（逆序执行）
  ↓
视图渲染
  ↓
Interceptor#afterCompletion（逆序执行，无论是否异常）
  ↓
Filter#doFilter 后续逻辑
  ↓
响应
```

过滤器按注册顺序执行；拦截器的执行顺序由配置顺序决定。

> 注意：如果某个拦截器 `preHandle` 返回 false，则该拦截器后续的所有 `postHandle` 都不执行，直接进入 `afterCompletion` 回调。

---

## 1.4 拦截器定义

拦截器需实现 `org.springframework.web.servlet.HandlerInterceptor`，三个方法均可选择性重写：

| 方法 | 执行时机 | 返回值 |
|:---|:---|:---|
| `preHandle` | 处理器方法调用之前 | boolean：true 放行，false 拦截，请求不再继续 |
| `postHandle` | 处理器方法调用之后、视图渲染之前 | 无返回值 |
| `afterCompletion` | 视图渲染完成后（无论是否异常） | 无返回值 |

> Spring 5.3 起，`HandlerInterceptorAdapter` 已废弃，推荐直接实现 `HandlerInterceptor` 接口。

### 1.4.1 获取 Controller 方法信息

`preHandle` 和 `postHandle` 方法中的 `Object handler` 参数，在处理 Controller 请求时实际类型为 `HandlerMethod`，可从中获取方法信息：

```java
@Component
public class MethodInfoInterceptor implements HandlerInterceptor {

    @Override
    public boolean preHandle(HttpServletRequest request, HttpServletResponse response,
                              Object handler) throws Exception {
        if (handler instanceof HandlerMethod) {
            HandlerMethod hm = (HandlerMethod) handler;
            String className = hm.getBeanType().getSimpleName();   // Controller 类名
            String methodName = hm.getMethod().getName();           // 方法名
            System.out.println("调用的方法: " + className + "." + methodName);
        }
        return true;
    }
}
```

### 1.4.2 完整示例

```java
public class MyInterceptor implements HandlerInterceptor {

    @Override
    public boolean preHandle(HttpServletRequest request, HttpServletResponse response,
                              Object handler) throws Exception {
        System.out.println("preHandle: " + request.getRequestURI());
        return true;  // true 放行，false 拦截
    }

    @Override
    public void postHandle(HttpServletRequest request, HttpServletResponse response,
                           Object handler, ModelAndView modelAndView) throws Exception {
        System.out.println("postHandle: " + modelAndView);
    }

    @Override
    public void afterCompletion(HttpServletRequest request, HttpServletResponse response,
                                Object handler, Exception ex) throws Exception {
        System.out.println("afterCompletion: 异常 = " + ex);
    }
}
```

## 1.5 拦截器注册与配置

### 1.5.1 Spring MVC（XML 配置）

```xml
<!-- 方式一：直接配置全类名 -->
<mvc:interceptors>
    <bean class="com.example.interceptors.Interceptor1"/>
</mvc:interceptors>

<!-- 方式二：引用 Bean（需 @Component 标注且在包扫描范围内） -->
<mvc:interceptors>
    <ref bean="interceptor1"/>
</mvc:interceptors>
```

基本配置默认拦截所有请求。

### 1.5.2 Spring Boot（Java 配置，推荐）

实现 `WebMvcConfigurer` 接口，重写 `addInterceptors` 方法：

```java
@Configuration
public class WebMvcConfig implements WebMvcConfigurer {

    @Override
    public void addInterceptors(InterceptorRegistry registry) {
        registry.addInterceptor(new MyInterceptor())
                .addPathPatterns("/**")              // 拦截路径
                .excludePathPatterns("/error", "/login");  // 排除路径
    }
}
```

`addInterceptor` 返回 `InterceptorRegistration`，链式调用 `addPathPatterns` 和 `excludePathPatterns` 配置拦截范围。

---

## 1.6 路径匹配规则

| 匹配符 | 含义 | 示例 |
|:---|:---|:---|
| `/**` | 拦截所有路径，包括子路径 | `/user/**` 匹配 `/user/list`、`/user/admin/detail` |
| `/*` | 拦截一级路径 | `/user/*` 匹配 `/user/list`，不匹配 `/user/admin/detail` |
| `/` | 精确匹配单个路径 | `/user` 仅匹配 `/user` |

排除路径示例：

```java
registry.addInterceptor(new LoginInterceptor())
        .addPathPatterns("/**")
        .excludePathPatterns("/login", "/register", "/static/**", "/error");
```

> 拦截器默认会拦截 `/error` 端点，如果不想拦截错误处理，需显式排除。

## 1.7 源码分析

### 1.7.1 DispatcherServlet 中的调用

```java
public class DispatcherServlet extends FrameworkServlet {
    protected void doDispatch(HttpServletRequest request, HttpServletResponse response) {
        // 1. 顺序执行所有拦截器的 preHandle
        if (!mappedHandler.applyPreHandle(processedRequest, response)) {
            return;
        }
        // 2. 执行 Controller 处理器方法
        mv = ha.handle(processedRequest, response, mappedHandler.getHandler());
        // 3. 逆序执行所有拦截器的 postHandle
        mappedHandler.applyPostHandle(processedRequest, response, mv);
        // 4. 渲染视图
        processDispatchResult(processedRequest, response, mappedHandler, mv, dispatchException);
    }

    private void processDispatchResult(...) {
        render(mv, request, response);
        // 5. 逆序执行所有拦截器的 afterCompletion
        mappedHandler.triggerAfterCompletion(request, response, null);
    }
}
```

### 1.7.2 HandlerExecutionChain 中的执行逻辑

```java
public class HandlerExecutionChain {
    // preHandle：顺序执行，有返回值
    boolean applyPreHandle(HttpServletRequest request, HttpServletResponse response) {
        for (int i = 0; i < interceptorList.size(); i++) {
            if (!interceptorList.get(i).preHandle(request, response, handler)) {
                triggerAfterCompletion(request, response, null);  // 逆序回调
                return false;
            }
            interceptorIndex = i;
        }
        return true;
    }

    // postHandle：逆序执行
    void applyPostHandle(HttpServletRequest request, HttpServletResponse response,
                         @Nullable ModelAndView mv) {
        for (int i = interceptorList.size() - 1; i >= 0; i--) {
            interceptorList.get(i).postHandle(request, response, handler, mv);
        }
    }

    // afterCompletion：逆序执行
    void triggerAfterCompletion(HttpServletRequest request, HttpServletResponse response,
                                @Nullable Exception ex) {
        for (int i = this.interceptorIndex; i >= 0; i--) {
            interceptorList.get(i).afterCompletion(request, response, handler, ex);
        }
    }
}
```

### 1.7.3 preHandle 返回 false 时的执行规则

当某个拦截器 `preHandle` 返回 false 时：

1. 所有 `postHandle` 都不执行
2. 从当前拦截器往前数，逆序执行各拦截器的 `afterCompletion`
3. 后续拦截器和 Controller 均不执行

## 1.8 典型应用场景

### 1.8.1 登录验证

```java
@Component
public class LoginInterceptor implements HandlerInterceptor {

    @Override
    public boolean preHandle(HttpServletRequest request, HttpServletResponse response,
                              Object handler) throws Exception {
        Object user = request.getSession().getAttribute("user");
        if (user == null) {
            response.sendRedirect("/login");
            return false;
        }
        return true;
    }
}
```

### 1.8.2 请求日志记录

```java
@Component
public class LogInterceptor implements HandlerInterceptor {

    @Override
    public boolean preHandle(HttpServletRequest request, HttpServletResponse response,
                              Object handler) throws Exception {
        long start = System.currentTimeMillis();
        request.setAttribute("startTime", start);
        return true;
    }

    @Override
    public void afterCompletion(HttpServletRequest request, HttpServletResponse response,
                                 Object handler, Exception ex) throws Exception {
        long start = (Long) request.getAttribute("startTime");
        long cost = System.currentTimeMillis() - start;
        System.out.println(request.getRequestURI() + " 耗时: " + cost + "ms");
    }
}
```

### 1.8.3 统一响应头

```java
public class HeaderInterceptor implements HandlerInterceptor {

    @Override
    public void postHandle(HttpServletRequest request, HttpServletResponse response,
                           Object handler, ModelAndView modelAndView) throws Exception {
        response.setHeader("X-Application", "MyApp");
    }
}
```
