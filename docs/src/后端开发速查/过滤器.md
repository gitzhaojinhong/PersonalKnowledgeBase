# 一、SpringBoot中使用过滤器

---

## 1.1 基本概念

Filter 是 Web 三大组件之一，基于 Servlet 规范实现，以函数回调方式对请求和响应进行链式拦截与预处理。

Filter 不是 Servlet，不能直接向客户端生成响应，只能拦截已有请求并进行预处理。多个过滤器按链式顺序依次调用。

**拦截范围由配置的 URL 模式决定**，不是"所有资源"。例如配置 `urlPatterns = "/admin/*"` 就只拦截 admin 路径下的请求。

> Spring Boot 3.x 使用的是 Jakarta EE 9+，因此包名从 `javax.servlet` 变更为 `jakarta.servlet`。

---

## 1.2 过滤器与拦截器的区别

**过滤器（Filter）：**

- 作用于 Servlet 层，请求进入容器之后、到达 Controller 之前
- 基于 Servlet 规范，不依赖 Spring
- 能操作请求/响应对象
- 不能获取 Controller 方法信息
- 执行早于拦截器
- 默认只拦截 REQUEST 类型（ERROR/FORWARD/INCLUDE/ASYNC 需显式配置）

**拦截器（Interceptor）：**

- 作用于 Controller 层，请求到达控制器前后
- 基于 SpringMVC，依赖 Spring 容器
- 能操作请求/响应对象
- 能获取 Controller 方法信息（通过 HandlerMethod）
- 执行晚于过滤器
- 所有请求类型均拦截（除非通过 excludePathPatterns 排除）

## 1.3 生命周期

**init()** — 容器启动时调用，整个生命周期只执行一次，用于初始化

**doFilter()** — 每次请求被拦截时调用，核心业务逻辑在此实现

**destroy()** — 容器关闭时调用，用于释放资源

---

## 1.4 三种定义和注册方式

三种方式的本质区别是**注册机制不同**：

- @WebFilter 靠 `@ServletComponentScan` 扫描发现
- @Component 直接注册为 Spring Bean
- Java Config 用 `@Bean` + `FilterRegistrationBean` 手动注册

---

### 1.4.1 @WebFilter 注解

**注册方式：** 通过 `@ServletComponentScan` 扫描 `@WebFilter` 注解自动注册

使用 `@WebFilter` 声明过滤器，需配合 `@ServletComponentScan` 让 Spring 扫描到：

```java
@WebFilter(urlPatterns = "/admin/*", filterName = "loginFilter")
public class LoginFilter implements Filter {

    @Override
    public void init(FilterConfig filterConfig) throws ServletException {
    }

    @Override
    public void doFilter(ServletRequest request, ServletResponse response, FilterChain chain)
            throws ServletException, IOException {
        HttpServletRequest req = (HttpServletRequest) request;
        String visitIp = req.getRemoteAddr();
        visitIp = "0:0:0:0:0:0:0:1".equals(visitIp) ? "127.0.0.1" : visitIp;
        System.out.println("访问 IP = " + visitIp);
        chain.doFilter(req, response);
    }

    @Override
    public void destroy() {
    }
}
```

启动类添加扫描注解：

```java
@SpringBootApplication
@ServletComponentScan
public class Application {
    public static void main(String[] args) {
        SpringApplication.run(Application.class, args);
    }
}
```

**@WebFilter 常用属性：**

- `urlPatterns` — 拦截的 URL 模式，支持数组指定多个
- `filterName` — 过滤器名称，用于决定执行顺序，不写则默认类名
- `dispatcherTypes` — 拦截的请求类型，默认为 REQUEST
- `initParams` — 初始化参数

**执行顺序：** 按 `filterName` 的字母顺序依次执行。如需控制顺序，可通过命名间接控制（如 `A_LoginFilter` 在 `B_LoginFilter` 之前）。

> 注意：@WebFilter 没有直接设置顺序的属性

---

### 1.4.2 @Component 注解

**注册方式：** 直接注册为 Spring Bean，配合 `@Order` 控制顺序

将 Filter 注册为 Spring Bean，配合 `@Order` 注解精确设置执行顺序：

```java
@Order(1)
@Component
public class WebVisitFilter implements Filter {

    @Override
    public void init(FilterConfig filterConfig) throws ServletException {
    }

    @Override
    public void doFilter(ServletRequest request, ServletResponse response, FilterChain chain)
            throws ServletException, IOException {
        chain.doFilter(request, response);
    }

    @Override
    public void destroy() {
    }
}
```

**@Component 的局限性：**

- 无法直接指定拦截路径，默认拦截所有请求
- 如需指定路径，可在 doFilter 中手动判断：

```java
@Override
public void doFilter(ServletRequest request, ServletResponse response, FilterChain chain) {
    HttpServletRequest req = (HttpServletRequest) request;
    String uri = req.getRequestURI();

    if (uri.startsWith("/admin") || uri.startsWith("/api")) {
        // 业务处理
    } else {
        chain.doFilter(request, response);
    }
}
```

### 1.4.3 Java Config 配置类（推荐）

**注册方式：** 使用 `@Configuration` + `@Bean` + `FilterRegistrationBean` 手动注册

使用 `@Configuration` + `@Bean` 的方式，通过 `FilterRegistrationBean` 注册过滤器，灵活度最高：

```java
public class LoginFilter implements Filter {

    @Override
    public void init(FilterConfig filterConfig) throws ServletException {
    }

    @Override
    public void doFilter(ServletRequest request, ServletResponse response, FilterChain chain)
            throws ServletException, IOException {
        // 登录校验逻辑
        chain.doFilter(request, response);
    }

    @Override
    public void destroy() {
    }
}
```

```java
@Configuration
public class FilterConfig {

    @Bean
    public FilterRegistrationBean<LoginFilter> loginFilterRegistration() {
        FilterRegistrationBean<LoginFilter> registration = new FilterRegistrationBean<>();
        registration.setFilter(new LoginFilter());
        registration.addUrlPatterns("/admin/*", "/user/*");
        registration.setName("loginFilter");
        registration.setOrder(1);
        registration.setDispatcherTypes(DispatcherType.REQUEST, DispatcherType.ERROR);
        return registration;
    }
}
```

**推荐理由：**
- 可精确配置拦截路径
- 可设置执行顺序
- 可配置 DispatcherType
- 可添加初始化参数

#### 1.4.3.1 FilterRegistrationBean 常用配置方法

- `setFilter()` — 绑定过滤器实例
- `addUrlPatterns()` — 添加拦截路径，支持多个
- `setName()` — 设置过滤器名称
- `setOrder()` — 执行顺序，数字越小优先级越高
- `setEnabled()` — 是否启用
- `addInitParameter()` — 添加初始化参数
- `setDispatcherTypes()` — 配置拦截的请求类型

**DispatcherType 枚举值：**

- `REQUEST` — 普通请求（默认）
- `FORWARD` — 转发请求
- `INCLUDE` — 包含请求（jsp:include）
- `ASYNC` — 异步请求
- `ERROR` — 错误页面请求

示例：配置拦截普通请求和错误页面

```java
registration.setDispatcherTypes(DispatcherType.REQUEST, DispatcherType.ERROR);
```

## 1.5 FilterChain 执行原理

FilterChain 是责任链模式的实现，每个 Filter 处理完后调用 `chain.doFilter()` 将请求传递给下一个 Filter。

```java
public interface FilterChain {
    void doFilter(ServletRequest request, ServletResponse response);
}
```

Tomcat 中由 `ApplicationFilterChain` 实现，内部用数组存储所有过滤器，通过索引依次取出并调用。

**执行链路：**

```
请求
  ↓
Filter1#doFilter
  ↓
Filter2#doFilter
  ↓
...
  ↓
FilterN#doFilter
  ↓
Servlet（DispatcherServlet）
  ↓
Interceptor#preHandle
  ↓
Controller
  ↓
Interceptor#postHandle
  ↓
视图渲染
  ↓
Interceptor#afterCompletion
  ↓
响应
```

所有过滤器都执行完毕后，才进入 Servlet 层。

## 1.6 常见应用场景

- **登录验证** — 检查用户是否已登录，未登录则跳转
- **统一编码** — 设置请求和响应的字符编码，防止乱码
- **敏感词过滤** — 替换或拦截请求中的敏感词
- **跨域处理** — 设置 CORS 相关响应头
- **日志记录** — 记录请求访问日志

## 1.7 异常处理

Filter 中发生异常时，如果调用方没有额外处理，异常会向上抛出。可以根据业务需求决定如何处理：

```java
@Override
public void doFilter(ServletRequest request, ServletResponse response, FilterChain chain) {
    try {
        // 业务处理
        chain.doFilter(request, response);
    } catch (Exception e) {
        // 记录日志
        e.printStackTrace();
        // 转发到错误页面
        request.setAttribute("error", e.getMessage());
        request.getRequestDispatcher("/error").forward(request, response);
    }
}
```
