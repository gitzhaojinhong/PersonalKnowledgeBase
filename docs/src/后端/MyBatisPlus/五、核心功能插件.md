## 五、核心功能插件

### 5.1 逻辑删除

**逻辑删除**并非真正从数据库删除数据，而是通过一个标志位（如 `deleted` 字段）标识记录是否被删除。

#### 5.1.1 配置方式

**全局配置（推荐）**：

```yaml
mybatis-plus:
  global-config:
    db-config:
      logic-delete-field: deleted       # 全局逻辑删除字段名
      logic-delete-value: 1             # 删除后的值
      logic-not-delete-value: 0         # 未删除的值
```

**局部注解配置**：

```java
@Data
public class User {
    private Long id;
    private String username;
    
    @TableLogic  // 标识逻辑删除字段（局部优先于全局）
    private Integer deleted;
}
```

#### 5.1.2 使用示例

```java
// 逻辑删除（实际执行 UPDATE）
userService.removeById(1L);
// 等效 SQL：UPDATE t_user SET deleted=1 WHERE id=1 AND deleted=0

// 逻辑删除查询（自动附加 deleted=0 条件）
List<User> list = userService.list();
// 等效 SQL：SELECT * FROM t_user WHERE deleted=0

// 查询已删除的记录（需手动绕过）
List<User> deletedList = userMapper.selectList(
    Wrappers.<User>lambdaQuery()
        .eq(User::getDeleted, 1)
);
```

#### 5.1.3 逻辑删除字段填充

若需要在逻辑删除时同时更新删除时间，可使用**选装件** `deleteByIdWithFill`：

```java
// 配置 SQL 注入器（详见 7.3 节）
@Configuration
public class MybatisPlusConfig {
    
    @Bean
    public MybatisPlusInterceptor mybatisPlusInterceptor() {
        MybatisPlusInterceptor interceptor = new MybatisPlusInterceptor();
        // ... 其他插件
        return interceptor;
    }
    
    // 扩展注入器，添加逻辑删除填充方法
    @Bean
    public DefaultSqlInjector sqlInjector() {
        return new DefaultSqlInjector() {
            @Override
            public List<AbstractMethod> getMethodList(Class<?> mapperClass, TableInfo tableInfo) {
                List<AbstractMethod> methodList = super.getMethodList(mapperClass, tableInfo);
                methodList.add(new LogicDeleteByIdWithFill());
                return methodList;
            }
        };
    }
}
```

### 5.2 自动填充（创建/更新时间）

**自动填充**用于在插入或更新时自动为指定字段赋值（如创建时间、更新时间）。

#### 5.2.1 实体类注解配置

```java
@Data
public class User {
    private Long id;
    private String username;
    
    @TableField(fill = FieldFill.INSERT)           // 插入时填充
    private LocalDateTime createTime;
    
    @TableField(fill = FieldFill.INSERT_UPDATE)    // 插入和更新时填充
    private LocalDateTime updateTime;
    
    @TableField(fill = FieldFill.INSERT)
    private Long createUser;
    
    @TableField(fill = FieldFill.INSERT_UPDATE)
    private Long updateUser;
}
```

**填充策略枚举**：

| 策略 | 说明 |
|------|------|
| `FieldFill.DEFAULT` | 默认，不处理 |
| `FieldFill.INSERT` | 插入时填充 |
| `FieldFill.UPDATE` | 更新时填充 |
| `FieldFill.INSERT_UPDATE` | 插入和更新时均填充 |

#### 5.2.2 实现 MetaObjectHandler

```java
import com.baomidou.mybatisplus.core.handlers.MetaObjectHandler;
import org.apache.ibatis.reflection.MetaObject;
import org.springframework.stereotype.Component;
import java.time.LocalDateTime;

@Component
public class MyMetaObjectHandler implements MetaObjectHandler {
    
    @Override
    public void insertFill(MetaObject metaObject) {
        // 方式一：直接传值（立即执行）
        this.strictInsertFill(metaObject, "createTime", LocalDateTime.class, LocalDateTime.now());
        
        // 方式二：传 Supplier（延迟执行，推荐）
        this.strictInsertFill(metaObject, "updateTime", LocalDateTime.class, LocalDateTime::now);
        this.strictInsertFill(metaObject, "createUser", Long.class, this::getCurrentUserId);
        this.strictInsertFill(metaObject, "updateUser", Long.class, this::getCurrentUserId);
        
        // 初始化乐观锁版本号
        this.strictInsertFill(metaObject, "version", Integer.class, () -> 1);
        
        // 初始化逻辑删除标志
        this.strictInsertFill(metaObject, "deleted", Integer.class, () -> 0);
    }
    
    @Override
    public void updateFill(MetaObject metaObject) {
        // 更新时自动填充
        this.strictUpdateFill(metaObject, "updateTime", LocalDateTime.class, LocalDateTime::now);
        this.strictUpdateFill(metaObject, "updateUser", Long.class, this::getCurrentUserId);
    }
    
    private Long getCurrentUserId() {
        // 从 SecurityContext 或 ThreadLocal 获取当前用户ID
        // 此处简化处理
        return 1L;
    }
}
```

> **注意**：`strictInsertFill` 和 `strictUpdateFill` 会判断字段是否已有值，有值则不会覆盖。如需强制覆盖，使用 `setFieldValByName`。

### 5.3 乐观锁

**乐观锁**适用于读多写少的场景，通过版本号机制避免并发更新冲突。

#### 5.3.1 配置与使用

**Step 1：实体类添加 @Version 注解**

```java
@Data
public class Product {
    private Long id;
    private String name;
    private BigDecimal price;
    private Integer stock;
    
    @Version  // 标识乐观锁版本号字段
    private Integer version;
}
```

**Step 2：配置乐观锁插件**

```java
@Bean
public MybatisPlusInterceptor mybatisPlusInterceptor() {
    MybatisPlusInterceptor interceptor = new MybatisPlusInterceptor();
    
    // 添加乐观锁插件
    interceptor.addInnerInterceptor(new OptimisticLockerInnerInterceptor());
    
    return interceptor;
}
```

**Step 3：业务中使用**

```java
// 扣减库存（乐观锁防止超卖）
public boolean deductStock(Long productId, Integer quantity) {
    // 1. 查询商品信息
    Product product = productMapper.selectById(productId);
    if (product == null || product.getStock() < quantity) {
        return false;  // 库存不足
    }
    
    // 2. 扣减库存（WHERE 条件自动附加 version = ?）
    Product update = new Product();
    update.setId(productId);
    update.setStock(product.getStock() - quantity);
    update.setVersion(product.getVersion());  // 必须传入当前版本号
    
    int rows = productMapper.updateById(update);
    
    // 3. 判断更新结果
    if (rows == 0) {
        // 版本号不匹配，说明数据已被其他线程修改，可重试或抛异常
        throw new ConcurrentModificationException("数据已被修改，请重试");
    }
    
    return true;
}
```

**等效 SQL**：

```sql
UPDATE t_product 
SET stock = stock - 1, version = version + 1 
WHERE id = 1 AND version = 5
```

#### 5.3.2 乐观锁注意事项

- **版本号字段类型**：支持 `int`、`Integer`、`long`、`Long`、`Date`、`Timestamp`
- **更新失败处理**：返回影响行数为 0 表示版本冲突，业务层需决定重试或报错
- **只支持 updateById 和 update**：自定义 SQL 不会自动附加版本号条件
- **不支持嵌套事务中的批量更新**：`updateBatchById` 每条记录单独处理版本号

### 5.4 主键策略

MyBatis-Plus 支持多种主键生成策略，通过 `@TableId` 注解的 `type` 属性指定。

#### 5.4.1 主键策略类型

| 策略 | 说明 | 适用场景 |
|------|------|---------|
| `AUTO` | 数据库自增，依赖数据库自增字段 | **推荐**，性能最好 |
| `NONE` | 无策略，不设置主键（若手动赋值则直接使用） | 特殊业务场景 |
| `INPUT` | 手动输入，插入前需手动赋值 | 已有主键生成规则 |
| `ASSIGN_ID` | 分配 ID，使用雪花算法生成 Long 型 ID | 分布式系统 |
| `ASSIGN_UUID` | 分配 UUID，生成 UUID 字符串（无中划线） | 需要全局唯一标识 |

#### 5.4.2 配置示例

```java
// 数据库自增（推荐）
@TableId(type = IdType.AUTO)
private Long id;

// 雪花算法（分布式场景）
@TableId(type = IdType.ASSIGN_ID)
private Long id;

// UUID
@TableId(type = IdType.ASSIGN_UUID)
private String id;
```

#### 5.4.3 全局主键策略配置

```yaml
mybatis-plus:
  global-config:
    db-config:
      id-type: assign_id  # 全局默认主键策略
```

#### 5.4.4 雪花算法 ID 详解

`ASSIGN_ID` 使用改进的雪花算法生成 64 位 Long 型 ID：

```
| 1 bit 符号位 | 41 bit 时间戳 | 10 bit 工作机器ID | 12 bit 序列号 |
```

**特点**：
- 趋势递增，性能优异
- 不依赖数据库，分布式场景下全局唯一
- 生成 ID 为 Long 型，前端接收时可能出现精度丢失（需转为 String）

#### 5.4.5 自定义 ID 生成器

当内置主键策略无法满足需求时，可实现 `IdentifierGenerator` 接口自定义 ID 生成逻辑。

```java
import com.baomidou.mybatisplus.core.incrementer.IdentifierGenerator;
import org.springframework.stereotype.Component;

@Component
public class CustomIdGenerator implements IdentifierGenerator {
    
    @Override
    public Number nextId(Object entity) {
        // 自定义 Number 类型 ID 生成逻辑
        // 例如：基于 Redis 自增、号段模式等
        return generateId();
    }
    
    @Override
    public String nextUUID(Object entity) {
        // 自定义 String 类型 UUID 生成逻辑
        return generateCustomUUID();
    }
    
    private long generateId() {
        // 实际业务实现
        return System.currentTimeMillis();
    }
    
    private String generateCustomUUID() {
        // 实际业务实现
        return java.util.UUID.randomUUID().toString().replace("-", "");
    }
}
```

> **注意**：自定义生成器需注册为 Spring Bean，MyBatis-Plus 会自动检测并使用。

### 5.5 多租户（Tenant Line）

**多租户**用于 SaaS 系统，实现同一张表隔离不同租户的数据。

#### 5.5.1 配置多租户插件

```java
@Bean
public MybatisPlusInterceptor mybatisPlusInterceptor() {
    MybatisPlusInterceptor interceptor = new MybatisPlusInterceptor();
    
    // 添加多租户插件
    interceptor.addInnerInterceptor(
        new TenantLineInnerInterceptor(new TenantLineHandler() {
            
            @Override
            public Expression getTenantId() {
                // 从当前上下文获取租户ID
                Long tenantId = TenantContextHolder.getCurrentTenantId();
                return new LongValue(tenantId);
            }
            
            @Override
            public String getTenantIdColumn() {
                return "tenant_id";  // 租户字段名
            }
            
            @Override
            public boolean ignoreTable(String tableName) {
                // 忽略特定表（如全局配置表不需要租户隔离）
                return "t_config".equals(tableName) 
                    || "t_dict".equals(tableName);
            }
        })
    );
    
    return interceptor;
}
```

#### 5.5.2 租户上下文

```java
public class TenantContextHolder {
    private static final ThreadLocal<Long> TENANT_ID = new ThreadLocal<>();
    
    public static void setCurrentTenantId(Long tenantId) {
        TENANT_ID.set(tenantId);
    }
    
    public static Long getCurrentTenantId() {
        return TENANT_ID.get();
    }
    
    public static void clear() {
        TENANT_ID.remove();
    }
}
```

#### 5.5.3 拦截效果

配置后，所有 SQL 会自动附加 `AND tenant_id = ?` 条件：

```sql
-- 原始 SQL
SELECT * FROM t_user WHERE age > 18

-- 自动改写为
SELECT * FROM t_user WHERE age > 18 AND tenant_id = 1001
```

### 5.6 动态表名

**动态表名**用于分表场景（如按年月分表），根据运行时条件动态替换表名。

#### 5.6.1 配置动态表名插件

```java
@Bean
public MybatisPlusInterceptor mybatisPlusInterceptor() {
    MybatisPlusInterceptor interceptor = new MybatisPlusInterceptor();
    
    // 动态表名插件
    interceptor.addInnerInterceptor(
        new DynamicTableNameInnerInterceptor(
            // 表名替换逻辑
            (sql, tableName) -> {
                // 只对 t_order 表进行动态替换
                if ("t_order".equals(tableName)) {
                    String month = OrderTableContextHolder.getCurrentMonth();  // 如 "202401"
                    return tableName + "_" + month;  // t_order_202401
                }
                return tableName;
            }
        )
    );
    
    return interceptor;
}
```

#### 5.6.2 使用示例

```java
// 设置当前月份
OrderTableContextHolder.setCurrentMonth("202401");

try {
    // 查询时会自动路由到 t_order_202401 表
    List<Order> orders = orderMapper.selectList(
        Wrappers.<Order>lambdaQuery().eq(Order::getUserId, 1L)
    );
} finally {
    OrderTableContextHolder.clear();
}
```

### 5.8 流式查询（3.5.4+）

**流式查询**通过 `ResultHandler` 接口实现结果集的流式处理，适用于**大数据量跑批**或**内存敏感**的场景，避免一次性加载全部数据到内存导致 OOM。

#### 5.8.1 使用示例

```java
// 示例1：全表流式处理
userMapper.selectList(Wrappers.emptyWrapper(), new ResultHandler<User>() {
    int count = 0;
    
    @Override
    public void handleResult(ResultContext<? extends User> resultContext) {
        User user = resultContext.getResultObject();
        long resultCount = resultContext.getResultCount();  // 从 1 开始计数
        
        // 逐条处理业务逻辑
        processUser(user);
        
        // 可随时中断（类似 break）
        if (count++ >= 1000) {
            resultContext.stop();
        }
    }
});

// 示例2：结合分页进行批量流式处理（处理 10 万条数据）
Page<User> page = new Page<>(1, 100000);
userMapper.selectList(page, Wrappers.emptyWrapper(), new ResultHandler<User>() {
    @Override
    public void handleResult(ResultContext<? extends User> resultContext) {
        User user = resultContext.getResultObject();
        // 逐条处理，内存中只保留当前记录
        exportToFile(user);
    }
});
```

#### 5.8.2 注意事项

- **版本要求**：需 MyBatis-Plus **3.5.4 及以上版本**
- **分页配合**：结合分页使用时建议关闭 COUNT 查询：`new Page<>(1, 100000, false)`
- **中断处理**：调用 `resultContext.stop()` 可随时终止结果集处理
- **连接保持**：流式查询期间数据库连接保持打开，处理完成后自动释放

### 5.9 数据权限插件（DataPermissionInterceptor）

**数据权限插件**用于实现行级数据权限控制，原理与多租户插件类似，在 SQL 执行前动态拼接权限相关的 WHERE 条件。

#### 5.9.1 配置数据权限插件

```java
@Bean
public MybatisPlusInterceptor mybatisPlusInterceptor() {
    MybatisPlusInterceptor interceptor = new MybatisPlusInterceptor();
    
    // 添加数据权限插件
    interceptor.addInnerInterceptor(
        new DataPermissionInterceptor(new DataPermissionHandler() {
            
            @Override
            public Expression getSqlSegment(Expression where, String mappedStatementId) {
                // 获取当前用户的数据权限范围
                DataScope scope = SecurityUtils.getCurrentUserDataScope();
                
                if (scope == null || scope.isAll()) {
                    return where;  // 全部权限，不限制
                }
                
                // 构建权限条件：dept_id IN (1, 2, 3)
                InExpression inExpression = new InExpression();
                inExpression.setLeftExpression(new Column("dept_id"));
                ExpressionList items = new ExpressionList();
                items.setExpressions(
                    scope.getDeptIds().stream()
                        .map(LongValue::new)
                        .collect(Collectors.toList())
                );
                inExpression.setRightExpression(items);
                
                // 与原 WHERE 条件拼接
                if (where == null) {
                    return inExpression;
                }
                return new AndExpression(where, inExpression);
            }
        })
    );
    
    return interceptor;
}
```

#### 5.9.2 权限上下文

```java
public class SecurityUtils {
    private static final ThreadLocal<DataScope> DATA_SCOPE = new ThreadLocal<>();
    
    public static void setCurrentUserDataScope(DataScope scope) {
        DATA_SCOPE.set(scope);
    }
    
    public static DataScope getCurrentUserDataScope() {
        return DATA_SCOPE.get();
    }
    
    public static void clear() {
        DATA_SCOPE.remove();
    }
}
```

> **适用场景**：部门隔离、数据范围控制（如只能查看本部门及下级部门数据）。
