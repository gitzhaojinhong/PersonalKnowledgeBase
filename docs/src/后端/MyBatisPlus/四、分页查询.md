## 四、分页查询

### 4.1 分页插件配置

**MyBatis-Plus 3.5.x 推荐使用新版分页插件配置**：

```java
import com.baomidou.mybatisplus.annotation.DbType;
import com.baomidou.mybatisplus.extension.plugins.MybatisPlusInterceptor;
import com.baomidou.mybatisplus.extension.plugins.inner.PaginationInnerInterceptor;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

@Configuration
public class MybatisPlusConfig {
    
    @Bean
    public MybatisPlusInterceptor mybatisPlusInterceptor() {
        MybatisPlusInterceptor interceptor = new MybatisPlusInterceptor();
        
        // 添加分页插件（必须指定数据库类型）
        PaginationInnerInterceptor paginationInterceptor = new PaginationInnerInterceptor(DbType.MYSQL);
        paginationInterceptor.setMaxLimit(1000L);  // 单页最大 1000 条
        paginationInterceptor.setOverflow(true);   // 页码溢出时回到最后一页
        
        interceptor.addInnerInterceptor(paginationInterceptor);
        
        return interceptor;
    }
}
```

> **重要**：**分页插件必须配置数据库类型**（`DbType.MYSQL`、`DbType.POSTGRE_SQL` 等），否则分页 SQL 无法正常生成。

### 4.2 基础分页用法

```java
// 1. 无条件分页（查询第 1 页，每页 10 条）
Page<User> page = new Page<>(1, 10);
Page<User> result = userService.page(page);

System.out.println("当前页：" + result.getCurrent());
System.out.println("每页大小：" + result.getSize());
System.out.println("总记录数：" + result.getTotal());
System.out.println("总页数：" + result.getPages());
System.out.println("数据列表：" + result.getRecords());

// 2. 条件分页
Page<User> page2 = new Page<>(2, 20);
LambdaQueryWrapper<User> wrapper = Wrappers.lambdaQuery()
    .gt(User::getAge, 18)
    .orderByDesc(User::getCreateTime);

Page<User> result2 = userService.page(page2, wrapper);

// 3. Mapper 层直接分页
Page<User> page3 = new Page<>(1, 10);
Page<User> result3 = userMapper.selectPage(page3, 
    Wrappers.<User>lambdaQuery().eq(User::getStatus, 1)
);
```

### 4.3 分页对象 Page 详解

```java
// Page 构造方法
Page<T> page = new Page<>();
Page<T> page = new Page<>(long current);           // 指定当前页
Page<T> page = new Page<>(long current, long size); // 指定当前页和每页大小
Page<T> page = new Page<>(current, size, boolean searchCount); // 是否查询总记录数
Page<T> page = new Page<>(current, size, long total); // 直接传入总记录数（不执行 COUNT 查询）
```

**Page 核心属性**：

| 属性                 | 说明                      |
|--------------------|-------------------------|
| `current`          | 当前页码（从 1 开始）            |
| `size`             | 每页显示条数                  |
| `total`            | 总记录数                    |
| `pages`            | 总页数                     |
| `records`          | 当前页数据列表                 |
| `orders`           | 排序信息                    |
| `optimizeCountSql` | 是否优化 COUNT SQL（默认 true） |
| `searchCount`      | 是否进行 COUNT 查询（默认 true）  |

### 4.4 分页最佳实践

#### 4.4.1 PageParam 分页入参

```java
@Data
public class PageParam {
    @Min(value = 1, message = "页码最小为1")
    private Integer pageNum = 1;

    @Min(value = 1, message = "每页条数最小为1")
    @Max(value = 100, message = "每页条数最大为100")
    private Integer pageSize = 10;

    /** 排序字段（白名单校验防 SQL 注入） */
    private String orderBy;

    /** 排序方式：ASC / DESC */
    private String orderDir = "ASC";

    /** 排序字段白名单，防 SQL 注入 */
    private static final Set<String> ALLOWED_COLUMNS = Set.of(
        "id", "create_time", "update_time", "status"
    );

    public static String safeColumn(String column) {
        if (!ALLOWED_COLUMNS.contains(column)) {
            throw new IllegalArgumentException("非法排序字段: " + column);
        }
        return column;
    }
}
```

#### 4.4.2 PageResult 分页出参

```java
@Data
public class PageResult<T> {
    private List<T> records;
    private Long total;
    private Long current;
    private Long size;
    private Long pages;

    public static <T> PageResult<T> of(IPage<T> page) {
        PageResult<T> result = new PageResult<>();
        result.setRecords(page.getRecords());
        result.setTotal(page.getTotal());
        result.setCurrent(page.getCurrent());
        result.setSize(page.getSize());
        result.setPages(page.getPages());
        return result;
    }
}
```

#### 4.4.3 通用分页工具类

```java
/**
 * MyBatis-Plus 分页工具类
 * 封装两种通用分页方式，抽取 buildPage 消除重复
 */
public class PageUtil {

    private static <T> Page<T> buildPage(PageParam param) {
        long current = Math.max(1, param.getPageNum());
        long size = Math.max(1, param.getPageSize());
        Page<T> page = new Page<>(current, size);

        // 排序（白名单校验后设置）
        if (StringUtils.hasText(param.getOrderBy())) {
            String column = PageParam.safeColumn(param.getOrderBy());
            boolean isAsc = "ASC".equalsIgnoreCase(param.getOrderDir());
            page.addOrder(new OrderItem(column, isAsc));
        }
        return page;
    }

    /**
     * 方式1：lambda 内部自己带 wrapper，适合灵活写条件
     */
    public static <T> PageResult<T> queryPage(PageParam param,
                                              Function<Page<T>, IPage<T>> queryFunction) {
        @SuppressWarnings("unchecked")
        Page<T> page = (Page<T>) buildPage(param);
        return PageResult.of(queryFunction.apply(page));
    }

    /**
     * 方式2：传入 wrapper + 双参数函数，支持方法引用 userMapper::selectPage
     */
    public static <T> PageResult<T> queryPage(PageParam param,
                                              Wrapper<T> wrapper,
                                              BiFunction<Page<T>, Wrapper<T>, IPage<T>> queryFunction) {
        @SuppressWarnings("unchecked")
        Page<T> page = (Page<T>) buildPage(param);
        return PageResult.of(queryFunction.apply(page, wrapper));
    }
}
```

#### 4.4.4 使用示例

方式1 — lambda 内部带 wrapper（适合灵活查询）：

```java
PageParam param = new PageParam();
param.setPageNum(1);
param.setPageSize(10);

PageResult<User> pageResult = PageUtil.queryPage(param, page ->
    userMapper.selectPage(page,
        Wrappers.<User>lambdaQuery().eq(User::getStatus, 1)
    )
);
```

方式2 — wrapper + 方法引用（最简洁）：

```java
PageParam param = new PageParam();
LambdaQueryWrapper<User> wrapper = Wrappers.<User>lambdaQuery()
        .eq(User::getStatus, 1);

PageResult<User> pageResult = PageUtil.queryPage(param, wrapper, userMapper::selectPage);
```

#### 4.4.5 两种方式对比

| 维度 | 方式1 Function | 方式2 BiFunction |
|------|---------------|-----------------|
| 调用写法 | lambda 表达式 | 方法引用，更简洁 |
| 灵活性 | 任意查询逻辑均可 | 仅限标准 selectPage 场景 |
| 适用场景 | 自定义 XML 查询、复杂条件 | 标准 Mapper 分页查询 |
| 推荐 | 通用场景首选 | 简单查询更优雅 |

> 两者可并存：日常用方式2简洁调用，复杂查询回退方式1。

#### 4.4.6 分页插件配置要点

```java
@Bean
public MybatisPlusInterceptor mybatisPlusInterceptor() {
    MybatisPlusInterceptor interceptor = new MybatisPlusInterceptor();

    PaginationInnerInterceptor pagination = new PaginationInnerInterceptor(DbType.MYSQL);
    pagination.setMaxLimit(100L);    // 全局兜底，单页最多100条
    pagination.setOverflow(true);    // 页码溢出返回最后一页而非空

    interceptor.addInnerInterceptor(pagination);
    // ⚠️ 分页插件必须放最后！否则 COUNT SQL 可能不准
    return interceptor;
}
```

#### 4.4.7 常见踩坑

| 坑 | 说明 |
|---|---|
| 分页插件必须放最后 | 多插件时 `PaginationInnerInterceptor` 放最后，否则 COUNT SQL 不准 |
| 务必指定 DbType | 单数据源必须指定，如 `DbType.MYSQL`，否则可能生成错误方言 |
| 自定义 SQL 分页 | Mapper 方法第一个参数必须是 `IPage` / `Page`，否则 MP 不拦截 |
| 大数据量深度分页 | `LIMIT 1000000, 10` 极慢，考虑游标分页（基于 lastId） |
| 不需要 total 时 | `page.setSearchCount(false)` 可省掉一次 COUNT 查询 |
| Left Join 别名 | 生成 countSql 时会自动移除不参与 where 的 left join，务必加别名避免优化错误 |
