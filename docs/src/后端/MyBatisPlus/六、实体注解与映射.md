## 六、实体注解与映射

### 6.1 常用注解速查表

| 注解 | 作用目标 | 说明 |
|------|---------|------|
| `@TableName` | 类 | 指定数据库表名 |
| `@TableId` | 字段 | 标识主键字段 |
| `@TableField` | 字段 | 标识普通字段，可配置映射关系 |
| `@TableLogic` | 字段 | 标识逻辑删除字段 |
| `@Version` | 字段 | 标识乐观锁版本号字段 |
| `@EnumValue` | 枚举字段 | 标识枚举值映射字段 |
| `@KeySequence` | 类 | 指定 Oracle 等序列名 |

### 6.2 @TableName 详解

```java
// 基础用法：指定表名
@TableName("t_user")
public class User {}

// 指定 schema 和表名
@TableName(value = "t_user", schema = "mydb")
public class User {}

// 动态表名（配合动态表名插件）
@TableName("t_user")
public class User {}

// 排除全局表名前缀
@TableName(value = "user", keepGlobalPrefix = false)
public class User {}

// 指定 resultMap
@TableName(value = "t_user", resultMap = "userResultMap")
public class User {}
```

### 6.3 @TableId 详解

```java
public class User {
    
    // 基础用法：数据库自增主键
    @TableId(type = IdType.AUTO)
    private Long id;
    
    // 指定主键列名（当字段名与列名不一致时）
    @TableId(value = "user_id", type = IdType.ASSIGN_ID)
    private Long id;
}
```

### 6.3.1 @KeySequence（Oracle / PostgreSQL 序列）

对于使用数据库序列（如 Oracle、PostgreSQL）生成主键的场景，使用 `@KeySequence` 注解：

```java
@Data
@KeySequence(value = "SEQ_USER", dbType = DbType.ORACLE)  // 指定序列名和数据库类型
@TableName("t_user")
public class User {
    @TableId(type = IdType.INPUT)  // 序列方式需配合 INPUT 策略
    private Long id;
}
```

**全局序列配置（Spring Boot）**：

```java
@Configuration
public class MybatisPlusConfig {
    
    @Bean
    public MybatisPlusPropertiesCustomizer plusPropertiesCustomizer() {
        return properties -> {
            GlobalConfig globalConfig = properties.getGlobalConfig();
            globalConfig.getDbConfig().setKeyGenerator(
                new OracleKeyGenerator()  // 或 PostgreKeyGenerator
            );
        };
    }
}
```

### 6.4 @TableField 详解

```java
public class User {
    
    // 1. 字段名映射（数据库列名与属性名不一致）
    @TableField("user_name")
    private String username;
    
    // 2. 排除字段（不参与 SQL 拼接）
    @TableField(exist = false)
    private String remark;
    
    // 3. 自动填充配置
    @TableField(fill = FieldFill.INSERT)
    private LocalDateTime createTime;
    
    // 4. 字段查询条件（SELECT 时不查询该字段）
    @TableField(select = false)
    private String password;
    
    // 5. 自定义 JDBC 类型
    @TableField(jdbcType = JdbcType.VARCHAR)
    private String jsonData;
    
    // 6. 自定义 TypeHandler
    @TableField(typeHandler = JacksonTypeHandler.class)
    private UserExtra extra;
    
    // 7. 字段级插入/更新策略（覆盖全局配置）
    @TableField(insertStrategy = FieldStrategy.NOT_NULL, 
                updateStrategy = FieldStrategy.NOT_NULL)
    private String phone;
}
```

#### 6.4.1 @TableField 的字段策略

| 属性 | 说明 | 可选值 |
|------|------|--------|
| `insertStrategy` | 插入时的字段策略 | `DEFAULT`, `IGNORED`, `NOT_NULL`, `NOT_EMPTY`, `NEVER` |
| `updateStrategy` | 更新时的字段策略 | `DEFAULT`, `IGNORED`, `NOT_NULL`, `NOT_EMPTY`, `NEVER` |
| `whereStrategy` | WHERE 条件中的字段策略 | `DEFAULT`, `IGNORED`, `NOT_NULL`, `NOT_EMPTY`, `NEVER` |

**FieldStrategy 说明**：

| 策略 | 说明 |
|------|------|
| `DEFAULT` | 默认，跟随全局配置 |
| `IGNORED` | 忽略判断，字段值始终参与 SQL |
| `NOT_NULL` | 非 NULL 判断，字段值不为 NULL 才参与 SQL |
| `NOT_EMPTY` | 非空判断，字段值不为 NULL 且不为空字符串才参与 SQL |
| `NEVER` | 从不参与 SQL |

**全局字段策略配置**：

```yaml
mybatis-plus:
  global-config:
    db-config:
      insert-strategy: not_null    # 全局插入策略
      update-strategy: not_null    # 全局更新策略
      select-strategy: not_null    # 全局 WHERE 条件策略
```

### 6.5 枚举类型处理

#### 6.5.1 默认处理器：CompositeEnumTypeHandler

MyBatis-Plus 3.5.2+ 默认使用 `CompositeEnumTypeHandler` 作为枚举处理器。它的核心行为是：

1. **写入数据库时**：遍历枚举字段，查找标记了 `@EnumValue` 的字段值，将其作为数据库存储值
2. **从数据库读取时**：遍历枚举字段，将数据库值与 `@EnumValue` 标记字段的值做匹配，找到对应枚举实例
3. **Fallback 机制**：如果枚举类上没有 `@EnumValue` 注解，则退化使用枚举的 `name()` 字符串（等效于 MyBatis 原生的 `EnumTypeHandler`）

> **与 MyBatis 原生处理器的区别**：
>
> - `EnumTypeHandler`：存储枚举 `name()` 字符串（如 `"ENABLED"`）
> - `EnumOrdinalTypeHandler`：存储枚举 `ordinal()` 整数值（如 `1`）
> - `CompositeEnumTypeHandler`：存储 `@EnumValue` 标记字段的值（如 `1`，对应 `code` 字段），行为更可控

#### 6.5.2 定义通用枚举

```java
import com.baomidou.mybatisplus.annotation.EnumValue;
import com.fasterxml.jackson.annotation.JsonValue;

public enum UserStatus {
    DISABLED(0, "禁用"),
    ENABLED(1, "正常"),
    VIP(2, "VIP");

    @EnumValue  // 标记数据库存储值（CompositeEnumTypeHandler 识别这个注解）
    private final int code;

    @JsonValue  // 标记 JSON 序列化值（供 Jackson 使用）
    private final String desc;

    UserStatus(int code, String desc) {
        this.code = code;
        this.desc = desc;
    }
}
```

**数据库中的存储结果**：

| 枚举值 | 数据库存储值 |
|--------|-------------|
| `UserStatus.DISABLED` | `0` |
| `UserStatus.ENABLED` | `1` |
| `UserStatus.VIP` | `2` |

#### 6.5.3 实体类中使用

```java
@Data
public class User {
    private Long id;
    private String username;
    private UserStatus status;  // 直接使用枚举类型，CompositeEnumTypeHandler 自动处理
}
```

#### 6.5.4 配置扫描枚举包（Spring Boot）

```yaml
mybatis-plus:
  type-enums-package: com.example.enums  # 枚举包扫描路径，扫描后 CompositeEnumTypeHandler 才能正确反序列化
```

#### 6.5.5 手动指定枚举处理器

如果某个字段不想用默认的 `CompositeEnumTypeHandler`，可以手动指定：

```java
@Data
public class User {
    private Long id;

    // 使用 MyBatis 原生 EnumOrdinalTypeHandler（存储 ordinal 整数值）
    @TableField(typeHandler = EnumOrdinalTypeHandler.class)
    private UserStatus status;
}
```

**全局修改默认枚举处理器**：

```yaml
mybatis-plus:
  configuration:
    default-enum-type-handler: org.apache.ibatis.type.EnumOrdinalTypeHandler
```

> **注意**：修改全局默认处理器后，`@EnumValue` 将不再生效，所有未显式指定 `typeHandler` 的枚举字段都会走 `EnumOrdinalTypeHandler`。

#### 6.5.6 自定义枚举处理器（老版本兼容）

MyBatis-Plus 3.5.2 之前没有 `CompositeEnumTypeHandler`，推荐实现 `IEnum` 接口：

```java
// 3.5.2 之前版本：实现 IEnum 接口
public enum UserStatus implements IEnum<Integer> {
    DISABLED(0, "禁用"),
    ENABLED(1, "正常");

    private final int value;
    private final String desc;

    @Override
    public Integer getValue() {
        return this.value;
    }
}
```

此时 `getValue()` 的返回值会被 `IEnumTypeHandler` 存入数据库，行为与 `@EnumValue` 类似。

### 6.6 字段类型处理器 TypeHandler

用于处理 JSON、加密等特殊字段类型。

#### 6.6.1 JSON 字段处理（Jackson）

```java
@Data
public class User {
    private Long id;
    
    // 自动将 UserExtra 对象与 JSON 字符串互相转换,
    //JacksonTypeHandler只做 对象 ↔ JSON 字符串 序列化,如需额外操作可自定义 EncryptTypeHandler：做 明文 ↔ 加密密文
    @TableField(typeHandler = JacksonTypeHandler.class)
    private UserExtra extra;
}
```

#### 6.6.2 自定义 TypeHandler

```java
@MappedTypes(String.class)
@MappedJdbcTypes(JdbcType.VARCHAR)
public class EncryptTypeHandler extends BaseTypeHandler<String> {
    
    @Override
    public void setNonNullParameter(PreparedStatement ps, int i, 
                                     String parameter, JdbcType jdbcType) throws SQLException {
        // 写入时加密
        ps.setString(i, AESUtil.encrypt(parameter));
    }
    
    @Override
    public String getNullableResult(ResultSet rs, String columnName) throws SQLException {
        // 读取时解密
        String value = rs.getString(columnName);
        return value == null ? null : AESUtil.decrypt(value);
    }
    
    @Override
    public String getNullableResult(ResultSet rs, int columnIndex) throws SQLException {
        String value = rs.getString(columnIndex);
        return value == null ? null : AESUtil.decrypt(value);
    }
    
    @Override
    public String getNullableResult(CallableStatement cs, int columnIndex) throws SQLException {
        String value = cs.getString(columnIndex);
        return value == null ? null : AESUtil.decrypt(value);
    }
}
```
