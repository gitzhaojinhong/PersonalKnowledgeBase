## 7.1 CountDownLatch

### 7.1.1 核心原理

`CountDownLatch`（倒计时门栓）允许一个或多个线程**等待其他线程完成**。通过一个计数器实现，计数器初始为正数，调用 `countDown()` 时减 1，调用 `await()` 的线程阻塞直到计数器归零。

```java
public class CountDownLatchDemo {

    // 场景：主线程等待 N 个子线程执行完毕再继续
    public void demo() throws InterruptedException {
        int threadCount = 10;
        CountDownLatch latch = new CountDownLatch(threadCount);

        for (int i = 0; i < threadCount; i++) {
            new Thread(() -> {
                try {
                    System.out.println("线程 " + Thread.currentThread().getName() + " 执行中");
                    Thread.sleep(100);
                } finally {
                    latch.countDown(); // 计数器 -1
                }
            }).start();
        }

        latch.await(); // 阻塞，直到计数器归零
        System.out.println("所有线程执行完毕，主线程继续");
    }
}
```

### 7.1.2 典型场景

| 场景 | 示例 |
|:---|:---|
| **多线程初始化** | 等待所有服务启动完成后才接收请求 |
| **多线程计算合并** | N 个线程分别计算部分结果，主线程汇总 |
| **超时控制** | `await(timeout, unit)` 超时后不再等待继续执行 |

> **注意**：`CountDownLatch` 是一次性的，计数器归零后无法重置。若需要重用，使用 `CyclicBarrier`。

## 7.2 CyclicBarrier

### 7.2.1 核心原理

`CyclicBarrier`（循环栅栏）让一组线程相互等待，**直到所有线程都到达某个屏障点后再同时继续执行**。支持**自动重置**（循环使用）。

```java
public class CyclicBarrierDemo {

    // 场景：N 个线程同时开始执行（齐头并进）
    public void demo() throws InterruptedException {
        int threadCount = 5;
        CyclicBarrier barrier = new CyclicBarrier(threadCount, () -> {
            // 所有线程到达后执行此 Runnable（可选）
            System.out.println("所有线程都已到达屏障点，协调线程执行汇总");
        });

        for (int i = 0; i < threadCount; i++) {
            new Thread(() -> {
                try {
                    System.out.println("线程 " + Thread.currentThread().getName() + " 准备中...");
                    barrier.await(); // 等待所有线程
                    System.out.println("线程 " + Thread.currentThread().getName() + " 开始执行！");
                } catch (InterruptedException | BrokenBarrierException e) {
                    e.printStackTrace();
                }
            }).start();
        }
    }
}
```

### 7.2.2 与CountDownLatch对比

| 维度 | CountDownLatch | CyclicBarrier |
|:---|:---|:---|
| **用途** | 一个/多个线程等待其他线程 | 一组线程相互等待 |
| **重置** | ❌ 一次性（计数器不可重置） | ✅ 可循环使用 |
| **触发机制** | 外部调用 `countDown()` | 各线程内部调用 `await()` |
| **典型场景** | 主线程等待子线程完成 | 多线程分阶段任务同步 |

## 7.3 Semaphore

### 7.3.1 核心原理

`Semaphore`（信号量）用于**控制同时访问特定资源的线程数量**，通过计数器管理许可证的发放和回收。

```java
public class SemaphoreDemo {

    // 场景：限制数据库连接池最多 5 个并发连接
    public void databaseConnectionPool() throws InterruptedException {
        Semaphore semaphore = new Semaphore(5); // 5 个许可证

        for (int i = 0; i < 20; i++) {
            new Thread(() -> {
                try {
                    semaphore.acquire(); // 获取许可证（无可用则阻塞）
                    System.out.println("线程 " + Thread.currentThread().getName()
                        + " 获取到连接");

                    // 模拟使用数据库连接
                    Thread.sleep(1000);

                    semaphore.release(); // 释放许可证
                } catch (InterruptedException e) {
                    Thread.currentThread().interrupt();
                }
            }).start();
        }
    }

    // 场景：公平 vs 非公平
    public void fairDemo() throws InterruptedException {
        Semaphore fairSemaphore = new Semaphore(3, true); // 公平模式
        Semaphore unfairSemaphore = new Semaphore(3);    // 非公平模式（默认）
    }
}
```

### 7.3.2 典型场景

| 场景 | 说明 |
|:---|:---|
| **限流** | 控制同时访问某个资源的并发数（如 API 调用频率限制） |
| **资源池** | 管理数据库连接池、线程池等有限资源 |
| **令牌桶** | 配合计数器实现流量控制 |

## 7.4 Phaser

### 7.4.1 核心原理

`Phaser`（阶段同步器）是 JDK 7 引入的强大工具，支持**动态注册 parties**、**多阶段屏障**，比 `CountDownLatch` 和 `CyclicBarrier` 更灵活。

```java
public class PhaserDemo {

    // 场景：多阶段任务并行执行
    public void multiPhaseDemo() throws InterruptedException {
        Phaser phaser = new Phaser(3); // 初始 3 个参与者

        for (int i = 0; i < 3; i++) {
            final int phase = i;
            new Thread(() -> {
                System.out.println("阶段" + phase + " - 线程"
                    + Thread.currentThread().getName() + " 第1步完成");
                phaser.arriveAndAwaitAdvance(); // 等待其他线程到达

                System.out.println("阶段" + phase + " - 线程"
                    + Thread.currentThread().getName() + " 第2步完成");
                phaser.arriveAndAwaitAdvance(); // 等待其他线程到达

                System.out.println("阶段" + phase + " - 线程"
                    + Thread.currentThread().getName() + " 第3步完成");
                phaser.arriveAndDeregister(); // 完成任务并注销
            }).start();
        }

        phaser.awaitAdvance(phaser.getPhase()); // 主线程等待所有阶段完成
        System.out.println("所有阶段执行完毕");
    }

    // 动态注册参与者
    public void dynamicRegistration() throws InterruptedException {
        Phaser phaser = new Phaser();
        phaser.register(); // 注册一个参与者

        new Thread(() -> {
            phaser.arriveAndAwaitAdvance();
            System.out.println("线程1到达");
        }).start();

        new Thread(() -> {
            phaser.register(); // 动态注册新参与者
            phaser.arriveAndAwaitAdvance();
            System.out.println("线程2到达");
        }).start();

        Thread.sleep(100);
        phaser.arriveAndDeregister();
    }
}
```

## 7.5 Exchanger

### 7.5.1 核心原理

`Exchanger`（交换器）用于两个线程之间**交换数据**，当两个线程都到达交换点时互相交换数据。

```java
public class ExchangerDemo {
    private Exchanger<String> exchanger = new Exchanger<>();

    public void demo() throws InterruptedException {
        Thread t1 = new Thread(() -> {
            try {
                String data = exchanger.exchange("数据来自线程1");
                System.out.println("线程1 收到: " + data);
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            }
        }, "T1");

        Thread t2 = new Thread(() -> {
            try {
                String data = exchanger.exchange("数据来自线程2");
                System.out.println("线程2 收到: " + data);
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            }
        }, "T2");

        t1.start();
        t2.start();
    }
}
```

> **注意**：`Exchanger` 只能用于两个线程之间的数据交换，**不适合两个以上线程的场景**。

## 7.6 同步工具类对比总结

| 工具 | 作用 | 特点 |
|:---|:---|:---|
| `CountDownLatch` | 倒计时门栓 | 一次性，线程等待其他线程完成 |
| `CyclicBarrier` | 循环栅栏 | 可重置，一组线程相互等待 |
| `Semaphore` | 信号量 | 控制并发数，限流 |
| `Phaser` | 阶段同步器 | 多阶段，动态注册，最灵活 |
| `Exchanger` | 交换器 | 两线程交换数据 |
