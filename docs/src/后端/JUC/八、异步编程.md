## 8.1 Future与FutureTask

### 8.1.1 Future接口

`Future` 表示一个异步计算的结果，提供以下方法：

| 方法 | 说明 |
|:---|:---|
| `get()` | 阻塞获取结果（可中断） |
| `get(timeout, unit)` | 超时获取结果 |
| `cancel(boolean mayInterruptIfRunning)` | 取消任务 |
| `isDone()` | 判断是否完成 |
| `isCancelled()` | 判断是否被取消 |

```java
public class FutureDemo {
    public void demo() throws Exception {
        ExecutorService executor = Executors.newFixedThreadPool(4);

        Future<Integer> future = executor.submit(() -> {
            Thread.sleep(1000);
            return 42;
        });

        // 方式一：阻塞等待
        Integer result = future.get(); // 最多等待
        System.out.println("结果: " + result);

        // 方式二：轮询检查（不推荐，浪费 CPU）
        while (!future.isDone()) {
            Thread.sleep(100);
        }

        // 方式三：超时获取
        future.get(5, TimeUnit.SECONDS);

        executor.shutdown();
    }
}
```

### 8.1.2 FutureTask

`FutureTask` 是 `Future` 的实现类，同时实现了 `Runnable` 和 `Future`，可直接交给 `Thread` 或线程池执行。

```java
public class FutureTaskDemo {
    public static void main(String[] args) throws Exception {
        FutureTask<Integer> task = new FutureTask<>(() -> {
            // Callable 实现
            Thread.sleep(1000);
            return 100;
        });

        // 方式一：交给 Thread 执行
        new Thread(task).start();

        // 方式二：交给线程池执行
        ExecutorService executor = Executors.newCachedThreadPool();
        Future<Integer> future = executor.submit(task);

        Integer result = future.get();
        System.out.println("计算结果: " + result);

        executor.shutdown();
    }
}
```

> **注意**：`Future` 的局限性在于无法便捷地将多个异步任务组合、链式调用，也不支持回调。高阶异步编程推荐使用 `CompletableFuture`。

### 8.1.3 实战：缓存预热

利用 `FutureTask` 的**执行与获取结果分离**特性，实现高并发场景下**只加载一次**的缓存预热模式。

```java
public class FutureTaskCache {

    private static final ConcurrentHashMap<String, FutureTask<String>> cache =
        new ConcurrentHashMap<>();

    /**
     * 高并发安全获取：同一个 key 只有一个线程执行加载，其他线程等待结果
     */
    public static String get(String key) throws Exception {
        FutureTask<String> futureTask = cache.get(key);

        if (futureTask == null) {
            Callable<String> callable = () -> loadFromDB(key);
            FutureTask<String> newTask = new FutureTask<>(callable);

            // putIfAbsent 保证只有一个线程成功放入
            FutureTask<String> existing = cache.putIfAbsent(key, newTask);

            if (existing == null) {
                // 只有第一个线程执行加载
                futureTask = newTask;
                futureTask.run();
            } else {
                // 其他线程使用已存在的 task，等待结果
                futureTask = existing;
            }
        }

        return futureTask.get(); // 等待加载完成并返回结果
    }

    private static String loadFromDB(String key) {
        System.out.println("【加载】从数据库加载: " + key);
        try {
            Thread.sleep(1000); // 模拟耗时加载
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
        }
        return "DB_VALUE_" + key;
    }

    public static void main(String[] args) throws InterruptedException {
        // 模拟 10 个线程同时获取同一个 key
        CountDownLatch latch = new CountDownLatch(10);
        for (int i = 0; i < 10; i++) {
            new Thread(() -> {
                try {
                    String value = get("user:1001");
                    System.out.println(Thread.currentThread().getName() + " 获取到: " + value);
                } catch (Exception e) {
                    e.printStackTrace();
                } finally {
                    latch.countDown();
                }
            }).start();
        }
        latch.await();
        // 输出：只有一个【加载】，其他线程都直接返回缓存值
    }
}
```

**核心设计要点：**
- `ConcurrentHashMap.putIfAbsent()` 保证原子性，只有一个线程能成功放入 `FutureTask`
- 第一个线程调用 `futureTask.run()` 执行实际加载
- 其他线程获取到已有的 `FutureTask`，调用 `get()` 阻塞等待结果
- 加载完成后，后续所有请求直接返回缓存值，无需再次加载

## 8.2 CompletableFuture

### 8.2.1 创建CompletableFuture

```java
public class CompletableFutureCreation {
    private final ExecutorService executor = Executors.newFixedThreadPool(4);

    public void creationMethods() throws Exception {
        // 方式一：supplyAsync（有返回值）
        CompletableFuture<String> cf1 = CompletableFuture.supplyAsync(() -> {
            return "result"; // 异步执行，有返回值
        });

        // 方式二：runAsync（无返回值）
        CompletableFuture<Void> cf2 = CompletableFuture.runAsync(() -> {
            // 异步执行，无返回值
        });

        // 指定线程池
        CompletableFuture<String> cf3 = CompletableFuture.supplyAsync(() -> {
            return "custom executor";
        }, executor);
    }
}
```

### 8.2.2 常用API分类

**① 同步获取结果**

```java
// 阻塞获取
String result = cf.get();              // 无限等待
String result = cf.get(5, TimeUnit.SECONDS); // 超时获取
String result = cf.join();              // 同 get()，但不抛受检异常
String result = cf.getNow("默认值");    // 立即返回，已完成则返回值，否则返回默认值
```

**② thenApply（结果转换）**

```java
CompletableFuture<Integer> cf = CompletableFuture
    .supplyAsync(() -> "Hello")
    .thenApply(s -> s.length())        // "Hello" → 5
    .thenApply(len -> len * 2);         // 5 → 10
```

**③ thenAccept / thenRun（消费）**

```java
// thenAccept：消费上一步结果，无返回值
CompletableFuture.supplyAsync(() -> "World")
    .thenAccept(s -> System.out.println("收到: " + s));

// thenRun：不依赖上一步结果，仅执行一个动作
cf.thenRun(() -> System.out.println("任务完成"));
```

**④ 异常处理**

```java
CompletableFuture<Integer> cf = CompletableFuture
    .supplyAsync(() -> 100 / 0)
    .exceptionally(ex -> {              // 捕获异常并处理
        System.out.println("异常: " + ex.getMessage());
        return 0;                        // 返回默认值
    });

// 也可以用 handle（无论是否异常都会调用）
cf.handle((result, ex) -> {
    if (ex != null) {
        return -1;
    }
    return result;
});
```

### 8.2.3 多任务组合

```java
public class CompletableFutureCombination {

    // ① thenCombine：合并两个独立的 CompletableFuture
    public CompletableFuture<Integer> combineDemo() {
        CompletableFuture<Integer> cf1 = CompletableFuture
            .supplyAsync(() -> 10);
        CompletableFuture<Integer> cf2 = CompletableFuture
            .supplyAsync(() -> 20);

        return cf1.thenCombine(cf2, Integer::sum); // 合并结果：10 + 20 = 30
    }

    // ② allOf：等待所有任务完成
    public void allOfDemo() throws Exception {
        CompletableFuture.allOf(
            CompletableFuture.supplyAsync(() -> task1()),
            CompletableFuture.supplyAsync(() -> task2()),
            CompletableFuture.supplyAsync(() -> task3())
        ).get(); // 阻塞直到全部完成
        System.out.println("所有任务完成");
    }

    // ③ anyOf：任意一个任务完成即可
    public void anyOfDemo() throws Exception {
        Object result = CompletableFuture.anyOf(
            CompletableFuture.supplyAsync(() -> { Thread.sleep(500); return "A"; }),
            CompletableFuture.supplyAsync(() -> { Thread.sleep(100); return "B"; })
        ).get();
        System.out.println("最先完成的: " + result); // B（先完成）
    }

    // ④ thenCompose：扁平化异步链（避免 CompletableFuture<CompletableFuture<T>>）
    public CompletableFuture<String> composeDemo() {
        return CompletableFuture
            .supplyAsync(() -> 10)
            .thenCompose(price -> getDiscount(price)); // 返回 CompletableFuture
    }

    private CompletableFuture<String> getDiscount(int price) {
        return CompletableFuture.supplyAsync(() -> "折扣:" + price * 0.8);
    }
}
```

### 8.2.4 实战：聚合查询

```java
public class AggregationQuery {

    private final ExecutorService executor = Executors.newFixedThreadPool(10);

    public ProductVO query(Long productId) throws Exception {
        // 并行发起多个查询
        CompletableFuture<Product> productFuture = CompletableFuture
            .supplyAsync(() -> queryProduct(productId), executor);
        CompletableFuture<List<Comment>> commentFuture = CompletableFuture
            .supplyAsync(() -> queryComments(productId), executor);
        CompletableFuture<List<SKU>> skuFuture = CompletableFuture
            .supplyAsync(() -> querySKUs(productId), executor);

        // 等待所有结果
        Product product = productFuture.get();
        List<Comment> comments = commentFuture.get();
        List<SKU> skus = skuFuture.get();

        // 聚合返回
        return new ProductVO(product, comments, skus);
    }

    // 实际项目建议用 allOf + thenAccept 方式（无阻塞聚合）
    public CompletableFuture<ProductVO> queryAsync(Long productId) {
        CompletableFuture<Product> productFuture = CompletableFuture
            .supplyAsync(() -> queryProduct(productId), executor);
        CompletableFuture<List<Comment>> commentFuture = CompletableFuture
            .supplyAsync(() -> queryComments(productId), executor);
        CompletableFuture<List<SKU>> skuFuture = CompletableFuture
            .supplyAsync(() -> querySKUs(productId), executor);

        return CompletableFuture.allOf(productFuture, commentFuture, skuFuture)
            .thenApply(v -> new ProductVO(
                productFuture.join(),
                commentFuture.join(),
                skuFuture.join()
            ));
    }
}
```

### 8.2.5 同步回调与异步回调

`CompletableFuture` 的回调方法分为**同步版本**和**异步版本**，核心区别在于回调函数执行的线程：

| 同步方法 | 异步方法 | 执行线程 |
|:---|:---|:---|
| `thenApply(fn)` | `thenApplyAsync(fn)` | 同步：父任务线程；异步：ForkJoinPool 或指定线程池 |
| `thenAccept(consumer)` | `thenAcceptAsync(consumer)` | 同上 |
| `thenRun(action)` | `thenRunAsync(action)` | 同上 |
| `whenComplete(action)` | `whenCompleteAsync(action)` | 同上 |
| `handle(fn)` | `handleAsync(fn)` | 同上 |

```java
public class SyncVsAsync {

    private final ExecutorService executor = Executors.newFixedThreadPool(4);

    public void demo() {
        // 同步回调：回调与父任务在同一线程执行
        CompletableFuture.supplyAsync(() -> {
            System.out.println("父任务: " + Thread.currentThread().getName());
            return "Hello";
        }).thenApply(s -> {
            // 与父任务同线程（如果是supplyAsync的线程）
            System.out.println("同步回调: " + Thread.currentThread().getName());
            return s + " World";
        });

        // 异步回调：回调在 ForkJoinPool 中执行
        CompletableFuture.supplyAsync(() -> "Hello")
            .thenApplyAsync(s -> {
                System.out.println("异步回调(默认线程池): " + Thread.currentThread().getName());
                return s + " World";
            });

        // 异步回调：指定自定义线程池（生产环境推荐）
        CompletableFuture.supplyAsync(() -> "Hello")
            .thenApplyAsync(s -> {
                System.out.println("异步回调(自定义线程池): " + Thread.currentThread().getName());
                return s + " World";
            }, executor);
    }
}
```

> **生产环境建议**：始终使用 `thenXxxAsync(..., executor)` 指定自定义线程池，避免默认 `ForkJoinPool.commonPool()` 被占满导致其他 CompletableFuture 任务饥饿。

### 8.2.6 异常处理详解

`CompletableFuture` 提供了三种异常处理方式，各自有不同的返回值语义：

| 方法 | 触发时机 | 是否影响结果 | 返回值 |
|:---|:---|:---|:---|
| `exceptionally(fn)` | 上游发生异常时 | 用 fn 的返回值替代异常结果 | 有返回值（替代异常值） |
| `whenComplete(consumer)` | 无论是否异常都触发 | 不影响原结果（仅观察） | 无返回值（透传原结果或异常） |
| `handle(fn)` | 无论是否异常都触发 | 可修改结果 | 有返回值（可替换正常/异常结果） |

```java
public class CompletableFutureException {

    public void demo() {
        // ① exceptionally：捕获异常并返回默认值
        CompletableFuture<Integer> cf1 = CompletableFuture
            .supplyAsync(() -> 100 / 0)  // 抛 ArithmeticException
            .exceptionally(ex -> {
                System.out.println("异常被捕获: " + ex.getMessage());
                return -1; // 返回默认值替代异常
            });
        System.out.println(cf1.join()); // -1

        // ② whenComplete：观察结果，不影响原结果
        CompletableFuture<Integer> cf2 = CompletableFuture
            .supplyAsync(() -> 100 / 0)
            .whenComplete((result, ex) -> {
                if (ex != null) {
                    System.out.println("whenComplete 发现异常: " + ex.getMessage());
                }
                // whenComplete 的返回值被忽略，原异常继续传播
            });
        // cf2.join() 仍然会抛出 CompletionException

        // ③ handle：统一处理正常和异常结果
        CompletableFuture<Integer> cf3 = CompletableFuture
            .supplyAsync(() -> 100 / 0)
            .handle((result, ex) -> {
                if (ex != null) {
                    System.out.println("handle 处理异常: " + ex.getMessage());
                    return -1;
                }
                return result * 2;
            });
        System.out.println(cf3.join()); // -1

        // 正常场景下 handle 也生效
        CompletableFuture<Integer> cf4 = CompletableFuture
            .supplyAsync(() -> 50)
            .handle((result, ex) -> ex != null ? -1 : result * 2);
        System.out.println(cf4.join()); // 100
    }
}
```

> **最佳实践**：
> - 仅需异常降级 → 用 `exceptionally()`
> - 需要日志记录但保留原结果/异常 → 用 `whenComplete()`
> - 需要统一处理正常和异常分支 → 用 `handle()`

## 8.3 CompletionService

### 8.3.1 核心原理

`CompletionService` 将**任务提交**与**结果消费**解耦，按任务**完成顺序**获取结果（而非提交顺序）。其典型实现 `ExecutorCompletionService` 内部为每个任务维护一个 `BlockingQueue`，先完成的任务先入队。

```java
public class CompletionServiceDemo {

    public static void main(String[] args) throws Exception {
        ExecutorService executor = Executors.newFixedThreadPool(3);
        CompletionService<String> cs = new ExecutorCompletionService<>(executor);

        // 提交多个耗时不同的任务（提交顺序：慢 → 快 → 中）
        cs.submit(() -> { Thread.sleep(3000); return "慢任务（3s）"; });
        cs.submit(() -> { Thread.sleep(1000); return "快任务（1s）"; });
        cs.submit(() -> { Thread.sleep(2000); return "中任务（2s）"; });

        // 按完成顺序获取结果（获取顺序：快 → 中 → 慢）
        for (int i = 0; i < 3; i++) {
            Future<String> future = cs.take(); // 阻塞等待下一个完成的任务
            System.out.println("获取: " + future.get());
        }

        executor.shutdown();
    }
}
```

### 8.3.2 与Future批量获取对比

| 场景 | 普通 Future 列表 | CompletionService |
|:---|:---|:---|
| **获取顺序** | 按提交顺序（先提交先获取，即使后提交的先完成也要等待） | 按完成顺序（谁先完成谁先获取） |
| **适用场景** | 任务之间有依赖，必须按顺序处理结果 | 任务相互独立，希望尽快处理已完成的结果 |
| **阻塞风险** | 前一个任务未完成，无法获取后一个已完成任务的结果 | 无此风险，始终获取最早完成的任务 |

```java
// ❌ 普通 Future：按提交顺序获取，慢任务阻塞了快任务结果的处理
List<Future<String>> futures = new ArrayList<>();
futures.add(executor.submit(slowTask)); // 3s
futures.add(executor.submit(fastTask)); // 1s
for (Future<String> f : futures) {
    System.out.println(f.get()); // 先等3s的慢任务，虽然1s的快任务早已完成
}

// ✅ CompletionService：按完成顺序获取
CompletionService<String> cs = new ExecutorCompletionService<>(executor);
cs.submit(slowTask);
cs.submit(fastTask);
for (int i = 0; i < 2; i++) {
    System.out.println(cs.take().get()); // 1s 后先拿到快任务结果
}
```
