## 5.1 Executor 框架体系

Executor 框架是 JUC 中线程池的顶层抽象，整个体系由三大组件构成：

```
┌─────────────────────────────────────────────────────────────┐
│                     Executor 框架体系                         │
├─────────────────────────────────────────────────────────────┤
│  任务层：Runnable（无返回值） / Callable<T>（有返回值）         │
├─────────────────────────────────────────────────────────────┤
│  执行层：Executor → ExecutorService → ThreadPoolExecutor      │
│                         ↓                                     │
│            ScheduledExecutorService → ScheduledThreadPoolExecutor│
│                         ↓                                     │
│                   ForkJoinPool / Executors 工具类              │
├─────────────────────────────────────────────────────────────┤
│  结果层：Future<T> / FutureTask / CompletableFuture<T>        │
└─────────────────────────────────────────────────────────────┘
```

| 组件 | 说明 |
|:---|:---|
| **任务** | `Runnable`（无返回值） / `Callable<T>`（有返回值，可抛异常） |
| **执行器** | `Executor` → `ExecutorService` → `ThreadPoolExecutor` / `ScheduledThreadPoolExecutor` |
| **结果** | `Future<T>`（异步结果句柄）、`CompletableFuture<T>`（链式异步编排） |

```java
public class ExecutorFrameworkDemo {
    public static void main(String[] args) throws Exception {
        // 任务定义
        Callable<String> task = () -> {
            TimeUnit.SECONDS.sleep(1);
            return "Task Result";
        };

        // 执行器创建
        ExecutorService executor = Executors.newFixedThreadPool(2);

        // 提交任务并获取 Future
        Future<String> future = executor.submit(task);

        // 非阻塞检查
        while (!future.isDone()) {
            System.out.println("任务执行中...");
            Thread.sleep(200);
        }

        // 阻塞获取结果（带超时）
        String result = future.get(5, TimeUnit.SECONDS);
        System.out.println("结果: " + result);

        // 优雅关闭
        executor.shutdown();
        if (!executor.awaitTermination(10, TimeUnit.SECONDS)) {
            executor.shutdownNow(); // 强制关闭
        }
    }
}
```

## 5.2 ThreadPoolExecutor

### 5.2.1 七大核心参数

```java
public ThreadPoolExecutor(
    int corePoolSize,              // 核心线程数
    int maximumPoolSize,            // 最大线程数
    long keepAliveTime,            // 空闲线程存活时间
    TimeUnit unit,                 // 时间单位
    BlockingQueue<Runnable> workQueue,   // 任务队列
    ThreadFactory threadFactory,   // 线程工厂
    RejectedExecutionHandler handler     // 拒绝策略
)
```

### 5.2.2 执行流程

```
                ┌──────────────────────────────┐
                │  接收新任务                    │
                └──────────────┬───────────────┘
                               ▼
              ┌────────────────────────────────┐
              │  核心线程数是否已满？             │
              └──────────────┬───────────────────┘
                    是               否
                    ▼                ▼
         ┌────────────────┐   ┌─────────────────┐
         │  队列是否已满？   │   │  直接交给核心线程  │
         └────────┬────────┘   └─────────────────┘
           是          否
           ▼            ▼
  ┌────────────────┐  ┌────────────────┐
  │ 最大线程数已满？ │  │  进入队列等待    │
  └────────┬───────┘  └─────────────────┘
    是         否
    ▼           ▼
┌──────────────────┐  ┌────────────────┐
│ 执行拒绝策略      │  │ 创建临时线程执行 │
└──────────────────┘  └────────────────┘
```

**三道门槛**：核心线程数 → 任务队列 → 最大线程数

### 5.2.3 拒绝策略

| 策略 | 说明 |
|:---|:---|
| `AbortPolicy`（默认） | 抛出 `RejectedExecutionException` |
| `CallerRunsPolicy` | 由调用线程执行被拒绝的任务 |
| `DiscardPolicy` | 直接丢弃被拒绝的任务（无任何操作） |
| `DiscardOldestPolicy` | 丢弃队列中最老的任务，再尝试执行新任务 |

```java
public class ThreadPoolConfig {

    // 典型配置示例
    public ExecutorService createThreadPool() {
        return new ThreadPoolExecutor(
            8,                          // 核心线程数
            16,                        // 最大线程数
            60L, TimeUnit.SECONDS,     // 空闲线程存活时间
            new LinkedBlockingQueue<>(1000), // 队列容量
            Executors.defaultThreadFactory(), // 线程工厂
            // 自定义拒绝策略（记录日志 + 降级处理）
            (r, executor) -> {
                log.warn("线程池满了，拒绝任务: {}", r);
                // 降级：调用者线程执行
                if (!executor.isShutdown()) {
                    r.run();
                }
            }
        );
    }
}
```

### 5.2.4 动态调整参数

```java
public class DynamicThreadPool {
    private final ThreadPoolExecutor executor = new ThreadPoolExecutor(
        4, 20, 60L, TimeUnit.SECONDS,
        new LinkedBlockingQueue<>(500)
    );

    // 运行时动态调整
    public void adjustPool() {
        executor.setCorePoolSize(8);   // 动态调整核心线程数
        executor.setMaximumPoolSize(32); // 动态调整最大线程数
        executor.setKeepAliveTime(30L, TimeUnit.SECONDS);
    }
}
```

### 5.2.5 监控与状态流转

**线程池状态流转**：

```
RUNNING ──shutdown()──> SHUTDOWN ──队列为空且线程全停──> TIDYING ──terminated()──> TERMINATED
    │
    └──shutdownNow()──> STOP ──线程全停──> TIDYING ──terminated()──> TERMINATED
```

| 状态 | 说明 |
|:---|:---|
| **RUNNING** | 接收新任务，处理队列中的任务 |
| **SHUTDOWN** | 不接收新任务，但处理队列中剩余任务 |
| **STOP** | 不接收新任务，不处理队列任务，中断正在执行的任务 |
| **TIDYING** | 所有任务已终止，workerCount = 0，准备执行 `terminated()` 钩子 |
| **TERMINATED** | `terminated()` 执行完毕 |

```java
public class ThreadPoolMonitor {

    public static void printPoolStatus(ThreadPoolExecutor pool) {
        System.out.printf(
            "[线程池状态] 核心:%d 活跃:%d 最大:%d 队列任务:%d 已完成:%d 总任务:%d%n",
            pool.getCorePoolSize(),      // 核心线程数
            pool.getActiveCount(),       // 活跃线程数
            pool.getMaximumPoolSize(),   // 最大线程数
            pool.getQueue().size(),      // 队列中等待的任务数
            pool.getCompletedTaskCount(), // 已完成任务总数
            pool.getTaskCount()          // 已提交任务总数（含正在执行）
        );
    }

    public static void main(String[] args) {
        ThreadPoolExecutor pool = new ThreadPoolExecutor(
            2, 4, 60, TimeUnit.SECONDS,
            new LinkedBlockingQueue<>(10)
        );

        for (int i = 0; i < 5; i++) {
            pool.execute(() -> {
                try { Thread.sleep(1000); } catch (InterruptedException e) { Thread.currentThread().interrupt(); }
            });
        }

        printPoolStatus(pool);

        // 状态检查
        System.out.println("isShutdown: " + pool.isShutdown());   // false
        System.out.println("isTerminated: " + pool.isTerminated()); // false

        pool.shutdown();
        System.out.println("调用shutdown后 isShutdown: " + pool.isShutdown()); // true
    }
}
```

> **生产建议**：将线程池监控指标接入 Prometheus/Grafana，实时观察活跃线程数、队列堆积量、任务拒绝率，及时发现线程池瓶颈。

### 5.2.6 execute与submit区别

| 维度 | `execute(Runnable)` | `submit(Runnable/Callable)` |
|:---|:---|:---|
| **返回值** | `void` | `Future<T>` |
| **任务类型** | 仅 `Runnable` | `Runnable` 或 `Callable` |
| **异常处理** | 异常被 `UncaughtExceptionHandler` 捕获 | 异常被封装到 `Future`，调用 `get()` 时抛出 |
| **适用场景** | 无需返回值的异步任务 | 需要返回值或需要异常追踪的任务 |

```java
public class ExecuteVsSubmit {

    private final ExecutorService executor = Executors.newFixedThreadPool(2);

    public void demo() throws Exception {
        // execute：无返回值，异常直接抛出到 UncaughtExceptionHandler
        executor.execute(() -> {
            System.out.println("execute 执行任务");
            throw new RuntimeException("execute 异常"); // 被 UncaughtExceptionHandler 捕获
        });

        // submit：返回 Future，异常被封装
        Future<Integer> future = executor.submit(() -> {
            System.out.println("submit 执行任务");
            return 42;
        });
        Integer result = future.get(); // 正常返回 42

        // submit 的异常在 get() 时才抛出
        Future<?> errorFuture = executor.submit(() -> {
            throw new RuntimeException("submit 异常");
        });
        try {
            errorFuture.get(); // ExecutionException 包装了原始异常
        } catch (ExecutionException e) {
            System.out.println("捕获到异常: " + e.getCause().getMessage());
        }
    }
}
```

### 5.2.7 线程工厂定制

生产环境中强烈建议**自定义线程工厂**，为线程命名、设置守护状态、配置异常处理器等，便于问题排查。

```java
public class NamedThreadFactory implements ThreadFactory {

    private final AtomicInteger threadNumber = new AtomicInteger(1);
    private final String namePrefix;
    private final boolean daemon;
    private final Thread.UncaughtExceptionHandler handler;

    public NamedThreadFactory(String namePrefix, boolean daemon) {
        this(namePrefix, daemon, null);
    }

    public NamedThreadFactory(String namePrefix, boolean daemon,
                              Thread.UncaughtExceptionHandler handler) {
        this.namePrefix = namePrefix;
        this.daemon = daemon;
        this.handler = handler;
    }

    @Override
    public Thread newThread(Runnable r) {
        Thread t = new Thread(r, namePrefix + "-" + threadNumber.getAndIncrement());
        t.setDaemon(daemon);
        if (handler != null) {
            t.setUncaughtExceptionHandler(handler);
        }
        return t;
    }

    // 使用示例
    public static void main(String[] args) {
        ThreadPoolExecutor executor = new ThreadPoolExecutor(
            4, 8, 60, TimeUnit.SECONDS,
            new LinkedBlockingQueue<>(100),
            new NamedThreadFactory("order-pool", false, (t, e) -> {
                System.err.println("线程 [" + t.getName() + "] 异常: " + e.getMessage());
            }),
            new ThreadPoolExecutor.CallerRunsPolicy()
        );
    }
}
```

> **线程命名的好处**：线程dump（`jstack`）时可以直接看到线程归属，快速定位问题。

## 5.3 Executors 工具类

### 5.3.1 四种常见线程池

| 方法 | 类型 | 说明 | 适用场景 |
|:---|:---|:---|:---|
| `newFixedThreadPool(int n)` | 固定大小 | 核心=最大，无超时，队列无界 | 任务量平稳，长期运行 |
| `newCachedThreadPool()` | 缓存线程池 | 核心0，最大无界，60s超时 | 短时异步任务，任务量波动大 |
| `newSingleThreadExecutor()` | 单线程 | 单一线程，保证顺序执行 | 任务顺序有要求 |
| `newScheduledThreadPool(int core)` | 定时线程池 | 支持定时/周期性任务 | 定时任务调度 |
| `newWorkStealingPool()` | 工作窃取池 | ForkJoinPool 实现，任务分治 | 递归分解的并行计算 |

```java
public class ExecutorsDemo {

    // 固定大小线程池
    public ExecutorService fixedPool() {
        return Executors.newFixedThreadPool(10);
    }

    // 缓存线程池（注意：无界，可能无限创建线程）
    public ExecutorService cachedPool() {
        return Executors.newCachedThreadPool();
    }

    // 单线程线程池
    public ExecutorService singlePool() {
        return Executors.newSingleThreadExecutor();
    }

    // 定时线程池
    public ScheduledExecutorService scheduledPool() {
        return Executors.newScheduledThreadPool(4);
    }

    // 工作窃取线程池（JDK 8，底层 ForkJoinPool）
    public ExecutorService workStealingPool() {
        return Executors.newWorkStealingPool();
    }
}
```

### 5.3.2 生产环境建议

> **阿里巴巴 Java 规范禁止使用 Executors 创建线程池**，必须使用 `new ThreadPoolExecutor` 手动指定参数，防止无界队列导致 OOM。

```java
// ✅ 推荐：明确参数，有界队列
new ThreadPoolExecutor(
    corePoolSize, maxPoolSize,
    keepAliveTime, TimeUnit.SECONDS,
    new LinkedBlockingQueue<>(capacity), // 有界队列，防止OOM
    threadFactory,
    handler
);

// ❌ 避免：无界队列，可能OOM
Executors.newFixedThreadPool(10); // LinkedBlockingQueue 默认无界
Executors.newCachedThreadPool(); // 最大线程数无界
```

## 5.4 ScheduledThreadPoolExecutor

### 5.4.1 定时任务线程池

`ScheduledThreadPoolExecutor` 继承自 `ThreadPoolExecutor`，专门用于执行**定时任务**和**周期性任务**。

| 方法 | 说明 |
|:---|:---|
| `schedule(Runnable, delay, unit)` | 延迟执行一次 |
| `schedule(Callable, delay, unit)` | 延迟执行一次，带返回值 |
| `scheduleAtFixedRate(Runnable, initialDelay, period, unit)` | 固定频率执行（任务开始时间间隔固定） |
| `scheduleWithFixedDelay(Runnable, initialDelay, delay, unit)` | 固定延迟执行（任务结束到下次开始间隔固定） |

```java
public class ScheduledThreadPoolDemo {

    private final ScheduledExecutorService scheduler =
        new ScheduledThreadPoolExecutor(2,
            new NamedThreadFactory("scheduler", false, null),
            new ThreadPoolExecutor.DiscardPolicy()
        );

    public void demo() {
        // 延迟 3 秒后执行一次
        scheduler.schedule(() -> {
            System.out.println("3秒后执行");
        }, 3, TimeUnit.SECONDS);

        // 固定频率：每 5 秒执行一次（无论上次是否完成）
        scheduler.scheduleAtFixedRate(() -> {
            System.out.println("固定频率执行: " + System.currentTimeMillis());
        }, 0, 5, TimeUnit.SECONDS);

        // 固定延迟：上次完成后延迟 5 秒再执行
        scheduler.scheduleWithFixedDelay(() -> {
            System.out.println("固定延迟执行: " + System.currentTimeMillis());
            try {
                Thread.sleep(1000); // 模拟任务耗时
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            }
        }, 0, 5, TimeUnit.SECONDS);
    }
}
```

### 5.4.2 固定频率与固定延迟

```
scheduleAtFixedRate（固定频率）:
任务A: |----2s----|
任务B:           |----2s----|
任务C:                     |----2s----|
间隔:      2s         2s
（如果任务执行时间 > period，会连续执行，不会重叠）

scheduleWithFixedDelay（固定延迟）:
任务A: |----2s----|
等待:              |-3s-|
任务B:                  |----2s----|
等待:                                |-3s-|
任务C:                                     |----2s----|
间隔:              3s                3s
（上次结束 → 等待3s → 下次开始）
```

### 5.4.3 生产环境注意事项

> **致命陷阱**：`ScheduledThreadPoolExecutor` 中的任务如果抛出**未捕获异常**，后续定时任务会**停止执行**！必须做好异常处理。

```java
public class SafeScheduledTask implements Runnable {
    @Override
    public void run() {
        try {
            // 业务逻辑
        } catch (Exception e) {
            // 必须捕获所有异常，否则后续任务不再执行
            System.err.println("定时任务异常: " + e.getMessage());
        }
    }
}

// 或者用装饰器包装
public static Runnable safeTask(Runnable task) {
    return () -> {
        try {
            task.run();
        } catch (Exception e) {
            System.err.println("任务异常被捕获: " + e.getMessage());
        }
    };
}
```

## 5.5 ForkJoinPool

### 5.5.1 核心思想

ForkJoinPool 适合**分治思想**的任务：将大任务拆分为小任务并行执行，最后合并结果。使用了**工作窃取（Work-Stealing）**机制。

```java
public class ForkJoinDemo {

    // 继承 RecursiveTask（有返回值）或 RecursiveAction（无返回值）
    static class SumTask extends RecursiveTask<Long> {
        private static final int THRESHOLD = 1000;
        private long[] array;
        private int start, end;

        public SumTask(long[] array, int start, int end) {
            this.array = array;
            this.start = start;
            this.end = end;
        }

        @Override
        protected Long compute() {
            int length = end - start;
            if (length <= THRESHOLD) {
                // 达到阈值，直接计算
                long sum = 0;
                for (int i = start; i < end; i++) {
                    sum += array[i];
                }
                return sum;
            }

            // 拆分任务
            int mid = (start + end) / 2;
            SumTask left = new SumTask(array, start, mid);
            SumTask right = new SumTask(array, mid, end);

            // fork 异步执行
            left.fork();
            right.fork();

            // join 合并结果
            return left.join() + right.join();
        }
    }

    public static void main(String[] args) throws Exception {
        long[] array = new long[100000];
        Arrays.fill(array, 1L);

        ForkJoinPool pool = new ForkJoinPool();
        long result = pool.invoke(new SumTask(array, 0, array.length));
        System.out.println("Sum = " + result);

        pool.shutdown();
    }
}
```

### 5.5.2 与ThreadPoolExecutor对比

| 维度 | ForkJoinPool | ThreadPoolExecutor |
|:---|:---|:---|
| 任务类型 | 分治递归任务 | 独立任务 |
| 工作窃取 | ✅ 支持，空闲线程窃取其他队列任务 | ❌ 不支持 |
| 适合场景 | CPU 密集型（计算任务） | IO 密集型（异步任务） |
| 推荐线程数 | `Runtime.getRuntime().availableProcessors()` | 视场景而定 |
