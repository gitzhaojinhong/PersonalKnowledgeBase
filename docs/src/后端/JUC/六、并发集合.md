## 6.1 ConcurrentHashMap

### 6.1.1 JDK7与JDK8实现对比

| 版本        | 实现方式                      | 说明                     |
|:----------|:--------------------------|:-----------------------|
| **JDK 7** | Segment 分段锁               | 将桶数组分为多段，每段独立加锁，锁粒度粗   |
| **JDK 8** | Node + CAS + synchronized | 锁粒度细化到链表/红黑树首节点，大幅减少竞争 |

JDK 8 数据结构：`数组 + 链表/红黑树`（链表长度 ≥ 8 且数组长度 ≥ 64 时转红黑树）

### 6.1.2 常用方法

```java
public class ConcurrentHashMapDemo {
    private ConcurrentHashMap<String, Integer> map = new ConcurrentHashMap<>();

    public void demo() {
        // 常用操作
        map.put("A", 1);                    // 插入
        map.get("A");                       // 读取
        map.remove("A");                   // 删除
        map.size();                         // 大小
        map.containsKey("A");              // 是否包含

        // 原子操作（JUC特有）
        map.putIfAbsent("B", 2);            // key不存在才插入
        map.replace("A", 1, 10);           // CAS替换（key=A && value=1 → value=10）
        map.compute("A", (k, v) -> v == null ? 1 : v + 1); // 计算（原子）
        map.merge("A", 5, (old, newVal) -> old + newVal);  // 合并

        // 批量操作
        map.putAll(otherMap);              // 批量插入
        map.forEach((k, v) -> System.out.println(k + ":" + v)); // 遍历

        // search / reduce
        map.search(1, (k, v) -> v > 100 ? k : null); // 搜索第一个满足条件的键
        map.reduceEntries(1, (e1, e2) ->
            e1.getValue() > e2.getValue() ? e1 : e2);  // 规约
    }
}
```

### 6.1.3 并行遍历与聚合

`ConcurrentHashMap` 支持**并行度控制的批量操作**，在多核 CPU 上可显著提升大数据量处理效率。

```java
public class ConcurrentHashMapParallel {

    private ConcurrentHashMap<String, Integer> map = new ConcurrentHashMap<>();

    public void parallelOps() {
        // 填充数据
        for (int i = 0; i < 1000; i++) {
            map.put("key" + i, i);
        }

        // forEach：指定并行度阈值（遍历元素数 > 阈值时并行执行）
        map.forEach(2, (k, v) -> System.out.println(k + "=" + v)); // 并行度=2

        // search：并行搜索第一个满足条件的值
        String result = map.search(2, (k, v) -> v > 500 ? k : null);
        System.out.println("第一个值>500的key: " + result);

        // reduce：并行聚合
        // reduceValues：对所有 value 进行归约
        int sum = map.reduceValues(2, Integer::sum);
        System.out.println("所有 value 之和: " + sum);

        // reduceEntries：对 entry 进行归约
        Map.Entry<String, Integer> maxEntry = map.reduceEntries(2,
            (e1, e2) -> e1.getValue() > e2.getValue() ? e1 : e2);
        System.out.println("最大 value 的 entry: " + maxEntry);

        // reduceToLong / reduceToDouble：基本类型归约（避免装箱）
        long longSum = map.reduceValuesToLong(2, v -> v, 0, Long::sum);
    }
}
```

| 方法                                             | 说明         | 参数                         |
|:-----------------------------------------------|:-----------|:---------------------------|
| `forEach(parallelismThreshold, action)`        | 并行遍历       | parallelismThreshold：并行度阈值 |
| `search(parallelismThreshold, function)`       | 并行搜索       | 返回第一个非 null 结果             |
| `reduceValues(parallelismThreshold, reducer)`  | 并行归约 value | 需传入二元归约函数                  |
| `reduceEntries(parallelismThreshold, reducer)` | 并行归约 entry | 同上                         |
| `reduceValuesToLong/ToDouble`                  | 基本类型归约     | 避免 Integer/Long 装箱开销       |

> **注意**：`parallelismThreshold` 是触发并行的**元素数量阈值**，不是线程数。通常设为 `map.size() / (可用CPU核心数 * 2)`
> 较为合理。数据量较小时并行反而因线程切换开销降低性能。

### 6.1.4 使用注意事项

- **不允许 null 键和 null 值**（与普通 HashMap 不同）
- `size()` 在高并发下可能不精确（JDK 8+ 使用 `baseCount` + `counterCells` 分片计数，非实时精确值）
- 迭代器是**弱一致性**（不会抛出 ConcurrentModificationException）
- 复合操作（检查后操作）需要额外加锁，如 `if (!map.containsKey(key)) map.put(key, value)` 应改用 `putIfAbsent()`

```java
// ❌ 不安全的复合操作
if (!map.containsKey("key")) {
    map.put("key", value); // 可能被其他线程抢先插入
}

// ✅ 原子操作
map.putIfAbsent("key", value);
```

## 6.2 CopyOnWriteArrayList

### 6.2.1 写时复制机制

写操作时**复制整个底层数组**，在副本上修改，然后原子替换引用。读操作完全无锁。

```java
public class CopyOnWriteArrayListDemo {
    private CopyOnWriteArrayList<String> list = new CopyOnWriteArrayList<>();

    public void demo() {
        // 读操作：完全无锁，并发性能极高
        String first = list.get(0);
        boolean hasA = list.contains("A");
        int size = list.size();

        // 写操作：加锁，复制数组，写入后替换
        list.add("item");       // 尾部追加
        list.set(0, "new");      // 修改指定位置
        list.remove("item");     // 移除元素
        list.addIfAbsent("X");   // 不存在才添加（原子）
    }
}
```

### 6.2.2 适用与不适用场景

| ✅ 适用                  | ❌ 不适用        |
|:----------------------|:-------------|
| 读操作远多于写操作（读:写 ≥ 10:1） | 写操作频繁        |
| 数据量较小（复制成本低）          | 数据量极大（内存开销大） |
| 对实时一致性要求不高（最终一致性）     | 需要强实时一致性     |
| 事件监听列表、配置项列表          | 高频修改的缓存数据    |

## 6.3 CopyOnWriteArraySet

`CopyOnWriteArraySet` 是基于 `CopyOnWriteArrayList` 实现的线程安全 Set，内部通过 `addIfAbsent()` 保证元素唯一性。

```java
public class CopyOnWriteArraySetDemo {
    private CopyOnWriteArraySet<String> set = new CopyOnWriteArraySet<>();

    public void demo() {
        set.add("A");       // 添加（复制数组，O(n)去重检查）
        set.add("A");       // 重复，不会添加
        set.contains("A");  // 读操作，无锁
        set.remove("A");    // 删除（复制数组）
        set.size();
    }
}
```

> **注意**：`CopyOnWriteArraySet` 的 `add()` 操作是 O(n)，因为需要遍历数组检查元素是否已存在。如果写操作频繁且数据量大，建议用
`ConcurrentHashMap.newKeySet()` 替代。

| 特性       | CopyOnWriteArraySet | ConcurrentHashMap.newKeySet() |
|:---------|:--------------------|:------------------------------|
| **读性能**  | 极高（无锁）              | 高（分段锁/CAS）                    |
| **写性能**  | 差（复制整个数组 + O(n)去重）  | 好（O(1)）                       |
| **内存开销** | 大（写时复制）             | 小                             |
| **适用场景** | 读多写少、元素少            | 读写均衡、元素多                      |

## 6.4 阻塞队列 BlockingQueue

### 6.4.1 四组API

| 操作     | 抛出异常            | 返回特殊值                   | 阻塞等待     | 超时退出                   |
|:-------|:----------------|:------------------------|:---------|:-----------------------|
| **插入** | `add(e)` → true | `offer(e)` → true/false | `put(e)` | `offer(e, time, unit)` |
| **移除** | `remove()`      | `poll()` → null         | `take()` | `poll(time, unit)`     |
| **检查** | `element()`     | `peek()` → null         | —        | —                      |

### 6.4.2 常见实现类

| 实现类                     | 底层结构 | 有界性                       | 特点                          |
|:------------------------|:-----|:--------------------------|:----------------------------|
| `ArrayBlockingQueue`    | 数组   | ✅ 有界                      | 预分配内存，固定容量                  |
| `LinkedBlockingQueue`   | 链表   | 默认无界（`Integer.MAX_VALUE`） | 吞吐量高，注意指定容量                 |
| `PriorityBlockingQueue` | 堆    | ❌ 无界                      | 支持优先级排序，自动扩容                |
| `DelayQueue`            | 优先队列 | 无界                        | 元素需实现 `Delayed` 接口，支持延迟获取   |
| `SynchronousQueue`      | —    | 不存储元素                     | 每个 put 必须等待一个 take，适合线程直接交接 |

```java
public class BlockingQueueDemo {
    // 有界队列 —— 适合固定容量场景（如并发任务队列）
    private BlockingQueue<Task> arrayQueue = new ArrayBlockingQueue<>(100);

    // 建议指定容量的链表队列 —— 避免无界导致OOM
    private BlockingQueue<Task> linkedQueue = new LinkedBlockingQueue<>(1000);

    // 优先级队列 —— 任务调度
    private BlockingQueue<Task> priorityQueue = new PriorityBlockingQueue<>(
        11, Comparator.comparingInt(Task::getPriority));

    // 同步队列 —— 直接交接，不存储（适合线程池的SynchronousQueue）
    private BlockingQueue<Object> syncQueue = new SynchronousQueue<>();

    // 生产者-消费者模式示例
    public void producerConsumer() throws InterruptedException {
        BlockingQueue<Integer> queue = new ArrayBlockingQueue<>(5);

        Thread producer = new Thread(() -> {
            for (int i = 0; i < 10; i++) {
                try {
                    queue.put(i); // 队列满则阻塞
                    System.out.println("生产: " + i);
                } catch (InterruptedException e) {
                    Thread.currentThread().interrupt();
                }
            }
        });

        Thread consumer = new Thread(() -> {
            while (true) {
                try {
                    Integer value = queue.take();
                    System.out.println("消费: " + value);
                } catch (InterruptedException e) {
                    Thread.currentThread().interrupt();
                    break;
                }
            }
        });


        producer.start();
        consumer.start();
        producer.join();
    }
}
```

### 6.4.3 DelayQueue延时队列

`DelayQueue` 是无界阻塞队列，只有**延迟时间到期的元素**才能被取出。元素必须实现 `Delayed` 接口。

```java
public class DelayQueueDemo {

    // 延时任务元素
    static class DelayedTask implements Delayed {
        private final String name;
        private final long executeTime;

        public DelayedTask(String name, long delayMs) {
            this.name = name;
            this.executeTime = System.currentTimeMillis() + delayMs;
        }

        @Override
        public long getDelay(TimeUnit unit) {
            long diff = executeTime - System.currentTimeMillis();
            return unit.convert(diff, TimeUnit.MILLISECONDS);
        }

        @Override
        public int compareTo(Delayed o) {
            return Long.compare(this.executeTime, ((DelayedTask) o).executeTime);
        }

        @Override
        public String toString() {
            return name + " [" + executeTime + "]";
        }
    }

    public static void main(String[] args) throws InterruptedException {
        DelayQueue<DelayedTask> queue = new DelayQueue<>();

        queue.put(new DelayedTask("任务A", 3000)); // 3秒后到期
        queue.put(new DelayedTask("任务B", 1000)); // 1秒后到期
        queue.put(new DelayedTask("任务C", 5000)); // 5秒后到期

        while (!queue.isEmpty()) {
            DelayedTask task = queue.take(); // 阻塞等待到期元素
            System.out.println("执行: " + task);
        }
        // 输出顺序: B(1s) → A(3s) → C(5s)
    }
}
```

**典型场景**：订单超时自动取消、缓存过期清理、定时任务调度。

### 6.4.4 SynchronousQueue同步队列

`SynchronousQueue` **不存储任何元素**，每个 `put()` 必须等待一个 `take()`，反之亦然。它是一种**直接交接**机制。

```java
public class SynchronousQueueDemo {

    public static void main(String[] args) {
        SynchronousQueue<String> queue = new SynchronousQueue<>();

        Thread producer = new Thread(() -> {
            try {
                System.out.println("生产者准备 put...");
                queue.put("数据"); // 阻塞，直到有消费者 take
                System.out.println("生产者 put 完成");
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            }
        });

        Thread consumer = new Thread(() -> {
            try {
                Thread.sleep(1000);
                System.out.println("消费者准备 take...");
                String data = queue.take(); // 阻塞，直到有生产者 put
                System.out.println("消费者收到: " + data);
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            }
        });

        producer.start();
        consumer.start();
    }
}
```

> `Executors.newCachedThreadPool()` 内部使用 `SynchronousQueue`，实现"来一个任务就创建一个线程"的效果。

### 6.4.5 BlockingDeque双端阻塞队列

`BlockingDeque` 支持在**队列两端**进行阻塞插入和移除操作。

| 操作     | 首端                              | 尾端                            |
|:-------|:--------------------------------|:------------------------------|
| **插入** | `addFirst(e)` / `putFirst(e)`   | `addLast(e)` / `putLast(e)`   |
| **移除** | `removeFirst()` / `takeFirst()` | `removeLast()` / `takeLast()` |
| **检查** | `peekFirst()`                   | `peekLast()`                  |

```java
public class BlockingDequeDemo {
    private BlockingDeque<String> deque = new LinkedBlockingDeque<>(10);

    public void demo() throws InterruptedException {
        deque.putFirst("A");   // 首端插入
        deque.putLast("B");    // 尾端插入
        String first = deque.takeFirst(); // 首端移除
        String last = deque.takeLast();   // 尾端移除
    }
}
```

**典型场景**：工作窃取算法（Stealing Deque），线程从一端取任务，被窃取时从另一端取。

### 6.4.6 TransferQueue传输队列

`TransferQueue` 扩展了 `BlockingQueue`，增加了 `transfer()` 方法：生产者必须等待消费者直接接收元素后才返回。

```java
public class TransferQueueDemo {

    public static void main(String[] args) {
        TransferQueue<String> queue = new LinkedTransferQueue<>();

        Thread producer = new Thread(() -> {
            try {
                System.out.println("生产者 transfer 数据...");
                queue.transfer("重要数据"); // 阻塞直到消费者接收
                System.out.println("生产者确认数据已被接收");
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            }
        });

        Thread consumer = new Thread(() -> {
            try {
                Thread.sleep(1000);
                String data = queue.take();
                System.out.println("消费者收到: " + data);
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            }
        });

        producer.start();
        consumer.start();
    }
}
```

| 方法                              | 行为            |
|:--------------------------------|:--------------|
| `transfer(e)`                   | 阻塞，直到元素被消费者接收 |
| `tryTransfer(e)`                | 非阻塞，立即返回是否成功  |
| `tryTransfer(e, timeout, unit)` | 超时等待消费者接收     |
| `hasWaitingConsumer()`          | 是否有等待的消费者     |
| `getWaitingConsumerCount()`     | 等待的消费者数量      |

## 6.5 ConcurrentLinkedQueue

无界非阻塞队列，基于 **CAS 无锁算法**实现，高并发环境下性能优秀。

```java
public class ConcurrentLinkedQueueDemo {
    private ConcurrentLinkedQueue<String> queue = new ConcurrentLinkedQueue<>();

    public void demo() {
        queue.offer("A");          // 入队
        queue.add("B");             // 入队（add内部调用offer）
        String head = queue.poll(); // 出队（队列空返回null）
        String peek = queue.peek(); // 查看队首（不出队）
        boolean empty = queue.isEmpty();
        // ⚠️ size() 需遍历整个队列，O(n)，慎用
    }
}
```

> **注意**：`size()` 是 O(n) 操作，高并发场景下结果可能不准确。判断空队列应使用 `isEmpty()`。

## 6.6 ConcurrentSkipListMap / ConcurrentSkipListSet

线程安全的**有序** Map/Set，基于跳表（Skip List）实现，查找/插入/删除均为 **O(log N)**。

```java
public class ConcurrentSkipListMapDemo {
    // 有序 Map
    private ConcurrentSkipListMap<String, Integer> sortedMap =
        new ConcurrentSkipListMap<>();

    // 有序 Set
    private ConcurrentSkipListSet<Integer> sortedSet =
        new ConcurrentSkipListSet<>();

    public void demo() {
        sortedMap.put("Banana", 2);
        sortedMap.put("Apple", 1);
        sortedMap.put("Cherry", 3);
        // 按 key 排序遍历：Apple → Banana → Cherry

        // 范围查询
        Map<String, Integer> sub = sortedMap.subMap("Apple", "Cherry");
        // head / tail
        Map<String, Integer> head = sortedMap.headMap("Banana"); // < Banana
        Map<String, Integer> tail = sortedMap.tailMap("Banana"); // >= Banana
    }
}
```

| 对比   | ConcurrentHashMap | ConcurrentSkipListMap |
|:-----|:------------------|:----------------------|
| 元素顺序 | ❌ 无序              | ✅ 有序                  |
| 性能   | O(1) 平均           | O(log N)              |
| 范围查询 | ❌ 不支持             | ✅ 支持                  |

## 6.7 并发集合选型决策树

```
需要并发 Map？
├── 需要排序/范围查询？ → ConcurrentSkipListMap
└── 无需排序？ → ConcurrentHashMap

需要并发 List？
└── 读多写少？ → CopyOnWriteArrayList

需要并发 Set？
├── 需要排序？ → ConcurrentSkipListSet
└── 无需排序？ → ConcurrentHashMap.newKeySet()

需要并发 Queue？
├── 需要阻塞等待（生产者-消费者）？ → BlockingQueue
│   ├── 需要优先级？ → PriorityBlockingQueue
│   ├── 固定容量？ → ArrayBlockingQueue
│   └── 高吞吐量？ → LinkedBlockingQueue（指定容量）
└── 纯高并发，无需阻塞？ → ConcurrentLinkedQueue
```
