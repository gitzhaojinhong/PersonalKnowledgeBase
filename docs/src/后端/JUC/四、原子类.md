## 4.1 基础原子类

### 4.1.1 原子类概述

JUC 原子类基于 **CAS（Compare-And-Swap）** 原理实现线程安全的无锁操作，核心是 `Unsafe` 类的 `compareAndSwapObject`、`compareAndSwapInt`、`compareAndSwapLong` 三个 native 方法。

CAS 原理：比较内存中当前值与期望值，相同则更新，否则返回 false，调用者重新尝试（自旋）。

### 4.1.2 原子整数

| 类 | 说明 | 典型使用 |
|:---|:---|:---|
| `AtomicInteger` | 原子整数 | 计数器、版本号 |
| `AtomicLong` | 原子长整数 | 大数值计数 |
| `AtomicBoolean` | 原子布尔 | 开关、标志位 |

```java
public class AtomicIntegerDemo {
    private AtomicInteger counter = new AtomicInteger(0);

    // 基本操作
    public void demo() {
        counter.set(10);              // 设置值
        counter.get();                // 获取值
        counter.getAndIncrement();    // 先获取再 +1（返回旧值）
        counter.incrementAndGet();    // 先 +1 再获取（返回新值）
        counter.getAndDecrement();    // 先获取再 -1
        counter.decrementAndGet();    // 先 -1 再获取
        counter.addAndGet(5);          // 加指定值并返回结果
        counter.getAndAdd(5);         // 先获取再增加指定值
        counter.updateAndGet(x -> x * 2); // 通过函数式接口更新
    }

    // 典型使用：计数器
    private final AtomicInteger requestCount = new AtomicInteger(0);

    public void handleRequest() {
        requestCount.incrementAndGet(); // 线程安全计数
    }
}
```

### 4.1.3 CAS的ABA问题及解决

**ABA 问题**：线程 A 将值从 V 改为 B，线程 B 又将 V 从 B 改回 A，线程 C 检查发现值仍为 A（表面相同，实际中间被修改过）。

```java
public class ABADemo {
    // 解决 ABA 问题：使用带版本号的 AtomicStampedReference
    private AtomicStampedReference<Integer> stampedRef =
        new AtomicStampedReference<>(100, 0);

    public void safeCompareAndSwap() {
        int stamp = stampedRef.getStamp();
        Integer reference = stampedRef.getReference();

        // 传入新值和新的 stamp 版本号
        stampedRef.compareAndSet(reference, 200, stamp, stamp + 1);
    }

    // 也可以用 AtomicMarkableReference（仅标记，不记录版本）
    private AtomicMarkableReference<Integer> markableRef =
        new AtomicMarkableReference<>(100, false);
}
```

public class AtomicReferenceDemo {

        userRef.set(newUser);                         // 设置

### 4.1.5 原子字段更新器

通过**反射**操作类的指定字段，避免为每个字段创建包装类（节省内存）。

```java
public class AtomicUpdaterDemo {
    // 必须是 volatile 的非静态字段
    private volatile long count = 0L;
    private static final AtomicLongFieldUpdater<AtomicUpdaterDemo> updater =
        AtomicLongFieldUpdater.newUpdater(AtomicUpdaterDemo.class, "count");

    public void increment() {
        updater.incrementAndGet(this);
    }
}
```

常用字段更新器：

| 类 | 操作字段类型 |
|:---|:---|
| `AtomicIntegerFieldUpdater` | int |
| `AtomicLongFieldUpdater` | long |
| `AtomicReferenceFieldUpdater` | 引用类型 |

### 4.1.6 原子数组

当需要对数组中的元素进行原子操作时，JUC 提供了三个原子数组类：

| 类 | 说明 |
|:---|:---|
| `AtomicIntegerArray` | 原子更新 int 数组中的元素 |
| `AtomicLongArray` | 原子更新 long 数组中的元素 |
| `AtomicReferenceArray` | 原子更新引用类型数组中的元素 |

```java
public class AtomicArrayDemo {

    // 原子整数数组
    private AtomicIntegerArray intArray = new AtomicIntegerArray(5);

    public void demo() {
        intArray.set(0, 10);              // 设置索引0的值为10
        intArray.get(0);                  // 获取索引0的值
        intArray.incrementAndGet(0);      // 索引0的值+1
        intArray.addAndGet(0, 5);         // 索引0的值+5
        intArray.compareAndSet(0, 15, 20); // CAS：索引0的值==15则改为20
    }

    // 典型场景：分片计数器（每个分片一个计数位）
    private AtomicLongArray shardCounter = new AtomicLongArray(16);

    public void incrementShard(int shardIndex) {
        shardCounter.incrementAndGet(shardIndex % 16);
    }

    public long getTotal() {
        long total = 0;
        for (int i = 0; i < shardCounter.length(); i++) {
            total += shardCounter.get(i);
        }
        return total;
    }
}
```

> **与 AtomicInteger[] 的区别**：`AtomicIntegerArray` 直接操作原始数组的内存地址，性能更高且内存占用更小；`AtomicInteger[]` 是为每个元素创建对象，开销大。

## 4.2 LongAdder / LongAccumulator

### 4.2.1 LongAdder核心原理

`LongAdder` 是 JDK 8 提供的高性能原子计数类，在**高并发场景**下性能远优于 `AtomicLong`。

| 实现 | 原理 | 适用场景 | 性能 |
|:---|:---|:---|:---|
| **AtomicLong** | 单一变量 + CAS 自旋 | 并发度低（< 1000） | 尚可 |
| **LongAdder** | 分散热点到多个 cell，减少竞争 | 高并发（≥ 1000） | 极高 |

`LongAdder` 内部维护一个 `Cell[]` 数组，每个线程操作自己对应的 cell，减少 CAS 竞争热点。最后求和时将所有 cell 累加。

```java
public class LongAdderDemo {
    private final LongAdder counter = new LongAdder();

    public void increment() {
        counter.increment(); // 线程安全，胜过高并发下的 AtomicLong++
    }

    public long getCount() {
        return counter.sum(); // 求和
    }

    // 典型场景：接口调用量统计
    private final LongAdder requestCounter = new LongAdder();
    private final LongAdder errorCounter = new LongAdder();

    public void recordRequest(boolean success) {
        requestCounter.increment();
        if (!success) errorCounter.increment();
    }
}
```

### 4.2.2 LongAccumulator

`LongAccumulator` 是 `LongAdder` 的通用版本，支持自定义累积函数。

```java
public class LongAccumulatorDemo {
    // 乘积累积器（初始值1）
    private LongAccumulator multiplier = new LongAccumulator((a, b) -> a * b, 1L);

    // 最大值累积器
    private LongAccumulator maxer = new LongAccumulator(Long::max, Long.MIN_VALUE);

    public void demo() {
        multiplier.accumulate(3L); // 1 * 3 = 3
        multiplier.accumulate(4L); // 3 * 4 = 12
        maxer.accumulate(100L);
        maxer.accumulate(50L);

        multiplier.sum();  // 12
        maxer.get();      // 100
    }
}
```

### 4.2.3 DoubleAdder与DoubleAccumulator

与 `LongAdder` / `LongAccumulator` 对称的双精度版本，适用于浮点数统计场景（如金额累加、收益计算）。

```java
public class DoubleAdderDemo {

    // 高并发下浮点累加
    private final DoubleAdder totalRevenue = new DoubleAdder();

    public void recordSale(double amount) {
        totalRevenue.add(amount); // 线程安全累加
    }

    public double getTotal() {
        return totalRevenue.sum(); // 获取总和
    }

    // DoubleAccumulator：支持自定义聚合函数
    public void accumulatorDemo() {
        DoubleAccumulator maxPrice = new DoubleAccumulator(Double::max, 0.0);
        maxPrice.accumulate(99.99);
        maxPrice.accumulate(188.50);
        maxPrice.accumulate(55.00);
        System.out.println("最高价格: " + maxPrice.get()); // 188.50
    }
}
```

| 原子类 | 数据类型 | 适用场景 |
|:---|:---|:---|
| `LongAdder` | long | 整数计数、QPS 统计 |
| `DoubleAdder` | double | 金额累加、浮点统计 |
| `LongAccumulator` | long | 自定义整数聚合（max/min/乘积等） |
| `DoubleAccumulator` | double | 自定义浮点聚合 |

> **注意**：`DoubleAdder` / `DoubleAccumulator` 基于 `LongAdder` 相同的 Cell 分散热点机制，高并发下性能远优于 `AtomicLong` + 手动转换。

## 4.3 ThreadLocalRandom

### 4.3.1 为什么需要ThreadLocalRandom

`java.util.Random` 是线程安全的（内部使用 `AtomicLong` + CAS），但在**高并发场景**下多个线程竞争同一个种子变量，导致大量 CAS 自旋，性能急剧下降。

`ThreadLocalRandom`（JDK 7 引入）为**每个线程维护独立的随机种子**，彻底消除了竞争，高并发下性能可提升 5~10 倍。

```java
public class ThreadLocalRandomDemo {

    // ❌ 高并发下性能差
    private Random random = new Random();

    // ✅ 高并发下性能极好
    public void generateRandom() {
        // 获取当前线程的 ThreadLocalRandom 实例
        int num = ThreadLocalRandom.current().nextInt(100);      // [0, 100)
        long longNum = ThreadLocalRandom.current().nextLong(1000);
        double doubleNum = ThreadLocalRandom.current().nextDouble();

        // 生成指定范围的随机数
        int rangeNum = ThreadLocalRandom.current().nextInt(10, 100); // [10, 100)
    }

    // 典型场景：随机负载均衡
    public String selectServer(List<String> servers) {
        int index = ThreadLocalRandom.current().nextInt(servers.size());
        return servers.get(index);
    }
}
```

### 4.3.2 与Random性能对比

| 维度 | Random | ThreadLocalRandom |
|:---|:---|:---|
| **线程安全** | ✅ 是（AtomicLong + CAS） | ✅ 是（线程隔离） |
| **高并发性能** | ❌ 差（大量 CAS 竞争） | ✅ 极好（无竞争） |
| **使用方式** | `new Random()` | `ThreadLocalRandom.current()` |
| **适用场景** | 单线程或低并发 | 高并发随机数生成 |

> **注意**：`ThreadLocalRandom` 不能手动创建实例，必须通过 `ThreadLocalRandom.current()` 获取。
