## 10.1 死锁

### 10.1.1 死锁的必要条件

| 条件 | 说明 |
|:---|:---|
| **互斥** | 资源同一时刻只能被一个线程持有 |
| **占有且等待** | 线程持有资源的同时等待其他资源 |
| **不可抢占** | 已持有的资源在未使用完毕前不能被强制夺走 |
| **循环等待** | 线程之间形成资源循环等待链 |

### 10.1.2 死锁示例

```java
public class DeadLockDemo {

    private final Object lockA = new Object();
    private final Object lockB = new Object();

    // 线程1：先拿A再拿B
    public void thread1() {
        synchronized (lockA) {
            System.out.println("线程1持有锁A，等待锁B...");
            synchronized (lockB) {
                System.out.println("线程1同时持有A和B");
            }
        }
    }

    // 线程2：先拿B再拿A（与线程1形成循环等待）
    public void thread2() {
        synchronized (lockB) {
            System.out.println("线程2持有锁B，等待锁A...");
            synchronized (lockA) {
                System.out.println("线程2同时持有A和B");
            }
        }
    }

    public static void main(String[] args) {
        DeadLockDemo demo = new DeadLockDemo();
        new Thread(demo::thread1).start();
        new Thread(demo::thread2).start();
        // 两个线程互相等待对方释放锁，形成死锁
    }
}
```

### 10.1.3 死锁检测与解决

**① JDK 自带工具检测**：

```bash
# 列出所有 Java 进程
jps -l

# 打印线程堆栈，定位死锁
jstack <pid>
# 输出中包含 "Found 1 deadlock." 即为死锁
```

**② 避免死锁的策略**：

| 策略 | 说明 |
|:---|:---|
| **固定加锁顺序** | 所有线程按相同顺序获取锁 |
| **Lock.tryLock() 超时** | 无法获取锁时回退或重试 |
| **减少锁粒度** | 避免一个线程长时间持有多个锁 |
| **ThreadLocal 破环共享** | 减少对共享资源的竞争 |

**③ 固定加锁顺序解决死锁**：

```java
public class FixedOrderLocking {

    // 无论哪个线程，都按 lockA → lockB 的顺序获取
    public void doSomething() {
        synchronized (lockA) {
            synchronized (lockB) {
                // 业务逻辑
            }
        }
    }

    // 或者使用 tryLock 超时回退
    public void tryLockApproach() {
        ReentrantLock lockA = new ReentrantLock();
        ReentrantLock lockB = new ReentrantLock();

        while (true) {
            if (lockA.tryLock()) {
                try {
                    if (lockB.tryLock()) {
                        try {
                            // 业务逻辑
                            break;
                        } finally {
                            lockB.unlock();
                        }
                    }
                } finally {
                    lockA.unlock();
                }
            }
            Thread.sleep(10); // 短暂等待后重试
        }
    }
}
```

## 10.2 活锁与饥饿

### 10.2.1 活锁

线程虽然没有阻塞，但因不断重试某个操作而**始终无法前进**。

```java
public class LivelockDemo {

    // 活锁示例：两个线程互相礼让，但始终无法前进
    private final AtomicBoolean flag = new AtomicBoolean(true);

    public void livelockExample() {
        Thread t1 = new Thread(() -> {
            while (flag.get()) {
                System.out.println("线程1准备处理...");
                // 模拟检测到冲突就重试
                if (flag.compareAndSet(true, false)) {
                    // 处理成功
                }
                // 礼让：立即重置标志
                flag.set(true);
            }
        });

        Thread t2 = new Thread(() -> {
            while (flag.get()) {
                System.out.println("线程2准备处理...");
                if (flag.compareAndSet(true, false)) {
                    // 处理成功
                }
                flag.set(true);
            }
        });

        t1.start();
        t2.start();
    }

    // 解决活锁：添加随机退避
    public void fixedLivelock() {
        Random random = new Random();
        while (true) {
            try {
                // 随机等待，避免同时冲突
                Thread.sleep(random.nextInt(100));
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            }
        }
    }
}
```

### 10.2.2 饥饿

线程因优先级低或锁竞争激烈而**长期无法获得 CPU 时间片或锁**。

| 问题 | 原因 | 解决方案 |
|:---|:---|:---|
| **CPU 饥饿** | 低优先级线程被高优先级线程占满 CPU | 合理设置优先级，避免设置过低 |
| **锁饥饿** | 高并发下某些线程总抢不到锁 | 使用公平锁（`ReentrantLock(true)`） |

```java
// 公平锁解决饥饿
public class FairLockDemo {
    private final ReentrantLock fairLock = new ReentrantLock(true); // 公平锁

    public void fairWork() {
        fairLock.lock();
        try {
            // 严格按申请顺序获取锁，不会饥饿
        } finally {
            fairLock.unlock();
        }
    }
}
```

### 10.2.3 死锁、活锁、饥饿对比

| 特性 | 死锁 | 活锁 | 饥饿 |
|:---|:---|:---|:---|
| **线程状态** | 阻塞（BLOCKED/WAITING） | 运行中（RUNNABLE，但空转） | 阻塞或低优先级运行 |
| **是否消耗 CPU** | ❌ 否（线程休眠） | ✅ 是（持续自旋） | ❌ 否 |
| **能否自行恢复** | ❌ 不能（需外部干预） | ✅ 可能自行恢复（概率低） | ✅ 可能（取决于调度） |
| **危害** | 线程永久挂起 | CPU 空转浪费资源 | 任务长期得不到执行 |
