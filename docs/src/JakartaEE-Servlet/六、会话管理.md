## 六、会话管理

### 6.1 会话概述

HTTP是无状态协议，会话技术用于跟踪用户状态。

**常用会话技术：**
- **Cookie**：客户端存储
- **Session**：服务器端存储
- **Token**：令牌机制

### 6.2 Cookie

Cookie是服务器发送到浏览器的小型文本数据，浏览器会保存并在后续请求中携带。

#### Cookie基本操作

```java
// 创建Cookie
Cookie cookie = new Cookie("username", "admin");

// 设置有效期（秒）
cookie.setMaxAge(60 * 60 * 24 * 7);  // 7天

// 设置路径
cookie.setPath("/");

// 设置HttpOnly（禁止JS访问）
cookie.setHttpOnly(true);

// 设置Secure（仅HTTPS传输）
cookie.setSecure(true);

// 添加到响应
response.addCookie(cookie);
```

#### 读取Cookie

```java
// 获取所有Cookie
Cookie[] cookies = request.getCookies();
if (cookies != null) {
    for (Cookie cookie : cookies) {
        if ("username".equals(cookie.getName())) {
            String value = cookie.getValue();
            System.out.println("用户名：" + value);
        }
    }
}
```

#### 删除Cookie

```java
Cookie cookie = new Cookie("username", "");
cookie.setMaxAge(0);  // 立即过期
cookie.setPath("/");
response.addCookie(cookie);
```

#### Cookie属性详解

| 属性 | 说明 | 示例 |
|------|------|------|
| `name` | Cookie名称 | `sessionId` |
| `value` | Cookie值 | `abc123` |
| `maxAge` | 有效期（秒） | `3600`（1小时） |
| `path` | 生效路径 | `/`（全站） |
| `domain` | 生效域名 | `.example.com` |
| `httpOnly` | 禁止JS访问 | `true`（防XSS） |
| `secure` | 仅HTTPS传输 | `true` |
| `sameSite` | 跨站请求限制 | `Strict/Lax/None` |

### 6.3 Session

Session是服务器端保存用户会话数据的机制。

#### Session基本操作

```java
// 获取Session（不存在则创建）
HttpSession session = request.getSession();

// 获取Session（不存在返回null）
HttpSession session = request.getSession(false);

// 存储数据
session.setAttribute("user", user);
session.setAttribute("cart", cart);

// 获取数据
User user = (User) session.getAttribute("user");

// 移除数据
session.removeAttribute("tempData");

// 获取Session ID
String sessionId = session.getId();

// 获取创建时间
long createTime = session.getCreationTime();

// 获取最后访问时间
long lastAccess = session.getLastAccessedTime();

// 使Session失效（退出登录）
session.invalidate();
```

#### Session配置

```xml
<!-- web.xml -->
<session-config>
    <!-- 超时时间（分钟） -->
    <session-timeout>30</session-timeout>
    
    <!-- Cookie配置 -->
    <cookie-config>
        <name>MYSESSIONID</name>
        <http-only>true</http-only>
        <secure>true</secure>
    </cookie-config>
</session-config>
```

#### Session工作原理

```
1. 首次请求：服务器创建Session → 生成JSESSIONID → 通过Cookie返回给浏览器
2. 后续请求：浏览器携带JSESSIONID Cookie → 服务器根据ID找到对应Session
3. 数据存储：Session数据保存在服务器内存中
```

#### Session与Cookie对比

| 特性 | Cookie | Session |
|------|--------|---------|
| 存储位置 | 客户端（浏览器） | 服务器端 |
| 安全性 | 较低（可被篡改） | 较高 |
| 存储大小 | 单个4KB，总数有限制 | 无限制（受服务器内存） |
| 生命周期 | 可长期保存 | 默认30分钟无访问过期 |
| 性能影响 | 每次请求携带，增加流量 | 服务器内存占用 |
| 适用场景 | 记住用户名、跟踪标识 | 登录状态、购物车 |

### 6.4 记住我功能（Remember Me）

使用持久化Token实现自动登录功能。

#### 数据库设计

```sql
CREATE TABLE user_remember_tokens (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT NOT NULL,
    token VARCHAR(255) NOT NULL UNIQUE,
    series VARCHAR(255) NOT NULL,
    fingerprint VARCHAR(255) NOT NULL,
    expires_at TIMESTAMP NOT NULL,
    revoked TINYINT DEFAULT 0,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### 登录时生成Token

```java
@WebServlet("/login")
public class LoginServlet extends HttpServlet {
    
    @Override
    protected void doPost(HttpServletRequest req, HttpServletResponse resp) 
            throws ServletException, IOException {
        
        String username = req.getParameter("username");
        String password = req.getParameter("password");
        boolean rememberMe = "on".equals(req.getParameter("rememberMe"));
        
        // 验证用户名密码...
        
        if (rememberMe) {
            // 生成Token
            String token = UUID.randomUUID().toString();
            String series = UUID.randomUUID().toString();
            
            // 生成设备指纹
            String ip = req.getRemoteAddr();
            String userAgent = req.getHeader("User-Agent");
            String fingerprint = DigestUtils.sha256Hex(ip + userAgent + "salt");
            
            // 保存到数据库
            saveTokenToDB(userId, token, series, fingerprint);
            
            // 写入Cookie
            Cookie cookie = new Cookie("remember_token", token);
            cookie.setMaxAge(60 * 60 * 24 * 30);  // 30天
            cookie.setPath(req.getContextPath());
            cookie.setHttpOnly(true);
            if (req.isSecure()) {
                cookie.setSecure(true);
            }
            resp.addCookie(cookie);
        }
        
        // 创建Session...
    }
}
```

#### 过滤器自动验证Token

```java
@WebFilter("/*")
public class AutoLoginFilter implements Filter {
    
    @Override
    public void doFilter(ServletRequest req, ServletResponse resp, FilterChain chain) 
            throws IOException, ServletException {
        
        HttpServletRequest request = (HttpServletRequest) req;
        HttpServletResponse response = (HttpServletResponse) resp;
        HttpSession session = request.getSession(false);
        
        // 已登录，放行
        if (session != null && session.getAttribute("user") != null) {
            chain.doFilter(req, resp);
            return;
        }
        
        // 检查Remember Token
        Cookie[] cookies = request.getCookies();
        if (cookies != null) {
            for (Cookie cookie : cookies) {
                if ("remember_token".equals(cookie.getName())) {
                    String token = cookie.getValue();
                    
                    // 验证Token有效性
                    if (validateToken(token, request)) {
                        // 自动登录，创建Session
                        User user = getUserByToken(token);
                        HttpSession newSession = request.getSession(true);
                        newSession.setAttribute("user", user);
                    } else {
                        // Token无效，删除Cookie
                        deleteCookie(response, cookie);
                    }
                    break;
                }
            }
        }
        
        chain.doFilter(req, resp);
    }
}
```
