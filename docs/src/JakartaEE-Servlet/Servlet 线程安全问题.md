## Servlet 线程安全问题
### Tomcat 线程池
#### 什么是线程池
线程池（Thread Pool）是一种**多线程处理机制**，它预先创建一组线程并管理它们的生命周期，避免频繁创建和销毁线程带来的性能开销。线程池的核心思想是**复用线程**，减少系统资源消耗，提高任务处理效率。  

#### 线程池的作用
1. **降低资源消耗**：复用线程，减少频繁创建和销毁线程的开销。  
2. **提高响应速度**：任务到达时，可直接使用空闲线程执行，无需等待线程创建。  
3. **控制并发数量**：防止线程过多导致系统崩溃（如OOM）。  
4. **统一管理线程**：提供任务队列、拒绝策略等机制，增强系统稳定性。

#### Tomcat 线程池的配置
在 `conf/server.xml` 中，`<Connector>` 标签可以配置线程池参数，例如：  

```xml
<Connector 
    port="8080" 
    protocol="HTTP/1.1"
    maxThreads="200"       <!-- 最大线程数 -->
    minSpareThreads="10"   <!-- 最小空闲线程数 -->
    acceptCount="100"      <!-- 等待队列长度 -->
    connectionTimeout="20000"
/>
```

#### 关键参数说明
| **参数** | **说明** |
| --- | --- |
| `maxThreads` | 最大线程数（默认200），决定Tomcat的并发处理能力 |
| `minSpareThreads` | 最小空闲线程数（默认10），避免请求到来时临时创建线程 |
| `acceptCount` | 当所有线程都在忙时，新请求进入等待队列的最大数量（默认100） |
| `connectionTimeout` | 连接超时时间（毫秒） |


#### Tomcat 线程池的工作流程
1. 客户端发送 HTTP 请求到 Tomcat。  
2. Tomcat 从线程池中分配一个空闲线程处理请求。  
3. 如果所有线程都在忙，新请求进入等待队列（`acceptCount`）。  
4. 如果队列已满，Tomcat 会拒绝请求（返回 `503 Service Unavailable`）。

### Servlet 线程安全问题
1. Servlet 是单实例的。
2. Web 服务器都是支持多线程并发的，Web 服务器内部一般会提供线程池。例如 Tomcat 服务器中就有线程池。
3. 因此 Servlet 是在单实例多线程的环境下运行的，如果 Servlet 中定义成员变量，并且成员变量涉及到修改操作，必然存在线程安全问题。
4. 实际开发中不建议在 Servlet 中定义成员变量，尽量使用局部变量。如果使用成员变量，尽量不要做修改操作，如果做修改操作，需要使用线程同步机制保证数据的安全。
