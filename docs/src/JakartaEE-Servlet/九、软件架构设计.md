
## 9.1 三层架构

三层架构将应用划分为表示层、业务层、持久层。

```
浏览器 → 表示层(Controller/Servlet) → 业务层(Service) → 持久层(DAO) → 数据库
```

### 各层职责

| 层次 | 职责 | 对应类 |
|------|------|--------|
| 表示层 | 接收请求、参数校验、调用业务、返回响应 | Servlet、Controller |
| 业务层 | 业务逻辑、事务控制、数据校验 | Service |
| 持久层 | 数据库操作、SQL执行 | DAO、Mapper |

### 包结构规范

```
com.example.project
├── common/                 # 公共类
│   ├── util/              # 工具类
│   ├── constant/          # 常量
│   └── exception/         # 异常
├── controller/            # 表示层（Servlet）
├── service/               # 业务层接口
│   └── impl/              # 业务层实现
├── dao/                   # 持久层接口
│   └── impl/              # 持久层实现
├── entity/                # 实体类（PO）
├── dto/                   # 数据传输对象
└── vo/                    # 视图对象
```

## 9.2 常用对象类型

| 类型 | 全称 | 说明 |
|------|------|------|
| POJO | Plain Old Java Object | 普通Java对象，不依赖框架 |
| JavaBean | - | 符合规范的POJO（无参构造、getter/setter、Serializable） |
| PO | Persistent Object | 持久对象，与数据库表对应 |
| Entity | - | 领域实体，可能包含业务逻辑 |
| DTO | Data Transfer Object | 数据传输对象，用于层间传递 |
| VO | View Object | 视图对象，用于前端展示 |
| DAO | Data Access Object | 数据访问对象，封装数据库操作 |

## 9.4 ThreadLocal解决事务一致性问题

### 问题场景

Service 中多次调用 DAO，每次 DAO 方法执行都新建 Connection 对象，JDBC 默认每条 DML 自动提交，导致多个操作各自提交、无法回滚——钱转出去了，但另一边没转进来。

**根本原因：** Service 方法中多个 DAO 操作使用了不同的 Connection，无法控制为同一事务。

### 解决方案：ThreadLocal共享Connection

`ThreadLocal` 的作用：在**同一个线程**中共享同一个 Java 对象（这里是 Connection）。

```java
// ThreadLocal三个核心方法
void set(Object data);    // 向当前线程绑定数据
Object get();             // 获取当前线程绑定的数据
void remove();            // 清除当前线程的数据（防止内存泄漏）
```

**改造DbUtils工具类：**

```java
public class DbUtils {
    // 每个线程独立持有自己的Connection
    private static ThreadLocal<Connection> local = new ThreadLocal<>();

    public static Connection getConnection() throws SQLException {
        Connection conn = local.get();
        if (conn == null) {
            conn = DriverManager.getConnection(url, user, password);
            local.set(conn);  // 绑定到当前线程
        }
        return conn;  // 同一线程多次调用，返回同一个Connection
    }

    public static void close(Connection conn, Statement stmt, ResultSet rs) {
        // ... 关闭rs、stmt
        if (conn != null) {
            conn.close();
            local.remove();  // 关闭连接后必须清除ThreadLocal，防止泄漏
        }
    }
}
```

**Service层控制事务（完整流程）：**

```java
public void transfer(String fromActno, String toActno, double money) {
    Connection conn = null;
    try {
        // 1. 获取连接（ThreadLocal保证同一个Connection）
        conn = DbUtils.getConnection();
        // 2. 关闭自动提交，手动开启事务
        conn.setAutoCommit(false);

        // 3. 多个DAO操作，共用同一Connection
        dao.deductMoney(fromActno, money);   // 扣钱（不提交、不关连接）
        // 模拟异常：int i = 1 / 0;
        dao.addMoney(toActno, money);        // 加钱（不提交、不关连接）

        // 4. 全部成功 → 提交事务
        conn.commit();

    } catch (Exception e) {
        // 5. 出现异常 → 回滚事务
        if (conn != null) {
            try { conn.rollback(); } catch (SQLException ex) { ex.printStackTrace(); }
        }
        throw new RuntimeException(e);
    } finally {
        // 6. 统一关闭连接 + 清空ThreadLocal
        DbUtils.close(conn, null, null);
    }
}
```

> **注意：** DAO 方法中不能关闭 Connection，只能关闭 Statement 和 ResultSet，Connection 的关闭由 Service 层统一处理。
