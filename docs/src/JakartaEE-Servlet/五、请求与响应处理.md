## 五、请求与响应处理

### 5.1 获取请求参数

#### 获取单个参数

```java
// 获取用户名
String username = request.getParameter("username");

// 处理可能为null的情况
String keyword = request.getParameter("keyword");
if (keyword == null) {
    keyword = "";
}
```

#### 获取多个值（如复选框）

```java
// 获取选中的兴趣爱好（多个值）
String[] hobbies = request.getParameterValues("hobby");
if (hobbies != null) {
    for (String hobby : hobbies) {
        System.out.println(hobby);
    }
}
```

#### 获取所有参数

```java
// 获取所有参数名
Enumeration<String> names = request.getParameterNames();
while (names.hasMoreElements()) {
    String name = names.nextElement();
    String value = request.getParameter(name);
    System.out.println(name + " = " + value);
}

// 获取参数Map
Map<String, String[]> paramMap = request.getParameterMap();
```

### 5.2 请求属性操作（request域）

request 是**请求域**，生命周期短：从浏览器发起请求开始，到服务器响应结束为止。新的请求对应新的 request 对象，因此请求域只在同一次请求中有效。

```java
// 设置属性（用于Servlet间数据传递）
request.setAttribute("user", user);
request.setAttribute("errorMsg", "用户名或密码错误");

// 获取属性
User user = (User) request.getAttribute("user");
String errorMsg = (String) request.getAttribute("errorMsg");

// 移除属性
request.removeAttribute("tempData");
```

**域对象选择原则：优先选择最小的域，小的满足不了再选大的：**
- `request` 域：同一次请求内有效（最小，最优先）
- `session` 域：同一会话内有效
- `application` 域：整个应用有效（最大，最后选）

**验证request域是否为一次请求：** 转发时 request 域数据可以共享（一次请求），重定向时 request 域数据无法共享（两次请求）。




### 5.3 请求转发与重定向

#### 请求转发（Forward）

服务器内部跳转，URL不变，可共享request属性。

```java
// 转发到JSP页面
request.getRequestDispatcher("/user.jsp").forward(request, response);

// 转发到Servlet
request.getRequestDispatcher("/list").forward(request, response);
```

**特点：**
- 服务器内部跳转，浏览器无感知
- URL地址栏不变
- 可以共享request中的属性
- 只能转发到同一应用内的资源
- 效率高（一次请求）

#### 重定向（Redirect）

服务器通知浏览器重新请求新地址。

```java
// 重定向到登录页面
response.sendRedirect(request.getContextPath() + "/login");

// 重定向到外部网站
response.sendRedirect("https://www.example.com");
```

**特点：**
- 浏览器发起新请求
- URL地址栏会改变
- 不能共享request属性（新的请求）
- 可以跳转到任意URL
- 效率较低（两次请求）

#### 对比总结

| 特性 | 转发(Forward) | 重定向(Redirect) |
|------|---------------|------------------|
| 跳转方式 | 服务器内部 | 浏览器重新请求 |
| URL变化 | 不变 | 改变 |
| 共享数据 | 可共享request | 不可共享 |
| 跳转范围 | 同一应用内 | 任意URL |
| 请求次数 | 1次 | 2次 |
| 适用场景 | 内部页面跳转 | 登录后跳转、跨域跳转 |

### 5.4 响应输出

#### 输出文本/HTML

```java
// 设置内容类型和编码
response.setContentType("text/html;charset=UTF-8");

// 获取输出流
PrintWriter out = response.getWriter();
out.println("<!DOCTYPE html>");
out.println("<html><head><title>首页</title></head>");
out.println("<body><h1>欢迎访问</h1></body></html>");
```

#### 输出JSON

```java
// 设置JSON内容类型
response.setContentType("application/json;charset=UTF-8");

// 输出JSON字符串
PrintWriter out = response.getWriter();
out.println("{\"code\":200,\"msg\":\"success\",\"data\":{\"id\":1}}");
```

#### 设置状态码

```java
// 设置404状态
response.setStatus(HttpServletResponse.SC_NOT_FOUND);

// 发送错误页面
response.sendError(HttpServletResponse.SC_INTERNAL_SERVER_ERROR, "服务器内部错误");
```

### 5.5 文件上传处理

#### 页面表单

```html
<form action="/upload" method="post" enctype="multipart/form-data">
    <input type="file" name="file" accept=".jpg,.png">
    <button type="submit">上传</button>
</form>
```

**关键点：**
- `method="post"`：必须是POST
- `enctype="multipart/form-data"`：必须设置

#### Servlet处理

```java
@WebServlet("/upload")
@MultipartConfig(                          // 配置上传参数
    fileSizeThreshold = 1024 * 1024,       // 临时文件阈值（1MB）
    maxFileSize = 1024 * 1024 * 10,        // 单个文件最大（10MB）
    maxRequestSize = 1024 * 1024 * 50      // 总请求最大（50MB）
)
public class UploadServlet extends HttpServlet {
    
    @Override
    protected void doPost(HttpServletRequest req, HttpServletResponse resp) 
            throws ServletException, IOException {
        
        // 获取上传的文件
        Part filePart = req.getPart("file");
        
        // 获取文件名
        String fileName = filePart.getSubmittedFileName();
        
        // 保存路径
        String uploadPath = getServletContext().getRealPath("/uploads");
        File uploadDir = new File(uploadPath);
        if (!uploadDir.exists()) {
            uploadDir.mkdirs();
        }
        
        // 保存文件
        filePart.write(uploadPath + File.separator + fileName);
        
        resp.getWriter().write("上传成功：" + fileName);
    }
}
```

### 5.6 文件下载

```java
@WebServlet("/download")
public class DownloadServlet extends HttpServlet {
    
    @Override
    protected void doGet(HttpServletRequest req, HttpServletResponse resp) 
            throws ServletException, IOException {
        
        // 获取文件名
        String fileName = req.getParameter("file");
        
        // 文件路径
        String filePath = getServletContext().getRealPath("/uploads/" + fileName);
        File file = new File(filePath);
        
        // 设置响应头
        resp.setContentType("application/octet-stream");
        resp.setHeader("Content-Disposition", "attachment; filename=" + fileName);
        resp.setContentLength((int) file.length());
        
        // 输出文件内容
        try (FileInputStream fis = new FileInputStream(file);
             OutputStream os = resp.getOutputStream()) {
            byte[] buffer = new byte[4096];
            int len;
            while ((len = fis.read(buffer)) != -1) {
                os.write(buffer, 0, len);
            }
        }
    }
}
```
