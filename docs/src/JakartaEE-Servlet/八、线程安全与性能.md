
## 8.1 Tomcat线程池

Tomcat使用线程池处理并发请求。

### 线程池配置

```xml
<Connector 
    port="8080" 
    protocol="HTTP/1.1"
    maxThreads="200"          <!-- 最大线程数 -->
    minSpareThreads="10"      <!-- 最小空闲线程 -->
    maxConnections="10000"    <!-- 最大连接数 -->
    acceptCount="100"         <!-- 等待队列长度 -->
    connectionTimeout="20000" <!-- 连接超时（毫秒） -->
/>
```

**参数说明：**

| 参数 | 说明 | 默认值 | 调优建议 |
|------|------|--------|----------|
| `maxThreads` | 最大工作线程数 | 200 | CPU密集型：核数+1；IO密集型：核数*2 |
| `minSpareThreads` | 最小空闲线程 | 10 | 根据并发量调整 |
| `acceptCount` | 等待队列长度 | 100 | 队列满后拒绝连接 |
| `connectionTimeout` | 连接超时 | 20000 | 毫秒 |

### 线程池工作流程

```
1. 请求到达
2. 有空闲线程？→ 分配线程处理
3. 无空闲线程？→ 进入等待队列
4. 队列已满？→ 拒绝请求（返回503）
```

## 8.2 Servlet线程安全问题

**核心问题：** Servlet是单实例多线程运行的。

```java
@WebServlet("/counter")
public class CounterServlet extends HttpServlet {
    
    // 成员变量：线程不安全！
    private int count = 0;
    
    @Override
    protected void doGet(HttpServletRequest req, HttpServletResponse resp) 
            throws ServletException, IOException {
        
        // 局部变量：线程安全
        int localCount = 0;
        
        count++;           // 危险！多线程竞争
        localCount++;      // 安全
        
        resp.getWriter().write("Count: " + count);
    }
}
```

### 线程安全解决方案

**方案一：使用局部变量（推荐）**

```java
@Override
protected void doGet(HttpServletRequest req, HttpServletResponse resp) 
        throws ServletException, IOException {
    
    // 所有数据使用局部变量
    String param = req.getParameter("id");
    int result = process(param);
    resp.getWriter().write(String.valueOf(result));
}
```

**方案二：使用ThreadLocal**

```java
public class UserContext {
    private static final ThreadLocal<User> currentUser = new ThreadLocal<>();
    
    public static void set(User user) {
        currentUser.set(user);
    }
    
    public static User get() {
        return currentUser.get();
    }
    
    public static void remove() {
        currentUser.remove();
    }
}
```

**方案三：同步代码块（慎用）**

```java
private final Object lock = new Object();

@Override
protected void doGet(HttpServletRequest req, HttpServletResponse resp) 
        throws ServletException, IOException {
    
    synchronized (lock) {
        // 临界区代码
        count++;
    }
}
```

**最佳实践：**
1. 避免在Servlet中定义成员变量
2. 如需共享数据，使用ServletContext或数据库
3. 必须使用成员变量时，确保只读或使用同步机制
